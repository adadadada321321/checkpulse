Shopify Gift Card Balance Not Showing at Checkout — 8 Causes Explained

Gift card applied but showing wrong amount? Remaining balance invisible after partial use? Can't add a second gift card? Here are all 8 root causes — for customers and merchants debugging gift card balance display at Shopify checkout.

Updated June 2026 8 failure modes Partial balance · Multi-currency · Multiple cards · Store-specific codes · Headless API

Shopify's gift card system works differently from most e-commerce platforms. The checkout does not display a gift card's balance before the customer enters the code — it only reveals the available balance after the code is typed and applied. After applying, the amount deducted from the order is shown, but the remaining balance left on the card is not prominently displayed anywhere in the checkout flow.

This design creates predictable confusion: customers don't know how much they have before committing, partial-use balances are invisible after checkout, and multi-store or multi-currency scenarios produce unexpected numbers. There are 8 distinct failure modes, and most of them are misdiagnosed as checkout bugs when they're actually working as designed.

Gift Cards vs Discount Codes vs Store Credit
These three are separate systems in Shopify with different UIs and different rules. Gift cards are pre-loaded monetary values applied from the checkout gift card field — they reduce the order total and can partially cover it. Discount codes are promotional reductions applied from the discount code field — they stack differently with automatic discounts. Store credit is issued per-customer account and applied automatically on Plus plans or via apps like Rise.ai. This guide covers gift card balance display specifically. For gift card + discount stacking conflicts, see the Gift Card & Loyalty Stacking guide.

Failure Mode 1: No Balance Display Before Entering the Code

Shopify's hosted checkout does not show a gift card's current balance until the customer enters the code and taps "Apply." This is the expected behavior — it is not a bug. But customers who received a gift card months ago, or who partially used a gift card on a prior order, don't know how much is available before they start checkout.

How customers can check balance before checkout

Method Who Can Use It How
Gift card email link Customer The original gift card confirmation email contains a link to a Shopify-hosted gift card page that shows the current balance including any prior deductions
Admin gift card lookup Merchant Admin → Discounts → Gift cards → search by last 4 digits or full code — shows original value, used amount, and current balance
Custom balance check page Customer (if merchant built one) Some merchants build a Liquid-based balance check form using the /gift_cards/[code] endpoint — requires custom theme development
Enter code at checkout Customer Apply the code at checkout — Shopify displays the balance deducted and the remaining amount to pay; this is the simplest balance reveal
After Purchase: Where Does the Remaining Balance Show?
After a successful order, Shopify sends the customer a gift card usage confirmation showing the amount used and the remaining balance on the card. This is sent as part of the order confirmation email only if the merchant has not disabled the gift card usage section of the email template. If customers report "not knowing what's left," check Admin → Settings → Notifications → Gift card used email.

Failure Mode 2: Partial-Use Balance Invisible — Remaining Balance After Prior Orders

A customer received a $100 gift card and spent $35 on a previous order. The remaining balance is $65. When they return to the store and apply the same code, Shopify applies up to $65 toward the new order — but the checkout doesn't explain the opening balance or why it isn't $100 anymore.

Why this creates support tickets

  • Customer sees "$65 applied" and remembers they had $100 — assumes the store "took" their balance
  • There is no balance history visible to customers — they cannot see a transaction log of gift card use
  • Shopify does not send a balance-updated notification after each partial use; the only record is the original "Gift card used" email for that specific order
  • If the customer lost the original gift card email, they have no self-serve way to view their usage history

How merchants can help customers with partial-balance confusion

  1. Look up the gift card in Admin → Discounts → Gift cards and share the transaction history with the customer — the admin view shows every order where the card was used
  2. If the card is genuinely showing the wrong balance, check whether another device or person used the code — gift card codes are not account-bound
  3. Issue a replacement gift card for the correct amount if the original was used without the customer's knowledge

Failure Mode 3: Multiple Gift Cards — Second Card Field Not Visible

Shopify supports up to 8 gift cards per order. However, the checkout UI shows only a single "Gift card or discount code" input field. After the customer applies the first gift card and the order total is not fully covered, a second gift card input field appears below the first. This progressive reveal is not documented in the UI, so customers routinely assume only one gift card is accepted and abandon checkout.

The progressive reveal flow

Scenario What Happens Customer Impact
Gift card fully covers order ($50 card, $40 order) Card applied, remaining $10 stays on card, order completes — no second field needed Smooth — customer sees "$40 applied, $10 remaining on card"
Gift card partially covers order ($50 card, $80 order) $50 applied, remaining $30 to pay — second gift card field appears below first Confusing — customer often doesn't notice the second field or assumes they must pay $30 with a credit card
Customer has two gift cards both needed Must apply first, then notice second field, then apply second High abandonment if customer doesn't realize second field is available
Order total $0 after all gift cards applied Checkout allows $0 payment — no credit card required Works correctly — customer can complete $0 checkout without a payment method
How to Reduce Multiple-Card Abandonment
Add a small note near the gift card input — "You can apply multiple gift cards. Enter one at a time." — via a Checkout UI Extension banner (Plus) or by placing a note in the order summary section of your theme's cart page. Since customers arrive at checkout from the cart, a note on the cart page is often more effective than one in the checkout itself.

Failure Mode 4: Gift Card Code Works on Some Orders But Not Others — Store-Specific Restriction

Shopify gift cards are store-specific. A gift card issued by one Shopify store cannot be redeemed at any other Shopify store, including stores owned by the same merchant. The code will show "Gift card not found" or "Invalid gift card" on the wrong store.

Common scenarios that trigger this error

  • Merchant operates a US store and a UK store — gift cards issued on the US store are invalid on the UK store
  • Merchant rebranded or migrated to a new Shopify store — old gift cards from the previous store are invalid on the new one
  • Customer received a gift card from a brand's marketplace or wholesale portal running on a separate Shopify account
  • A third-party reseller sold "gift cards" that are actually promotional codes — these are discount codes, not Shopify gift card codes, and behave differently

Fix: cross-store gift cards require a third-party app

Shopify's native gift card system has no cross-store functionality. Merchants who need gift cards valid across multiple stores must use a third-party app — such as Gift Reggie, Govalo, or a custom integration using the Admin API to issue and validate gift cards programmatically across store instances. Migrating gift card balances from an old store to a new one also requires either manual re-issuance or an API migration script.

Store Migration Warning
If a merchant migrates their store to a new Shopify account (not just a new theme — an actual new account or store), all existing gift card codes become invalid on the new store. Shopify does not provide a built-in migration tool for gift card balances. Merchants must export their gift card list from the old store (Admin → Discounts → Gift cards → Export) and manually re-issue matching balances on the new store before the old store is deactivated.

Failure Mode 5: Gift Card Balance Wrong Due to Multi-Currency Display

On Shopify stores using Shopify Markets or multi-currency, gift cards are issued and stored in the store's primary currency. When a customer browses in a different display currency, the gift card value displayed at checkout is converted to the display currency using Shopify's exchange rate — which may differ from the customer's expectation.

How the discrepancy appears

  • Customer in the UK receives a gift card email showing $100 USD (the store's primary currency)
  • At checkout, they browse in GBP — the gift card field shows "£79.50" (Shopify's exchange rate)
  • The actual bank exchange rate when the customer made the original purchase might have been £81 — the customer sees a lower value than expected
  • The payment is processed in the store's primary currency — GBP display is only cosmetic; the actual charge/deduction is in USD

Why the gift card amount displayed might differ from what was purchased

Cause Example Resolution
Exchange rate changed between gift card issuance and redemption $100 USD gift card issued when 1 USD = 0.85 GBP; redeemed when 1 USD = 0.79 GBP — customer sees £79 instead of £85 Inform customers in gift card emails that values display in their local currency at current exchange rates
Customer browsing in wrong currency for their region Customer in Germany sees checkout in USD because Shopify Markets hasn't auto-detected their country Ensure currency selector and Shopify Markets country detection are working
Gift card issued in secondary currency via Markets Merchant issued an EUR gift card but store primary is USD — card redeems in EUR but checkout shows USD equivalent Avoid issuing gift cards in secondary currencies unless the app or flow explicitly supports it; use primary currency only

Failure Mode 6: Automatic Discount Active — Gift Card Field Appears to Conflict

When a Shopify automatic discount is active sitewide, Shopify hides the discount code input field at checkout. This does not affect gift cards — gift cards have their own separate input field and are not blocked by the one-discount-per-order rule. However, customers frequently confuse the two fields and assume their gift card is being blocked when in fact it is the discount code field that is hidden.

What the customer sees

  • Automatic discount (e.g., "SUMMER20" — 20% off all orders) is active and applying automatically
  • The discount code text input is hidden — Shopify removed it because a discount is already applied
  • Customer looks for a field to enter their gift card code and cannot find the combined "Gift card or discount code" field
  • Customer believes their gift card cannot be applied because the field is missing

What's actually happening

In Shopify's new checkout (post-2023 extensibility), the gift card field and the discount code field are displayed as a single combined input labeled "Gift card or discount code." When automatic discounts hide this field, both gift card and discount code entry disappear. This is a known Shopify checkout UX issue — the combined field hiding is all-or-nothing.

Shopify Plus Workaround
On Shopify Plus, you can use a Checkout UI Extension to add a separate gift card input that is always visible regardless of discount state. This requires developing and deploying a custom extension via the Shopify CLI. Non-Plus merchants have no native workaround — the only option is to ensure automatic discounts are scoped to specific products or collections so the general discount code field remains visible.

Failure Mode 7: Gift Card Balance Shows $0 — Card Expired or Already Fully Redeemed

A gift card with a $0 remaining balance applies $0 at checkout. Shopify still shows "Gift card applied" in the order summary — with a $0 deduction — which confuses customers who expected a discount. Two causes produce $0 balance:

Cause A: Gift card fully redeemed on a prior order

The card was used in full on a previous order. The code is still technically valid but has no remaining balance. The customer may have the code written down or saved and assume it still has value. Admin lookup shows "Balance: $0.00 / Original: $50.00."

Cause B: Merchant set an expiry date on the gift card

Shopify allows merchants to configure gift card expiry dates at Admin → Settings → Gift cards → Expiration. In some regions (EU member states, UK, California), gift card expiry is legally restricted or prohibited — but Shopify allows merchants to set expiry globally regardless of jurisdiction. If a card is past its expiry date, Shopify treats it as having $0 balance. The checkout shows "Gift card applied: $0" with no indication that the card expired.

Legal Note on Gift Card Expiry
In California, gift cards cannot legally expire (with narrow exceptions). In the EU, many member states limit or prohibit gift card expiry. In the UK, expiry must be clearly disclosed at point of sale. If your Shopify store sells to customers in these regions and you have gift card expiry configured, expired cards that show $0 balance at checkout may create legal liability. Review your gift card terms with a legal advisor before setting expiry dates.

Failure Mode 8: Headless Storefront — Gift Card Balance Not Accessible via Storefront API

For merchants running headless Shopify storefronts (Hydrogen, custom React/Next.js, or any build using the Storefront API instead of Shopify's hosted checkout), gift card handling requires explicit API calls — and the Storefront API does not expose gift card balance lookup for privacy reasons. A customer in a headless storefront cannot check their gift card balance via the frontend without a server-side proxy.

What headless developers must know

Operation API to Use Notes
Apply gift card to cart Storefront API: cartDiscountCodesUpdate does NOT work for gift cards Gift cards cannot be applied via Cart API; must use legacy Checkout API checkoutGiftCardsAppend mutation or route through hosted checkout
Check gift card balance Admin API only — not available in Storefront API Requires a server-side proxy that calls GET /admin/api/gift_cards.json?code=[code] and returns balance to the frontend; never expose Admin API keys client-side
Display applied gift card at checkout Storefront API checkout object includes appliedGiftCards field After applying via checkoutGiftCardsAppend, read checkout.appliedGiftCards[].amountUsed and .balance to display balance remaining
Remove an applied gift card Storefront API: checkoutGiftCardsRemove mutation Pass the gift card ID (not the code) to remove — retrieve the ID from checkout.appliedGiftCards[].id
Common Headless Mistake: Using cartDiscountCodesUpdate for Gift Cards
The Shopify Cart API's cartDiscountCodesUpdate mutation accepts discount codes but does not accept gift card codes. Gift card codes entered here are silently ignored — the cart shows no error and no applied gift card. Developers who build a combined "discount or gift card" field and route everything through cartDiscountCodesUpdate will see gift cards appear to apply without reducing the total. The fix is to detect gift card codes (they use a distinct format) and route them to checkoutGiftCardsAppend on the legacy Checkout API or handle them via your checkout completion flow.

Server-side gift card balance proxy (outline)

// Server endpoint: POST /api/gift-card-balance

Frequently Asked Questions

Can a customer apply a gift card and a discount code at the same time?

Yes. Gift cards and discount codes are separate systems in Shopify and do not conflict with each other. A customer can apply one discount code AND one or more gift cards to the same order. The discount code reduces the price, and the gift card covers part or all of the discounted total. The conflict that confuses merchants is with automatic discounts — when an automatic discount is active, the discount code field is hidden, which also hides the gift card field in Shopify's combined input UI.

What happens to the remaining gift card balance after a partial-use purchase?

The remaining balance stays on the gift card — the original code is still valid and can be used on future purchases at the same store. Shopify sends a "Gift card used" email after the order showing the amount used and the remaining balance. The card does not expire after partial use unless the merchant has set an expiry date. The customer can check the remaining balance by visiting the gift card page via the original gift card email link.

Can I issue a refund back to a gift card?

Yes. When processing a refund in Shopify Admin, merchants can choose to refund "To store credit" which issues a new gift card to the customer for the refunded amount. This is different from refunding to the original payment method. If the original order was paid partly by gift card, Shopify will refund the gift card portion back to the original gift card by default (restoring the balance) — though merchant can override this. Gift card refunds create a new gift card code, not a balance addition to the original code.

Why does the gift card show "applied" but the order total didn't change?

The most common cause is an expired gift card ($0 balance that Shopify still allows to be "applied") or a fully-redeemed card. The checkout shows "Gift card applied" regardless of whether the card had any remaining balance. Check the card in Admin → Discounts → Gift cards to confirm the current balance. If the card shows $0 balance and you believe the card should have value, check the transaction log for any unauthorized uses.
