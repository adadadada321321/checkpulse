CAUSE 01 / 08
Most Common

Code Expired or Not Yet Active

Every Shopify discount code has an optional start date and end date. If the current date is past the end date, or before the start date, Shopify rejects the code as invalid — even if the code string is correct. This is the #1 cause of discount codes that "worked last week but don't work today."

Seasonal campaigns (Black Friday, January sale) are especially prone to this: the promotional email goes out with a code, the end date passes, but the email stays in inboxes and recipients try the code months later.

FIX: Admin → Discounts → click the discount code → look at "Active dates." If the end date has passed, either remove it (no expiration) or extend it. If the start date is in the future, the code won't work until that date arrives. Save changes — takes effect immediately.
CAUSE 02 / 08
Configuration

Minimum Purchase Requirement Not Met

Shopify discount codes can require a minimum order amount (e.g., "$50 minimum") or a minimum item quantity. When the customer's cart doesn't meet the threshold, the code is rejected. Shopify usually shows a message stating the minimum requirement, but some storefronts suppress or obscure this message.

This cause is very common when influencer codes or email codes have a minimum that isn't prominently displayed — customers enter the code and get rejected without understanding why.

FIX: Admin → Discounts → click the code → "Minimum purchase requirements" section. Either lower/remove the minimum, or make the threshold visible to customers at cart/checkout so they can add more to qualify. Consider adding a cart notification that shows when a customer is close to the minimum.
CAUSE 03 / 08
Configuration

Usage Limit Reached

Shopify lets you cap how many total times a code can be used, and separately how many times a single customer can use it. When the total limit is reached, the code stops working for everyone. When the per-customer limit is reached, the code stops working for that specific customer but continues for others.

This is a silent failure from the merchant perspective — you may not notice the limit was hit unless you check the discount's usage counter. Meanwhile, customers who received the code (via email, influencer partnership) keep trying it and getting rejected.

FIX: Admin → Discounts → click the code → "Usage limits" section. Check "Times used" against your limit. To allow more usage: increase or remove the total limit. For per-customer limits, you can't reset individual customer usage from the admin — you'd need to create a new code for that customer. If the code was shared broadly and limits are exhausted, create a new code with a higher limit.
CAUSE 04 / 08
Configuration

Product or Collection Restriction

A discount code can be restricted to apply only to specific products or specific collections. If the customer's cart contains only items outside the eligible set, the code is rejected. If the cart is mixed (some eligible, some not), Shopify applies the discount only to eligible items — the customer may see a partial discount and assume the code is broken.

This is especially confusing when the code was shared in a context that implied it applied to all products (e.g., "20% off your order") but was actually created with product restrictions in admin.

FIX: Admin → Discounts → click the code → "Applies to" section. If it shows "Specific products" or "Specific collections," confirm the products in the customer's cart match. Either add the missing products/collections to the discount, change to "All products," or communicate the restriction clearly to customers in the promotional material.
CAUSE 05 / 08
Configuration

Customer Eligibility Restriction

Shopify discount codes can be restricted to specific customers (by customer tag or manually selected), specific customer segments, or first-time customers only. When a customer who doesn't meet the eligibility criteria tries the code, they get "This discount code isn't valid" — the same error as an expired code, with no indication that eligibility is the cause.

"First-time customers only" restrictions are a frequent source of confusion: a returning customer who received a referral code will be blocked, and the rejection message gives no hint that they've already purchased before.

FIX: Admin → Discounts → click the code → "Customer eligibility" section. If it shows "Specific customer segments" or "Specific customers," verify the customer qualifies. For first-time customers, check the customer's order history in Admin → Customers. To broaden eligibility, change to "All customers" — or create a separate code for returning customers without the restriction.
CAUSE 06 / 08
Configuration

Single-Use Code Already Redeemed

Some discount codes are generated as single-use — once one customer redeems them, they're invalid for everyone including the original recipient. This is common with bulk-generated unique codes (e.g., "generate 1,000 unique 20%-off codes for email campaign"), where each code is tied to exactly one use.

If a customer clicks an email link, adds to cart, doesn't complete the purchase, then returns later and the code was somehow used in between (e.g., forwarded to someone else), they'll see the code as invalid.

FIX: Admin → Discounts → click the code → confirm "Limit to one use per customer" is checked and check "Times used" (should be 1 if already redeemed). For bulk-generated unique codes, each customer should have their own code. If this customer's code was redeemed unintentionally, contact the customer directly and issue a new unique code — there's no way to "unredeeem" a single-use code in Shopify.
CAUSE 07 / 08
Configuration

Automatic Discount Conflict

Shopify has two types of discounts: manual codes (customers enter at checkout) and automatic discounts (applied automatically to qualifying carts without any code entry). By default, Shopify does not allow both to apply to the same order. If an automatic discount is active and applies to the customer's cart, they cannot also use a manual discount code — they'll see "Only one discount may be applied."

This is easy to trigger accidentally: you run a sitewide automatic discount for a sale, then a customer tries to use an influencer code on top of it. The manual code is rejected even though it's valid.

FIX: Admin → Discounts → Automatic discounts tab — check for any active automatic discounts. Two options: (1) Allow combinations — edit both the automatic discount and the manual code, enable "Discount combinations" so they can stack; (2) Pause or end the automatic discount if the sale period is over. Shopify's combination settings let you explicitly allow or block stacking between discount types on supported plans.
CAUSE 08 / 08
Silent Failure

Third-Party Discount App Failure

Third-party discount apps (Bold Discounts, Automatic Discounts Pro, Shopify Scripts, custom Shopify Functions) inject into the checkout pipeline to apply their own discount logic. When such an app experiences an API error, credential expiration, a deployment issue, or a conflict with a Shopify platform update, it can fail silently in one of two ways:

Soft failure: the app rejects a valid code as "invalid" — customers can't apply their discount but can still check out at full price. The checkout itself works; only the discount logic is broken.

Hard failure: the app crash corrupts the checkout session — customers can't complete the order at all, whether or not they're entering a discount code. The checkout URL still returns HTTP 200, so standard uptime monitoring shows all-green while orders are failing.

FIX (soft): Admin → Apps → find the discount app → check its dashboard for error logs. Try disabling the app in its own settings (not uninstalling) to confirm whether native Shopify discount codes start working again. Contact the app developer with the specific code and error message. FIX (hard): If checkout itself is broken, disable the discount app entirely from Admin → Apps → uninstall or disable. Then run a test checkout without the app to confirm the session is valid. Re-enable only after the app's support team resolves the issue. Use CheckPulse to get alerted the moment a session-level failure like this occurs — before customers start reporting it.
Troubleshooting Flow

5-Step Diagnosis When a Code Is Reported Broken

When a customer reports that a discount code doesn't work, go through these steps in order before contacting the customer — you'll find the cause in under 5 minutes in most cases.

1

Find the code in Admin

Go to Admin → Discounts → search for the exact code string the customer reported. Confirm it exists. If it doesn't exist or shows 0 results, the code was deleted or they have a typo.

2

Check Active dates and Status

Is the code showing as "Active"? Check the start date (hasn't arrived yet?) and end date (already passed?). This resolves the majority of "suddenly stopped working" reports.

3

Check Usage limits and Customer eligibility

Is "Times used" at or above the limit? Does Customer eligibility restrict to specific customers or segments? Ask the customer for their email to look up whether they meet the eligibility requirements.

4

Reproduce the failure yourself

Add the same products the customer had to a cart and apply the code in a test checkout. If it works for you but not them, the issue is customer-specific (eligibility, per-customer limit, or their browser/session). If it fails for you too, the issue is global.

5

Check for app conflicts

If steps 1–4 show nothing wrong, go to Admin → Apps and look for any installed discount apps. Try temporarily disabling them and reproduce the test. If the code starts working after disabling an app, that app is the conflict.

The Monitoring Gap

Why Most Discount Failures Are Invisible Until Customers Complain

Causes 1–7 are configuration issues: they're visible in Admin, fixable in minutes, and don't require any monitoring. Once you know the cause, you can prevent them with better launch checklists.

Cause 8 — third-party app failure — is different. When a discount app breaks, here is what your monitoring stack sees:

🔴
Ping monitor: your checkout URL returns HTTP 200. The page loads. The app failure is an application-layer event, invisible to TCP/HTTP checks.
🔴
Uptime Robot / Pingdom: same limitation — they verify the HTTP handshake, not whether the checkout session is valid or the discount pipeline is intact.
🟡
Shopify Analytics: conversion rate drops are visible after the fact — you'll see the damage in your dashboard, not before customers experience it.
🟡
Manual testing: only catches the issue if you happen to test during the failure window, with a cart that includes the affected product set.
🟢
End-to-end checkout simulation: verifies the full checkout session is valid — including that app-layer failures haven't corrupted the checkout pipeline — on an automated schedule.

FAQ

Shopify Discount Code Not Working — Frequently Asked Questions

Why is my Shopify discount code not working at checkout?
The most common reasons: the code has expired (check Active dates in Admin → Discounts), the usage limit has been reached (check Times used), the customer hasn't met the minimum purchase amount, or the code is restricted to specific products or customer segments. Go to Admin → Discounts, find the code, and review each setting — the cause will be visible in 90% of cases within 2 minutes.
Why does my Shopify discount code say "invalid" even though I set it up correctly?
If the code looks correct in Admin but shows "invalid" at checkout, the most likely cause is a Customer eligibility restriction — the specific customer entering the code doesn't qualify (wrong segment, not a first-time buyer, etc.). Another cause: a third-party discount app is overriding the native discount logic. Try disabling any installed discount apps and test the code again. If it works without the app, the app is the conflict.
Can a Shopify discount code work for some customers but not others?
Yes. Codes with "Specific customers" or "Specific customer segments" eligibility will only work for qualifying customers. Codes with "First-time customers only" will fail for returning customers. Codes with a per-customer usage limit will fail for customers who've already hit their redemption count. To see why a specific customer was rejected, go to Admin → Discounts → click the code → Customer eligibility section.
My discount code stopped working — why?
If a code was working and then stopped, in order of likelihood: (1) the end date passed — check Active dates; (2) the total usage limit was reached — check Times used vs. your cap; (3) a third-party discount app was updated and introduced a conflict. Go to Admin → Discounts → find the code and check the Active dates section and Usage limits section first. If both look fine, check for recent app updates or changes.
Why can't customers stack discount codes in Shopify?
Shopify does not stack a manual discount code with an active automatic discount by default. If you have an Automatic discount running and it applies to the customer's cart, a manually entered code will be rejected. To allow stacking, go to each discount's edit page and enable "Discount combinations" (available on Shopify plans that support it). Alternatively, pause the automatic discount during periods when you want manual codes to be the primary discount mechanism.
How can a discount code failure break the entire checkout — not just the discount?
Third-party discount apps inject into the checkout pipeline. When such an app crashes, it can corrupt the checkout session — not just reject the discount, but prevent orders from being placed entirely. The checkout URL still returns HTTP 200, so standard uptime monitoring won't catch it. End-to-end checkout simulation — which actually tests whether a checkout session is valid and completeable — is the only automated way to catch this class of failure before customers report order failures.
