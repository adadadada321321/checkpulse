Benchmarks

Shopify Checkout Speed Benchmarks by Step

Shopify's checkout runs in four distinct network steps. Each step adds latency independently — a fast checkout requires all four to be within range. Here's what "fast," "acceptable," and "slow" looks like at each step:

Checkout Step Fast (< ) Acceptable Slow (fix this)
Step 1 — Product & Cart
Add-to-cart + cart creation
 < 400ms 400–900ms > 1s
Step 2 — Checkout Session
Session initialization, URL generation
 < 600ms 600ms–1.5s > 2s
Step 3 — Payment Methods
Gateway API round trip, methods render
 < 1.2s 1.2–2.5s > 3s
Step 4 — Full Page Interactive
All scripts loaded, customer can type
 < 2s (desktop) 2–4s (mobile 3G) > 5s any device

Benchmarks are measured from a cold start (no browser cache) using Shopify's hosted checkout infrastructure without heavy customization. Actual times vary by region, gateway, and installed apps. Step 3 is the most variable: a slow payment gateway can add 1–4 seconds regardless of everything else.

Root Causes

What Makes Shopify Checkout Slow

These are the causes that account for the majority of slow Shopify checkouts, ranked by typical performance impact:

Third-Party App Scripts at Checkout
Checkout extension apps (loyalty widgets, upsell blocks, consent tools, analytics pixels) load JavaScript at runtime during checkout. Each well-optimized app adds 50–150ms; poorly optimized apps add 500–2000ms each. Installing 3–5 apps can push checkout from 2 seconds to 7+ seconds with no other changes. This is the single most common cause of sudden checkout slowdowns. See: App conflict guide →
+200–2000ms per app
Payment Gateway API Latency (Step 3)
The payment methods step requires a live API call to your payment gateway. If the gateway is slow — from high load, a partial incident, or geographic routing issues — this single step can add 1–4 seconds regardless of how optimized everything else is. Gateway slowdowns are often invisible to standard monitoring because the checkout returns 200 with payment methods visible, just slowly. See: Payment error guide →
+1,000–4,000ms
Unoptimized Checkout Header Images
Custom checkout header images added via Admin → Settings → Checkout → Logo should be under 100KB. A source image that's 800KB–2MB uploaded directly adds 500–1500ms of load time on mobile connections. Shopify doesn't automatically optimize checkout images — they serve as uploaded.
+200–1,500ms
Custom checkout.liquid Scripts (Legacy)
Legacy checkout.liquid customizations often include poorly structured JavaScript added over time by multiple developers or apps. Script execution order issues, redundant polyfills, and unminified code can collectively add 300–800ms of blocking time before checkout becomes interactive.
+300–800ms
Shopify Platform Degradation
Shopify's infrastructure can be degraded — slower than normal — without showing a full incident on status.shopify.com. These sub-threshold events add 500–2000ms to checkout steps across all stores on the affected infrastructure. They're impossible to fix on your end; the only mitigation is detecting them quickly with continuous monitoring.
+500–2,000ms
DNS Resolution on Custom Checkout Domain
If you use a custom checkout domain (checkout.yourdomain.com), misconfigured DNS or suboptimal TTL settings can add 100–400ms of DNS lookup time. Test with dig checkout.yourdomain.com or DNS Checker to verify resolution speed from multiple regions.
+100–400ms
Optimization

Shopify Checkout Speed Optimization Checklist

Run these in order — the first items have the highest impact per unit of effort:

  • 01
    Audit and remove non-essential checkout apps
    Go to Admin → Settings → Checkout → Checkout and account pages. List every active extension. For each one, ask: does it directly contribute to conversion, and is there evidence it does? Disable any that don't pass this test. Re-enable only if you observe a measurable negative impact on conversion.
  • 02
    Replace your checkout header image if it's over 100KB
    Download your current checkout logo from Admin → Settings → Checkout. Check the file size. If over 100KB, resize to the displayed resolution (usually 200–400px wide), re-export as WebP, and re-upload. This is often a 5-minute fix that saves 500ms+ on mobile.
  • 03
    Test your primary payment gateway response time
    Go through checkout to the payment methods step and time how long it takes for the payment options to appear. If it's over 2.5 seconds, your gateway may be degraded or misconfigured. Check your gateway's own status page. As a fallback, ensure Shopify Payments is also enabled as an alternative so customers always have an option.
  • 04
    Minimize checkout.liquid JavaScript (if on legacy)
    Open your checkout.liquid template (if you still have access via a Plus plan or legacy setup). Look for render-blocking script tags, external font loads, or analytics pixel calls that run synchronously. Move these to deferred loading or replace with the Checkout Extensibility equivalents, which load asynchronously.
  • 05
    Set up continuous checkout monitoring
    Speed changes happen silently — a new app install, a gateway degradation, a Shopify platform event. Without ongoing measurement, you won't know when your 2-second checkout becomes a 6-second checkout. Automated checkout simulation catches this and alerts you before customers abandon. FAQ

Frequently Asked Questions

How do I test my Shopify checkout speed?

Four methods: (1) CheckPulse free check — tests all 4 checkout steps externally in 10 seconds. (2) Chrome DevTools Network tab — incognito window, F12, complete checkout while watching the timing waterfall. (3) WebPageTest.org — enter your checkout URL, use a mobile device profile. (4) App isolation test — disable checkout extensions one by one to identify the slow one. The external check is the most reliable baseline since it's independent of your browser cache and connection.

What is a good Shopify checkout load time?

Fast Shopify checkout looks like: session initialization under 800ms, payment methods load under 1.5 seconds, full page interactive under 2.5 seconds on a fast connection. Warning zone starts at 3+ seconds per step; critical zone is 5+ seconds to full interactivity. Most Shopify-native checkout pages hit the "fast" range before customization — it's apps and images that push stores into the warning and critical zones.

Why is my Shopify checkout suddenly slow?

The most common trigger for sudden checkout slowdown is installing a new app that injects JavaScript into the checkout. Check what changed in your Admin in the past 2–4 weeks. Also check: did you add a checkout header image recently? Did your payment gateway change status? Run the app isolation test (disable extensions one by one) to identify the cause — this works 80% of the time.

Does slow checkout affect my SEO?

Checkout pages themselves are not indexed by Google — so checkout speed doesn't directly impact search rankings. But slow checkout increases cart abandonment, which reduces return visit rates and signals low engagement in your analytics. More directly: slow checkout erodes the ROI of paid ads, since you're paying the same CPM to drive traffic into a checkout that converts at half the rate it should. The impact is invisible in ad platforms — it looks like a targeting problem, not a technical one.

How often should I test my Shopify checkout speed?

Manually: run a full checkout test whenever you install or update a checkout-related app, change payment gateways, update your checkout branding, or notice a drop in conversion rate. Automatically: set up continuous checkout simulation that runs every 30 minutes. This catches gateway degradations, app conflicts after auto-updates, and Shopify platform slowdowns that you'd never know to manually test for.

Will a standard uptime monitor tell me if my checkout is slow?

No. Standard uptime monitors (Pingdom, UptimeRobot) check HTTP status codes — they verify your checkout URL returns 200, not that it loads quickly or that payment methods appear. A checkout that takes 8 seconds to load payment options passes every uptime check. To measure checkout performance, you need a tool that simulates the full checkout flow and measures timing at each step — not just URL availability.
