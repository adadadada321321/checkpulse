Shopify Checkout Link Not Working — Expired, Broken & Invalid Checkout URLs (2026)

Checkout link expired, draft order link showing "order not found", cart permalink loading empty, custom domain checkout URL broken — here are all 8 failure modes for Shopify checkout links that stop working, with diagnostic steps and exact fixes for each type.

Updated June 2026 8 failure modes Session Expired · Draft Link Expired · Variant Deleted · Custom Domain · URL Mismatch · Purged Cart · Market Mismatch · App-Generated Link

Shopify merchants share checkout links in a variety of ways: pasting a live checkout URL into an email or chat, sending draft order invoice links to wholesale or B2B customers, embedding cart permalink URLs in marketing campaigns, and using apps that generate checkout links automatically. Each of these link types has different expiry behavior, different failure modes, and different fixes when the link stops working.

When a Shopify checkout link doesn't work, buyers typically see one of three things: a blank or empty cart, an error page saying "link expired" or "order not found", or a redirect back to the store homepage with no explanation. None of these outcomes communicate what went wrong — and none of them reach your admin as an error log. The failures are silent from the merchant's perspective.

First: Identify Which Type of Checkout Link You're Using
Shopify has four distinct types of checkout links, each with different behavior: (1) Live session checkout URLs — the checkout.shopify.com/... URL copied from your browser during an active checkout session. These are session-specific and expire. (2) Draft order checkout links — generated from Admin → Orders → Drafts → Send Invoice. These have configurable expiry. (3) Cart permalink URLs — yourstore.com/cart/VARIANT_ID:QTY format that adds items to a fresh cart. These don't expire but depend on variant availability. (4) App-generated checkout links — URLs created by third-party apps using Shopify's Cart or Checkout API. These have their own expiry and validity rules depending on the app. Identifying which type you're using determines which failure mode and fix applies.

Failure Mode 1: Checkout Session Link Expired

The most common reason a Shopify checkout link "doesn't work" is that the URL being shared is a live session checkout URL — the address from the browser's address bar during an active checkout — rather than a shareable cart link. Shopify checkout sessions are tied to a specific browser session and expire after a period of inactivity. When the session expires, the URL becomes invalid.

How checkout session expiry works

  • Checkout sessions are not persistent links: The URL at checkout.shopify.com/SHOP/checkouts/TOKEN is a session token that represents a specific checkout instance. It is not a shareable cart permalink. Once the session expires or the original browser session is closed, the URL no longer points to a valid checkout.
  • Default session window is approximately 1–2 hours: Shopify's default checkout session lifetime is short relative to typical e-commerce session windows. Sessions do not stay active indefinitely in the background — inactivity causes expiry. Sessions may also expire sooner if the cart contents become invalid (e.g., a product goes out of stock).
  • Sharing from the address bar is the root cause: Merchants and sales teams who manually copy a checkout URL from their browser and paste it into an email, WhatsApp, or CRM system are sharing a session-specific URL. The buyer receives a link that may have already expired by the time they open it.
  • Expired session shows empty cart or error: When a buyer opens an expired checkout session URL, Shopify may redirect them to an empty cart page, the store homepage, or a generic error page. The behavior varies by theme and Shopify version.
Never Share Checkout Session URLs Directly — They Expire
The URL in your browser's address bar during checkout (e.g. checkout.shopify.com/...) is a session token, not a shareable link. If you paste this URL into an email or message for a buyer, it will almost certainly be expired by the time they click it. Instead, use cart permalink URLs (yourstore.com/cart/VARIANT_ID:QTY) or draft order invoice links from Admin → Orders → Drafts for any checkout link that needs to be shared.

How to create shareable checkout links correctly

  1. For cart pre-filling: Use a cart permalink URL in the format https://yourstore.myshopify.com/cart/VARIANT_ID:QUANTITY or with your custom domain https://yourstore.com/cart/VARIANT_ID:QUANTITY. These create a fresh cart session every time they are opened and do not expire as long as the product remains available.
  2. For multi-item carts: Use the format /cart/VARIANT_ID1:QTY1,VARIANT_ID2:QTY2 to pre-fill a cart with multiple items. You can also append discount codes with ?discount=CODE.
  3. For individual customer orders: Use Admin → Orders → Drafts → Create Draft → add items → Send Invoice. This generates a draft order checkout link with the buyer's details pre-filled.
  4. For app-generated links: Most cart/checkout link generator apps create persistent cart links using the Storefront API's checkoutCreate mutation, which has its own session window — verify the app's link expiry settings.
Finding Your Variant IDs for Cart Permalinks
To build a cart permalink, you need the variant ID for each product. Find it in Admin → Products → select a product → click a variant → the variant ID appears in the URL bar as the number after /variants/. For stores using the Storefront API, variant IDs in URLs use the base number (e.g., 12345678901234), not the gid format.

Failure Mode 2: Draft Order Checkout Link Expired

Draft order checkout links are the correct way to send a specific buyer a pre-filled, personalized checkout. But draft order links have their own expiry mechanism — once the link expires, buyers see an "order not found" or "this link is no longer valid" error, and there is no automatic re-send from Shopify.

How draft order link expiry works

  • Expiry is set when the invoice is sent: When you send a draft order invoice from Admin → Orders → Drafts → Send Invoice, Shopify generates a checkout URL that includes the draft order token. This link remains valid until the draft order is completed, cancelled, or the configured expiry date passes.
  • Default expiry behavior depends on Shopify plan and configuration: By default, draft order links do not have a hard expiry date set by Shopify — they remain valid as long as the draft order is open. However, some third-party apps or custom integrations set an expiry when creating draft orders programmatically via the Admin API. If the link was created with an invoice_url that includes a time-limited token, it will expire.
  • Link becomes invalid when the draft is completed or cancelled: Once a buyer completes payment via the draft order link (converting it to an order) or a merchant manually marks the draft as complete or cancels it, the checkout URL is invalidated. Buyers who click an old link to a completed draft see an error.
  • Re-sending the invoice generates a new link: If a buyer reports that their draft order link is not working, go to Admin → Orders → Drafts → find the draft order → click "Send Invoice" again. This generates a fresh checkout URL and sends a new email to the buyer.
Draft Order State Checkout Link Valid? Action
Draft — Open Yes (link works) Send invoice link to buyer; link is valid
Draft — Completed (payment received) No — link invalid Order is paid; link no longer needed
Draft — Cancelled No — link invalid Reopen draft or create a new one
Draft — Open but app-set expiry passed No — token expired Send invoice again from Admin to generate fresh link
Buyers Often Save Draft Order Emails and Click Them Later
B2B and wholesale buyers frequently save draft order invoice emails to reference later. If a buyer marks a draft order email as unread and returns to it weeks later — after the draft was completed or cancelled — they will click a broken link. Communicate clearly in draft order emails that the link is time-sensitive, or set up a process to notify buyers when a draft is about to be cancelled so they can use the link first.

Failure Mode 3: Product or Variant Deleted After Link Was Created

Cart permalink URLs reference specific Shopify variant IDs. If the product or variant referenced in the URL is deleted, archived, unpublished, or sold out after the link was created, the permalink will load an empty cart rather than the expected pre-filled cart. This failure is particularly common in marketing campaigns where links are embedded in emails or ads that run over a period of time.

How variant deletion breaks cart links

  • Shopify silently ignores invalid variant IDs in cart URLs: When a buyer opens a cart permalink that references a deleted or unavailable variant, Shopify does not show an error. It simply ignores the add-to-cart instruction and loads an empty cart. Buyers see a cart with no items and have no idea what went wrong.
  • Product archived or unpublished = same effect as deleted for cart links: Archiving a product or unpublishing it from the Online Store removes it from cart permalink availability. A link to an archived product's variant ID loads an empty cart, even though the product still exists in Admin.
  • Variant sold out with inventory tracking enabled: If inventory tracking is enabled and the variant reaches zero stock, Shopify may prevent it from being added to a cart via permalink depending on your "Continue selling when out of stock" setting. With this setting off, a zero-stock variant cannot be added via cart permalink.
  • Product replaced or remade with new variant IDs: When merchants delete and recreate a product (rather than editing it in place), the new variants get new IDs. Old cart permalink links referencing the original variant IDs break immediately. This is common after a product catalog migration or a theme rebuild that also recreated products.

How to diagnose and fix broken cart permalink links

  1. Take the variant ID from your cart permalink URL and search for it in Admin → Products → use the URL format admin.shopify.com/store/YOURSTORE/products?query=VARIANT_ID to verify the variant still exists and is active
  2. Confirm the product is published to the Online Store channel (Admin → Products → select product → check Sales channels)
  3. Check that "Track quantity" is enabled and stock is above zero, or that "Continue selling when out of stock" is enabled if you sell pre-orders
  4. If the variant was replaced, update all campaign URLs, email templates, and landing pages that contain the old variant ID with the new one
  5. For marketing campaigns running over time (email sequences, retargeting ads), use product page URLs rather than cart permalinks where possible — product pages dynamically reflect current variant availability rather than linking to a specific variant state

Failure Mode 4: Custom Domain Checkout Misconfiguration

Shopify allows merchants to configure a custom checkout domain so that the checkout experience uses their own domain (e.g., checkout.mybrand.com) instead of the default checkout.shopify.com. When this custom domain is misconfigured — incorrect DNS records, expired SSL certificate, wrong CNAME target — checkout links built using that domain fail while the underlying checkout itself may still work through the default Shopify domain.

Custom checkout domain requirements

  • CNAME record must point to the correct Shopify target: The CNAME for your custom checkout subdomain (e.g., checkout.yourdomain.com) must point to shops.myshopify.com. If the CNAME was set up correctly at first but the DNS record was accidentally modified or deleted later, checkout links using the custom domain will fail to resolve.
  • SSL certificate must be valid and current: Shopify provisions an SSL certificate for your custom checkout domain. If this certificate expires or fails to renew (which can happen if the CNAME record is temporarily removed or if Shopify's certificate renewal process hits a DNS propagation issue), buyers will see an SSL error when opening checkout links that use the custom domain.
  • Custom domain must match exactly what Shopify has on file: If you change your primary store domain or checkout subdomain in Shopify settings, existing checkout links that reference the old domain will break. Always update checkout link sources (email templates, marketing links, app configurations) when changing your checkout domain.
  • Only Shopify Plus stores support custom checkout domains: Standard Shopify plans do not support custom checkout domains. If you are on a standard plan, all checkout links use checkout.shopify.com and custom domain checkout configuration is not available — any attempt to configure it will not take effect.
Checkout Domain Issue Symptom Fix
CNAME record missing or incorrect Checkout link fails to load (DNS error) Re-add CNAME: checkout.yourdomain.com → shops.myshopify.com
SSL certificate expired Browser shows SSL/certificate error Remove and re-add custom checkout domain in Shopify Admin to trigger re-provisioning
Domain changed but links not updated Links show 404 or redirect to wrong page Update all checkout link sources with the new domain
Custom domain attempted on non-Plus plan Feature not available; links use default domain Use checkout.shopify.com links on standard plans

How to verify custom checkout domain health

  1. Go to Admin → Settings → Domains → check your checkout domain is listed and shows "Connected" with a green indicator
  2. Open a test checkout link using the custom domain URL in an incognito browser window and verify it loads without SSL errors
  3. Run a DNS lookup for your checkout subdomain (e.g., dig checkout.yourdomain.com CNAME) to confirm the CNAME resolves to shops.myshopify.com
  4. Check SSL certificate validity at https://checkout.yourdomain.com — the padlock icon in the browser should show a valid, Shopify-provisioned certificate
  5. If the SSL shows as expired, remove the custom checkout domain from Shopify Admin → Domains, wait 5 minutes, re-add it, and allow up to 24 hours for Shopify to re-provision the certificate

Failure Mode 5: Mismatched Store URL in the Checkout Link

Shopify stores have multiple URL formats for the same checkout: the myshopify.com URL, the custom domain URL, and the checkout.shopify.com URL. Checkout links built using one URL format may not work correctly when the store's primary URL has changed, or when links are shared that mix these formats inconsistently.

How URL mismatches break checkout links

  • Cart permalink URLs are domain-specific: A cart permalink built with yourstore.myshopify.com/cart/... and a cart permalink built with yourdomain.com/cart/... both work if both domains are configured — but links shared using the myshopify.com format may behave differently from links using the custom domain, particularly for stores with market-specific redirects or localization rules.
  • Checkout URLs are checkout-subdomain-specific: A checkout URL of checkout.shopify.com/SHOP/checkouts/TOKEN (the default) is different from checkout.yourdomain.com/SHOP/checkouts/TOKEN (custom domain). These cannot be swapped — a checkout session started on the default domain's checkout cannot be resumed via the custom checkout domain URL.
  • Storefront URL changes break embedded links: If a merchant changes their primary custom domain (e.g., from shop.mystore.com to mystore.com), old cart permalink links using the previous domain will fail. Shopify does not automatically redirect /cart/... paths from old domains to new ones — only the root domain redirect may be configured.
  • International market URL structures add complexity: For stores using Shopify Markets with market-specific subfolders (e.g., yourdomain.com/en-us/cart/... vs. yourdomain.com/cart/...), a cart permalink without the market subfolder may add items to the wrong market's cart or fail to add them at all if the market-specific URL structure is enforced.
Use Relative URLs in Templates Instead of Hardcoded Domains
Email templates, app configurations, and marketing platforms that build cart links often hardcode a specific domain. When your store domain changes, every hardcoded link breaks. Where possible, use relative cart URLs (/cart/VARIANT_ID:QTY) in Shopify email notification templates — Shopify automatically resolves these to the correct absolute URL for the buyer's session and store domain.

Failure Mode 6: Abandoned Cart Session Purged

Some merchants and third-party tools share the checkout URL from an actively abandoned cart rather than generating a fresh cart permalink. Shopify does retain abandoned checkout sessions for a period, and the Abandoned Checkouts feature in Admin can show these sessions — but the checkout URLs associated with abandoned carts have their own validity window and will eventually be purged.

How purged checkout sessions manifest

  • Abandoned checkout URLs from Admin → Orders → Abandoned checkouts: The checkout URLs listed in your abandoned checkouts report are live session URLs that were created when a buyer began checkout. These URLs are valid for recovering the abandoned checkout if the buyer returns within the recovery window, but they are not permanent links — Shopify purges the underlying session data after it ages out.
  • Recovery email checkout links stop working after the session ages out: Abandoned cart recovery emails sent by Shopify or by third-party apps (Klaviyo, Omnisend, Postscript) include checkout URLs that point to the buyer's abandoned session. If the buyer opens this email days or weeks after the session was purged, the link will no longer work. The recovery email should ideally be sent within the first 24 hours of abandonment to maximize the chance the session is still valid.
  • Product changes during the abandoned period can also break recovery links: Even within the recovery window, if an item in the abandoned cart is deleted or goes out of stock, the checkout session may become invalid — the buyer opens the link and finds either an error or a checkout with some items removed.
  1. For abandoned cart recovery, send the first recovery email within 1–4 hours of abandonment to maximize session validity
  2. Do not rely on checkout session URLs from the Admin's abandoned checkouts list as long-term links — they age out
  3. For customers who report broken recovery links, manually create a new draft order (Admin → Orders → Create Order → add items → Send Invoice) to create a fresh, valid checkout link for the specific buyer
  4. Configure your email marketing platform to include a "start a new cart" link alongside the recovery link so buyers have a fallback if the session URL is expired

Failure Mode 7: Multi-Currency or Market Mismatch

Shopify Markets allows stores to sell to international customers with localized currencies, languages, and pricing. Cart permalink links and checkout URLs are tied to specific market contexts. A checkout link generated in one market context (e.g., a USD checkout) may not function correctly when opened by a buyer in a different market (e.g., the EU market with EUR pricing), especially on stores with market-specific URL structures or pricing rules enforced at checkout.

How market context affects checkout links

  • Cart permalinks and market-specific currency: When a buyer opens a cart permalink URL, Shopify assigns their cart to a market based on their geolocation or the URL's market subfolder. If a cart link was built for a USD market and a buyer in the EU opens it, Shopify may convert the cart to the EU market context — which can change pricing, shipping availability, and payment method options. In some cases, market-specific required fields or restrictions cause the checkout to stall.
  • Market-specific URL structures enforce market context: Stores using Shopify Markets with subfolder routing (e.g., /en-us/, /en-gb/) need cart links that include the correct subfolder. A cart link without a market subfolder may route buyers to the store's default market, potentially with wrong pricing or availability for their location.
  • Currency mismatch at checkout: If a merchant shares a checkout URL that was initiated in one currency (e.g., GBP) and the buyer's session resolves to a different currency (e.g., USD), Shopify may convert the order currency — which can cause discrepancies between the price the buyer expected (from a quote or email) and the price at checkout.
  • Duties and taxes applied differently by market: A checkout link created without a specific market context may have duties or taxes calculated differently depending on where the buyer is located when they open it, causing the checkout total to differ from what was quoted.
Use Market-Specific Cart Links for International Campaigns
For marketing campaigns targeting specific international markets, build cart permalinks that include the market subfolder if your store uses subfolder-based market routing: yourdomain.com/en-gb/cart/VARIANT_ID:QTY for the UK market. This ensures the buyer opens the cart in the correct market context with the correct currency and pricing from the first click.

Failure Mode 8: App-Generated Checkout Link Malformed or Token Expired

A growing number of Shopify merchants use third-party apps to generate checkout links automatically — cart builder apps, headless checkout solutions, quote-to-order apps, and custom flow builders. These apps generate checkout links via Shopify's Storefront API or Admin API. When the app has a bug, misconfiguration, or integration failure, the generated links can be malformed, contain expired tokens, or reference checkout sessions that were never properly created.

How app-generated checkout links fail

  • Storefront API checkout token expired: Apps that use Shopify's Storefront API checkoutCreate mutation generate checkout tokens that have a validity window (typically 24 hours after creation if untouched). If the app generates a checkout link in advance (e.g., pre-generates links during a daily job) and the buyer receives the link hours or days later, the checkout token may be expired by the time they use it.
  • App using deprecated Checkout API endpoints: Shopify deprecated its Checkout API in favor of the Cart API and Storefront API for generating checkout links. Apps that still use the deprecated Checkout API endpoints may generate URLs that redirect to error pages or no longer function as expected.
  • Malformed URL generated by the app: Bugs in the app's link generation logic can produce checkout URLs with incorrect encoding, missing parameters, or wrong store identifiers. These links fail immediately when opened — they don't expire, they were never valid to begin with.
  • App authentication token expired — app loses API access: Third-party apps access Shopify via OAuth tokens. If an app's OAuth token expires (due to re-installation requirements, API version migrations, or the merchant's Shopify plan change), the app loses the ability to create valid checkout sessions via the API. It may continue generating checkout links but they will be invalid because the underlying API calls are failing.
  • Cart API vs. Checkout URL mismatch: Apps using the Cart API (for storefront cart manipulation) and apps using the Checkout API (for creating checkout sessions) produce different URL formats. Mixing these formats — or generating a cart permalink from Cart API data when a Checkout URL was expected — produces a link that doesn't route to the intended checkout state.

How to diagnose app-generated link failures

  1. Test a manually-created cart permalink (yourstore.com/cart/VARIANT_ID:QTY) for the same items to confirm the underlying checkout works — if the manual link works but the app link doesn't, the issue is the app's link generation, not the checkout itself
  2. Check the app's admin dashboard or logs for API errors — look for OAuth token errors, rate limit errors, or API version deprecation warnings
  3. Reinstall the app if the app's OAuth token may be expired — the reinstallation flow re-authorizes the app and issues a fresh OAuth token
  4. Contact the app's support team with the specific malformed URL pattern — most link generation bugs have been seen before by the app developer and have a known fix
  5. Check the Shopify App Store listing's recent reviews for reports of similar broken link issues — other merchants may have reported the same problem
  6. Verify that the app is not using a deprecated API version (Admin → Apps → find the app → check the API version in its settings or documentation) — apps using API versions that Shopify has sunset may experience unpredictable failures
App-Generated Link Failures Can Affect Large Volumes of Links Simultaneously
Unlike a single expired session link, an app-generated link failure can break every checkout link the app produces at once — if the app's OAuth token expires or its API integration breaks, all new links it generates become invalid simultaneously. This is particularly high-risk for apps that send automated checkout links via email sequences or SMS campaigns. Monitoring your checkout link success rate (separate from your order conversion rate) is the only way to detect this class of failure before it affects large numbers of buyers.
