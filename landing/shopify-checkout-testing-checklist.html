How to Test Your Shopify Checkout: Complete 2026 Checklist

Updated June 2026 · 8 min read · Covers Shopify, Shopify Plus

TL;DR

  • Before launch: enable Bogus Gateway or Shopify Payments test mode, place test orders through each payment method and shipping option, then disable test mode.
  • Test cards: Bogus Gateway uses card numbers 1/2/3. Shopify Payments test mode uses Stripe cards (4242… for success).
  • After launch: manual tests can't catch runtime failures — Shopify Payments suspensions, gateway expiry, 3DS breaks. Use automated monitoring for that.

Every Shopify store owner knows to test checkout before going live. Fewer know the right way to do it — and almost none have a plan for the failures that happen after they go live.

This guide covers both: the complete pre-launch testing checklist, the exact test cards to use, and the gap that manual testing cannot fill.

Step 1: Enable Test Mode

Shopify gives you two ways to test checkout without charging real cards. Use one depending on your payment setup.

Option A: Bogus Gateway (any Shopify plan)

The Bogus Gateway is Shopify's built-in fake payment provider. It's the fastest way to test without configuring a real gateway.

1

Open Payment Settings

In your Shopify admin go to Settings → Payments. Under "Payment providers," click Choose third-party provider.

2

Select Bogus Gateway

Search for "Bogus Gateway" in the provider list and activate it. This replaces your live gateway — only do this in development or before launch.

3

Use the test card numbers

In the card number field, enter 1 (success), 2 (failure), or 3 (gateway error). Use any future expiry and any 3-digit CVV.

Important

The Bogus Gateway does not simulate real payment network behavior — it won't catch 3DS authentication issues, gateway-specific error codes, or Shopify Payments-specific failures. Use it for flow testing only.

Option B: Shopify Payments Test Mode (more realistic)

If you have Shopify Payments, you can enable test mode directly — no gateway swap needed.

1

Open Shopify Payments settings

Go to Settings → Payments → Manage (under Shopify Payments). Check "Use test mode" and save.

2

Use Stripe test cards

Shopify Payments uses Stripe under the hood. Use the test card numbers below. Any future expiry date and any CVV work.

Shopify Test Card Numbers

Use these during checkout while test mode is enabled. These are Stripe test cards — valid for Shopify Payments test mode and most Stripe-based gateways.

Card Number Result Use For
4242 4242 4242 4242 SUCCESS Standard successful payment flow
4000 0000 0000 0002 DECLINED Test how your store handles card declines
4000 0025 0000 3155 3DS AUTH Test 3DS authentication pop-up (SCA)

4000 0000 0000 0069 EXPIRED Expired card decline path

Step 2: Run the Pre-Launch Checkout Checklist

With test mode on, work through each item. Check them off in order — some failures cascade (a broken discount code can break order totals, not just the code itself).

Core Purchase Flow

  • Single product purchase: add one product, go through checkout, complete with success card, confirm order appears in admin.
  • Multiple items: add 2-3 different products, verify cart totals, complete purchase.
  • Out-of-stock handling: if you have inventory limits, verify the "sold out" state and that out-of-stock items can't be added to cart.
  • Failed payment path: use the decline card (4000 0000 0000 0002) and confirm the error message is clear and the customer can retry.
  • Order confirmation email: after a successful test order, verify the confirmation email arrives and contains correct order details.

Shipping & Fulfillment

  • Each shipping zone: test checkout with a destination address in each shipping zone you've configured.
  • Free shipping threshold: if you have a free shipping rule, test with cart values just below and just above the threshold.
  • Local pickup (if enabled): confirm the pickup option appears and can be selected.
  • International shipping: if you ship internationally, test with a foreign address and verify correct rates appear.

Discounts & Promotions

  • Active discount codes: test every discount code currently active. Apply at checkout, verify the discount amount is correct.
  • Expired/invalid codes: try an invalid code and confirm the error message is appropriate.
  • Automatic discounts: if you have automatic discounts (cart-level or product-level), add qualifying items and verify discount applies without a code.
  • Buy X get Y promotions: add qualifying quantities and confirm the discount triggers correctly.

Payment Methods

  • Each payment provider: test checkout through every payment method you've enabled (credit card, PayPal, Shop Pay, etc.).
  • 3DS authentication: use the 3DS test card (4000 0025 0000 3155) and confirm the authentication popup appears and completes correctly.
  • Buy Now Pay Later: if you have Klarna, Afterpay, or similar, complete a test purchase to verify these work end-to-end.

Customer Experience

  • Guest checkout: test as a guest (incognito/private browsing) with no account. This is how most customers shop.
  • Account checkout: test logged in as a customer account to verify saved addresses and payment methods work.
  • Mobile checkout: complete the full flow on a real mobile device (iOS and Android if possible). Mobile often breaks where desktop doesn't.
  • Checkout customizations: if you've added checkout extensibility apps or custom fields, verify they appear and don't break the flow.

Testing tip

Test in a private/incognito window to avoid your admin session interfering with checkout. Admin-logged-in sessions can bypass certain checkout rules — including payment methods and discount validations — that real customers hit.

Step 3: Disable Test Mode Before Launch

This sounds obvious but it's one of the most common go-live mistakes. Stores launch with test mode still active — customers attempt to pay, enter their real cards, and get errors or fake "success" confirmations with no actual charge processed.

1

Remove Bogus Gateway (if used)

Go to Settings → Payments and remove the Bogus Gateway. Re-activate your live payment provider (Shopify Payments or your third-party gateway).

2

Disable test mode on Shopify Payments

Go to Settings → Payments → Manage and uncheck "Use test mode." Confirm the green "Active" status shows for your payment provider.

3

Place one final live test (optional)

Place a real $1 test order with a real card, then immediately refund it from your admin. Confirms everything works end-to-end with a live payment processor.

What Manual Testing Can't Catch

Pre-launch testing validates your configuration. It doesn't protect you from runtime failures — things that break after you launch, often without any visible warning.

These are the failure modes that manual testing will never catch, no matter how thorough you are:

Shopify Payments account suspension

Shopify can suspend your Payments account for compliance reviews — no banner, no visible error on your store. Your homepage loads fine. Customers reach the payment step and see an empty payment options area or generic error. No uptime monitor catches this.

Payment gateway credential expiry

API keys, OAuth tokens, and webhook secrets expire. After a password rotation or security update, the credential your gateway uses to communicate with Shopify becomes invalid. Checkouts fail; your store still looks healthy.

3DS redirect failure from bank policy changes

Banks update their 3DS redirect URLs and authentication endpoints. A 3DS flow that worked during testing can break weeks later when a bank rolls out a new authentication policy. The failure is invisible until a customer reports it.

Shopify API degradation

Shopify's Storefront API and checkout API occasionally degrade or return errors for specific regions or store plans. Your checkout can silently fail for a subset of customers while appearing normal to you and to every standard uptime monitor.

The silent failure problem

Standard uptime monitors check whether your homepage URL returns 200. They will show green while every one of these failures is actively happening. The homepage loads. The failure lives in the payment layer.

Most store owners find out about these failures from customers — sometimes hours later.

How Often Should You Re-Test?

Run a manual test after any of these events:

  • New app installed: especially checkout extension, cart modification, or upsell apps.
  • Theme update or redesign: theme changes can break custom checkout fields and button behavior.
  • Payment gateway change: after switching providers or updating gateway credentials.
  • Shopify plan upgrade: moving between plans can change checkout behavior, especially around Shopify Payments features.
  • After a Shopify maintenance window: Shopify's scheduled maintenance can occasionally leave checkout endpoints in a degraded state.
  • After any JavaScript customization: custom scripts in theme.liquid that run on checkout pages can break order buttons.

For everything else — the silent failures that happen on Tuesday at 2am for no apparent reason — manual re-testing isn't feasible. That's where automated monitoring runs in the background continuously.

Frequently Asked Questions

How do I test my Shopify checkout without charging a real card?

Enable Shopify's Bogus Gateway (Settings → Payments → Manage → Use Bogus Gateway for testing) or use Shopify Payments test mode. With Bogus Gateway, enter card number 1 (success), 2 (failure), or 3 (exception). With Shopify Payments test mode, use the Stripe test card 4242 4242 4242 4242 for a successful payment. No real charges are made.

What is the Shopify test card number?

With Bogus Gateway: card number 1 = success, 2 = decline, 3 = error. With Shopify Payments test mode (Stripe): 4242 4242 4242 4242 (success), 4000 0000 0000 0002 (decline), 4000 0025 0000 3155 (3DS authentication). Use any future expiry date and any 3-digit CVV.

Should I test Shopify checkout in incognito mode?

Yes. Incognito mode simulates guest checkout without your admin session interfering. Test both incognito (guest) and logged-in (customer account) for full coverage. Your admin cookies can bypass payment validation rules that real customers hit.

How often should I test my Shopify checkout?

After any app install, theme change, gateway update, or Shopify plan change. For continuous protection, use automated monitoring — manual testing can't catch runtime failures like Shopify Payments suspensions or gateway token expiry.

Can manual testing catch all Shopify checkout failures?

No. Manual testing validates your configuration at a point in time. It can't catch runtime failures that happen after launch: Shopify Payments account suspensions, gateway credential expiry, 3DS redirect failures from bank policy changes, or Shopify API degradation. These appear silently — your store looks normal while customers can't complete purchases.

What does Shopify test mode affect?

Test mode lets you place orders without real charges. Test orders appear in your admin with a "Test" badge. Emails are sent normally (useful to verify confirmation emails). Shopify does not send actual fulfillment notifications for test orders. Always disable test mode before going live — a common mistake is launching with test mode still active.
