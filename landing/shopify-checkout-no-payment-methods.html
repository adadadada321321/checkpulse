What Customers Actually See

The "no payment methods" failure does not look like a crash. It looks like an incomplete checkout. Depending on your Shopify theme, customers see one of these:

  • A blank payment section — the step loads but shows nothing where credit card / PayPal / Shop Pay would normally appear
  • The explicit message "No payment methods available" — some themes surface this directly
  • A greyed-out or unclickable "Complete order" button — the page renders but the button is disabled
  • A spinner that never resolves — the payment methods request is hanging

In every variant: no customer can place an order. Most close the tab and buy from a competitor. Very few email you to report it.

Why This Is a Silent Failure (and Why Most Monitoring Misses It)

The checkout payment methods call is a sub-request inside a normal-looking checkout page load. When it fails or returns empty, the overall HTTP response for the checkout URL is still 200 OK. Your store is not "down" by any standard definition.

# What the API returns when payment methods are empty
GET /checkouts/{token} → 200 OK ← uptime monitors see this
GET /cart/{token} → 200 OK
GET /checkouts/{token}/payment_url → 200 OK
{ "payment_gateways": [] } ← empty — NO PAYMENT OPTIONS
# Uptime monitor verdict: ALL GREEN ✓
# Checkout reality: BROKEN ✗

Standard uptime monitors — Pingdom, UptimeRobot, StatusCake, Better Uptime — check whether a URL returns a non-error HTTP status. They do not simulate a checkout flow. They will never catch this failure.

This is why stores often run with a broken payment layer for hours. Revenue drops. Conversion rate collapses. The first human who notices is usually the merchant checking their Shopify analytics at the end of the day.

All 9 Causes of "No Payment Methods Available"

#1

Shopify Payments Account Suspended

The most common cause. When Shopify suspends your Payments account — due to chargebacks, identity verification, or policy review — the payment gateway is removed from checkout immediately. The API returns an empty payment_gateways array.

Check: Shopify Admin → Settings → Payments. A suspension banner appears here and Shopify sends an email to your account address.

FIX → Activate a backup gateway (PayPal / Stripe) immediately + respond to Shopify's suspension email. See Shopify Payments Suspended guide.
#2

No Active Payment Gateway Configured

If all payment gateways were deactivated or deleted — sometimes accidentally during a settings cleanup or app uninstall — the checkout has no gateways to show. This is uncommon but happens after bulk settings changes or store migrations.

Check: Settings → Payments. Under "Payment providers," verify at least one gateway shows as active.

FIX → Reactivate Shopify Payments or add a third-party gateway in Settings → Payments.
#3

Currency or Country Mismatch

Shopify Payments is only available in specific countries. If your store currency is set to a currency that your active gateway does not support — or if a customer's billing country is unsupported — checkout returns no payment options for that session.

This is especially common after enabling multi-currency or changing the store's primary currency without checking gateway compatibility.

FIX → Settings → General → check Store currency matches your Shopify Payments country. Or: add a third-party gateway with broader currency support (PayPal, Stripe).
#4

Store in Test Mode with No Test Gateway

If your Shopify store is in development, the theme or checkout may be configured for test mode. In test mode, Shopify Payments requires the Bogus Gateway (test gateway) to be enabled, or a Shopify Payments test configuration. If test mode is active but no test gateway is set, checkout shows no methods.

FIX → Settings → Payments → Enable "Bogus Gateway" under test payment providers. Or: disable test mode if you're ready for live payments.
#5

Third-Party Gateway Credentials Expired or Invalid

Third-party gateways (PayPal, Stripe, Authorize.net, etc.) use API keys or OAuth tokens that can expire. When the token expires or the gateway account is suspended on the provider side, Shopify can no longer authenticate with it, and it is removed from the payment options returned to checkout.

FIX → Settings → Payments → click into your third-party gateway → re-authenticate or update API credentials. Check your provider account for any suspension notices.
#6

Checkout App or Extension Hiding Payment Methods

Shopify's checkout extensibility lets apps inject UI and logic into the payment step. A bug or misconfiguration in a checkout app — fraud prevention, subscription billing, upsell apps — can cause it to hide or remove payment method components from the rendered checkout.

This failure often appears after an app update or when a new app is installed that conflicts with an existing one.

FIX → Apps → disable the most recently installed/updated checkout app → test checkout. Identify the conflicting app via Settings → Checkout → Checkout extensions.
#7

Checkout Not Routed Through Shopify's Secure Domain

Shopify checkout must go through checkout.shopify.com (or your store's .myshopify.com checkout URL). If a custom domain redirect or proxy misconfiguration routes customers to the wrong checkout domain, payment method loading can fail due to cross-origin restrictions.

FIX → Verify your checkout link in the theme follows Shopify's standard checkout URL structure. Check any redirect rules in Online Store → Navigation and your DNS/proxy configuration.
#8

Payment Methods Restricted by Checkout Rules

Shopify's Checkout Rules (available on Shopify Plus) allow merchants to conditionally show or hide payment methods based on order value, customer tags, or product types. A misconfigured rule can inadvertently hide all payment methods for a broad set of customers.

FIX → Shopify Admin → Settings → Checkout → Checkout rules. Review any active rules and check whether they might match a wider audience than intended.
#9

Temporary Shopify Platform Issue

In rare cases, Shopify's own payment gateway API experiences degradation that affects all or some merchants. This is the least common cause and usually resolves within minutes, but it does happen — typically visible on Shopify's status page.

FIX → Check shopifystatus.com. If Shopify reports a payment incident, wait for resolution. Monitor your checkout to confirm when it recovers.

Quick Diagnosis: Identify Your Cause in 5 Minutes

Run through this checklist in order. Most causes become obvious at step 1 or 2.

01

Check Settings → Payments for any banners

A suspension notice, verification request, or "action required" banner here immediately tells you the cause. If you see a banner: that is your issue. Follow the instructions on screen.

02

Check your Shopify account email

Search for recent emails from Shopify regarding payments, verification, or account issues. A suspension or credential expiry almost always comes with an email notification — which many merchants miss if it went to spam.

03

Place a test order in an incognito window

Use a fresh incognito browser to test your checkout from the customer's perspective. This eliminates cached sessions or browser-specific issues and gives you the exact experience a customer sees. Note exactly which step fails.

04

Check when the issue started and what changed

Review recent app installs, updates, theme changes, or settings changes. If you can pinpoint a change that preceded the failure, you have your likely cause. Shopify Admin → Apps → sort by "Recently updated."

05

Check shopifystatus.com

If the above checks turn up nothing, a platform-level incident may be the cause. Shopify's status page lists active incidents and affected components. Payments and Checkout have their own status lines.

Frequently Asked Questions

Why does my Shopify checkout say "no payment methods available"?

Shopify shows this when the checkout payment methods API returns an empty gateways array. The most common causes are a Shopify Payments suspension, a currency/country mismatch with your active gateway, or a checkout app that is hiding payment method components. Start with Settings → Payments and check for any banners or suspension notices.

Does "no payment methods" mean my entire store is down?

No — and that is the danger. Your homepage, product pages, and cart all function normally. The failure is isolated to Step 3 or 4 of the checkout flow. Standard uptime monitors show green. Customers reach checkout, see no options, and leave. You have no outage alert and no indication anything is wrong until you notice a drop in orders.

How do I quickly check why Shopify checkout has no payment methods?

Go to Settings → Payments and look for any banners. Then check your account email for Shopify notifications. Place a test order in an incognito window to reproduce exactly what customers see. If everything looks normal in settings, check whether any checkout apps were recently installed or updated.

Can a Shopify app cause checkout to show no payment methods?

Yes. Checkout extensibility apps, fraud prevention apps, and subscription billing apps can inject logic that conditionally hides payment methods. If the error appeared after installing or updating an app, disable it via Apps and re-test checkout immediately.

How do I prevent this from happening again?

The only reliable method is automated checkout simulation — a tool that runs the full checkout flow and alerts you the moment payment methods return empty. Standard uptime monitors cannot detect this class of failure. What do customers see when there are no payment methods?

Most Shopify themes show either a blank payment section, a greyed-out checkout button, or the explicit message "No payment methods available." In all cases, customers cannot place an order and most leave without contacting you — making the failure accumulate silently until you notice a revenue drop.
