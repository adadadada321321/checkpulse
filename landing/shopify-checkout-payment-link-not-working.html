Shopify Checkout — Payment Links

Shopify Payment Link Not Working — Buy Button, Direct Checkout URL, Draft Order Link, Cart Permalink

Shopify offers four distinct ways to send customers directly to a purchase: Payment Links, draft order links, cart permalinks, and the Buy Button. Each breaks in different ways — draft order links expire silently, cart URLs break when you edit a product, Buy Button iframes get blocked by security headers, and embedded discount codes stop applying without any error message.

Root Causes

Five Root Causes Behind Broken Shopify Payment Links

These five causes account for the vast majority of payment link failures. They share a common trait: the link appears valid from the outside, but something has changed in the store or the link itself since it was created and shared.

Root Cause 1

Draft Order Payment Link Expired After 30 Days

When you send an invoice from a draft order (Admin → Drafts → Send invoice), Shopify generates a unique checkout URL with a time-limited authentication token: /a/orders/[token]/authenticate. This token expires after 30 days. After expiry, the URL returns a 404 or silently redirects to the storefront homepage — the customer sees nothing useful, with no indication that the link has expired or what to do next.

This is particularly painful for B2B sales cycles where invoices are sent weeks in advance, wholesale orders with long payment terms, or any sales process where the customer takes time to review before paying.

Fix Path

Open Admin → Drafts → select the draft order → click "Send invoice" again to generate a new payment link. The new link has another 30-day window. If the order is already marked as cancelled, you'll need to recreate the draft. For B2B contexts where long payment terms are common, consider setting a reminder to resend the invoice before day 28.

Root Cause 2

Cart Permalink Variant ID Changed After Product Edit

Cart permalink URLs encode a specific variant ID: /cart/42345678901:1. In Shopify, every product variant — each combination of size, color, material — has a unique numeric ID. When you edit a product and add, remove, or restructure its variants, Shopify assigns new IDs to the changed variants. The old variant ID in your permalink no longer exists.

The failure is silent from the customer's perspective: the URL loads, the cart page shows, but the cart is empty. There is no error message at the URL level. Customers often assume the link is broken and give up rather than looking for the product manually. This breaks marketing emails, affiliate links, influencer posts, and any campaign that pre-loaded a specific variant into a customer's cart.

Fix Path

After any product variant edit, regenerate all cart permalinks for affected products. Find current variant IDs via Admin → Products → [product] → scroll to Variants → click a variant → the ID is in the browser URL bar. For campaigns where cart permalinks are baked into emails already sent, create a redirect: add a URL redirect in Admin → Online Store → Navigation → URL Redirects, redirecting the old cart URL to the new one.

Root Cause 3

Product Deleted, Unpublished, or Sold Out After Link Was Shared

Shopify Payment Links (the Admin feature) and cart permalinks both reference a product or variant. If the product is unpublished (removed from the Online Store sales channel), deleted, or the variant goes out of stock with an "deny" inventory policy, the payment link fails in different ways: an unpublished product shows the storefront password page or 404, a deleted product link returns 404, and a sold-out variant with "deny" policy loads the checkout but blocks completion at the payment step with "items are no longer available."

This commonly happens after a limited-time sale or product launch where items are taken offline after stock runs out — but the payment links or cart URLs continue circulating in customer inboxes and social posts.

Fix Path

For sold-out items: consider switching the inventory policy to "continue selling" if you can backorder, or update product status to "archived" to ensure the link shows a proper unavailable message rather than a broken checkout. For payment links shared before a product goes offline: publish a redirect or a waitlist landing page at the old product URL. Monitor your Admin → Analytics → Online store sessions report for 404 spikes that indicate broken circulating links.

Root Cause 4

Buy Button Iframe Blocked by CSP or Cross-Origin Restrictions

The Shopify Buy Button renders an iframe on your external website that loads the Shopify checkout. This iframe is blocked by the host site's Content-Security-Policy (CSP) header if it doesn't explicitly allow Shopify's domain in the frame-src directive. Many modern websites, especially those using security-forward frameworks or hosting platforms with default CSP headers, block cross-origin iframes by default.

The failure is invisible to most users: the iframe area either shows blank, shows a generic "this content cannot be displayed" error, or simply doesn't render at all. Enterprise users with strict browser security policies may see a browser-level block even when the host site's CSP is permissive. Mixed content is another common cause: if the host site is served over HTTP (not HTTPS), browsers refuse to load the HTTPS Shopify checkout iframe.

Fix Path

Add Shopify domains to your host site's CSP header: frame-src 'self' https://*.myshopify.com https://*.shopify.com https://checkout.shopifycs.com;. If you don't control the CSP header (e.g., third-party website builder), use the Buy Button's "Link" mode instead of the embed mode — it opens checkout in a new tab rather than an iframe. Ensure the host site is served over HTTPS. If using a CDN or hosting platform, check whether it sets security headers automatically.

Root Cause 5

Embedded Discount Code Expired or Reached Usage Limit

Cart permalinks commonly include an embedded discount code via URL parameter: /cart/[variant]:[qty]?checkout[discount_code]=WELCOME20 or via the redirect method /discount/WELCOME20?redirect=/cart/[variant]:[qty]. When the discount code expires, hits its maximum number of uses, or has customer eligibility restrictions that this customer doesn't meet, the checkout loads normally but the discount is not applied — there is no error message in the URL, and the checkout form shows the full price without the discount.

Customers who clicked the link expecting a discount may complete the purchase at full price without noticing, or abandon when they realize the promised price isn't reflected. Single-use codes embedded in affiliate links fail for every customer after the first redemption.

Fix Path

For volume-discounted campaigns: create discount codes with high usage limits rather than single-use codes. For affiliate links: use unique single-use codes per affiliate rather than one shared code. Use automatic discounts (Admin → Discounts → Automatic) instead of URL codes where possible — automatic discounts apply to all eligible carts without any URL parameter, eliminating the URL expiry problem entirely.

Deep Dive A

Cart Permalink URL Anatomy — How to Build Them and Why Each Part Breaks

Cart permalink URLs are powerful but fragile. Understanding the exact URL structure reveals which parts can fail independently and how to build more resilient links.

Deep Dive A — URL Structure

Cart Permalink URL Format Reference

Shopify cart permalink URLs follow a specific structure. Each component can fail independently:

Basic cart permalink: https://yourstore.com/cart/[variant-id]:[quantity] Example (one item): /cart/42345678901234:1 Multiple items (pipe-separated): /cart/42345678901234:1,98765432109876:2 With embedded discount code (parameter method): /cart/42345678901234:1?checkout[discount_code]=WELCOME20 With discount redirect (applies discount, then adds to cart): /discount/WELCOME20?redirect=/cart/42345678901234:1 With cart attributes (line item properties NOT supported): /cart/42345678901234:1?attributes[source]=instagram → The "attributes" parameter sets cart-level attributes, NOT line item properties → Line item properties (engraving, monogram, gift message) CANNOT be passed via URL → Must be set on the product page via Shopify form fields

Why variant IDs break: Shopify assigns numeric variant IDs internally. When you restructure a product's options — reordering, merging, or deleting variants — Shopify creates new variant records with new IDs. The old ID either no longer exists (404 when Shopify tries to find the variant) or maps to a different product variant than intended. There is no redirect from old to new variant ID.

Find current variant IDs: Admin → Products → [product] → click a variant → the URL shows the ID as variants/[id]/edit. Alternatively, use the Shopify Admin API: GET /admin/api/2024-01/variants.json?product_id=[product-id] returns all variant IDs for a product.

Deep Dive A — Resilient Link Strategy

Building Cart Links That Don't Break After Product Edits

The most resilient approach avoids encoding the variant ID in campaign URLs that you can't update after sharing:

Option 1: Product page with variant pre-selected (more resilient): /products/[handle]?variant=[variant-id] → Shows the product page with the specific variant selected → If the variant ID changes, falls back to the default variant (visible, not silent error) → Customer can still add to cart from the product page Option 2: Product page with option parameters: /products/[handle]?color=blue&size=medium → Selects variant by option values rather than ID → More resilient to variant restructuring (ID can change, option names usually don't) → Only works if the option names match exactly Option 3: Automatic discount + product page: /discount/SUMMER30?redirect=/products/[handle] → Applies discount cookie, then sends to product page → Customer selects variant themselves — no ID encoding at all → Discount applies automatically without URL parameters on the checkout
Deep Dive B

Draft Order Payment Links — Expiry, Resending, and B2B Edge Cases

Draft order payment links are the right tool for custom quotes, wholesale orders, and B2B sales — but their 30-day expiry and one-time-use nature create operational traps.

Deep Dive B — Draft Order Lifecycle

Draft Order Payment Link States and Transitions

A draft order payment link can be in one of several states, each with different behavior when a customer clicks it:

State: ACTIVE (link works) → Draft order status: Open → Link age: Under 30 days from last "Send invoice" → Customer clicks: Goes directly to checkout with order pre-filled State: EXPIRED (link returns 404 or homepage) → Draft order status: Open (order still exists, link just expired) → Link age: Over 30 days → Fix: Admin → Drafts → [order] → Send invoice again → new link generated → Customer gets new email with fresh 30-day link State: ALREADY_PAID (link shows "already processed") → Draft order status: Completed (converted to order) → Customer clicks: Error — payment cannot be made twice → Fix: Create a new draft order if a second payment is needed State: CANCELLED (link returns error or homepage) → Draft order status: Cancelled → Fix: Recreate the draft order — cancelled orders cannot be reopened State: CONVERTED_TO_ORDER (link may redirect to order status page) → Draft order was submitted but not yet paid → Behavior varies by Shopify plan and configuration

B2B tip: For net-30 payment terms, the draft order link will expire before the payment is due. Set a calendar reminder on day 25 to resend the invoice. Each resend generates a fresh 30-day link. Alternatively, use Shopify's B2B Net Terms feature (Shopify Plus) which handles deferred payment workflows natively without the link expiry problem.

Deep Dive B — Resending and Recovery

How to Resend a Draft Order Invoice

Resend invoice (generates new 30-day link): Admin → Drafts → [select draft order] → Click "Send invoice" button (top right of draft order page) → Choose: Send via email (Shopify sends) or Copy link (paste manually) → Each "Send invoice" action generates a fresh token Copy the payment link directly (to paste into email or message): Admin → Drafts → [select draft order] → Click "Send invoice" → Select "Copy link" tab → Paste into your own email client Check if link is expired without customer reporting it: Admin → Drafts → filter by "Open" status → Sort by "Created at" — orders older than 30 days with "Open" status have expired links → Use bulk actions: select all → "Send invoice" to refresh all open draft links at once
Deep Dive C

Buy Button — CSP Configuration, Domain Allowlist, and Checkout Extensibility

The Shopify Buy Button is the correct tool for selling through external websites, blogs, or email newsletters. Its iframe-based architecture introduces unique failure modes that don't exist with native Shopify storefront checkout.

Deep Dive C — CSP Fix

Fixing Content-Security-Policy Errors for Buy Button

When the Buy Button iframe is blocked by CSP, the browser console shows an error like: Refused to display 'https://checkout.shopify.com/...' in a frame because an ancestor violates the following Content Security Policy directive: "frame-ancestors 'self'".

Required CSP directives for Buy Button iframe: frame-src 'self' https://*.myshopify.com https://*.shopify.com https://checkout.shopify.com https://pay.shopify.com; Full example CSP header (add to your host site's server config): Content-Security-Policy: default-src 'self'; frame-src 'self' https://*.myshopify.com https://*.shopify.com; script-src 'self' https://sdks.shopifycdn.com; connect-src 'self' https://*.myshopify.com; WordPress (via plugin or functions.php): header("Content-Security-Policy: frame-src 'self' https://*.myshopify.com;"); Cloudflare (Transform Rules): Add response header modification: Content-Security-Policy append frame-src https://*.myshopify.com

Alternative if you can't modify CSP: Switch the Buy Button from "Embed" mode to "Link" mode in the Buy Button configuration. Link mode opens the Shopify checkout in a new tab instead of an iframe — no CSP requirements, no cross-origin restrictions. The tradeoff is a slightly higher friction UX (page navigation instead of inline checkout).

Deep Dive C — Checkout Extensibility Compatibility

Buy Button and Checkout Extensibility

Shopify's Checkout Extensibility (required for Shopify Plus stores to customize checkout after August 2024) changed how the checkout renders. Some Buy Button configurations created before the extensibility migration may reference deprecated checkout page structures or UI Extension slots that no longer exist in the new checkout architecture.

Signs the Buy Button is affected by Checkout Extensibility migration: → Cart button works but checkout loads blank or partially → Checkout extensions (upsells, custom fields) don't appear in Buy Button checkout → Different checkout experience between storefront and Buy Button Fix: Admin → Sales Channels → Buy Button → [your button] → Click "Edit" → "Regenerate embed code" → Replace old embed code on your external site with the new code → The regenerated code uses the current checkout architecture
Diagnosis Checklist

Payment Link Fix Checklist — Start Here

  • 1
    Identify which link type is broken Check the URL pattern: /a/orders/[token] = draft order link; /cart/[number]:[number] = cart permalink; Buy Button = iframe embed on external site. Each has a different fix path — diagnosing the wrong type wastes time.
  • 2
    For draft order links: check if the order is still open and under 30 days Admin → Drafts → find the order. If status is "Open" and it was created more than 30 days ago: resend the invoice via "Send invoice" → "Copy link". If status is "Completed" (already paid) or "Cancelled": the link is permanently invalid and a new draft or a regular order is needed.
  • 3
    For cart permalinks: verify the variant ID still exists Extract the variant ID from the URL (the number before the colon in /cart/[variant-id]:[qty]). In Admin → Products, search for that product and open the variant — if it no longer exists, the ID is stale. Rebuild the cart URL with the current variant ID found in the variant edit page URL.
  • 4
    For discount codes in URLs: check code status in Admin → Discounts Open the discount code in Admin → Discounts. Check: end date (has it passed?), usage count vs. usage limit (has it hit the cap?), customer eligibility (first-order-only codes fail for returning customers). For codes near their limit, increase the usage count or create a new code.
  • 5
    For Buy Button: check browser console for CSP errors Open the external site where the Buy Button is embedded, open browser dev tools (F12), go to Console tab, and look for "Refused to display" or "Content Security Policy" errors referencing Shopify domains. If found: update the host site's CSP header to include frame-src https://*.myshopify.com. If you can't modify CSP: switch Buy Button to Link mode.
  • 6
    Verify the product is published and in stock Admin → Products → find the product. Check: Sales channel visibility (must include "Online Store"), inventory count (must be above 0 if inventory policy is "Deny" or "Continue selling" as appropriate), product status (must be "Active", not "Draft" or "Archived"). A payment link or cart permalink pointing to an archived or out-of-stock product will either 404 or block checkout completion.
FAQ

Shopify Payment Links — Frequently Asked Questions

Why is my Shopify payment link not working?
The most common reasons: the draft order link expired (30-day token limit), the cart permalink references a deleted or edited variant, the product is out of stock or unpublished, or a discount code embedded in the URL has expired or hit its usage limit. Identify which link type you're using first — the fix is different for each. Draft order links: resend the invoice. Cart permalinks: regenerate with the current variant ID. Discount codes: check status in Admin → Discounts and increase the usage limit or create a new code.
How long does a Shopify draft order payment link last?
Draft order payment links expire after 30 days from when the invoice was sent. After that, the URL returns a 404 or redirects to the storefront homepage. To generate a fresh link, open the draft order in Admin → Drafts and click "Send invoice" again — this creates a new 30-day token. The draft order itself doesn't expire; only the payment link token does. For B2B orders with net-30 terms, set a reminder to resend the invoice on day 25 to avoid the link expiring before the customer pays.
Why is my Shopify cart permalink link showing an empty cart?
Cart permalink URLs encode the Shopify variant ID directly. When you edit a product — adding, deleting, or restructuring variants — Shopify assigns new IDs to the changed variants. The old ID in your permalink no longer maps to a product, so the cart silently loads empty instead of with the intended item. Fix: find the current variant ID (Admin → Products → [product] → click the variant → ID is in the URL bar) and rebuild the cart permalink. For future campaigns, consider linking to the product page instead of a cart permalink — product page URLs use the product handle which is stable across variant edits.
Why is the Shopify Buy Button not loading on my website?
The most common causes: (1) your website's Content-Security-Policy header doesn't allow iframes from Shopify domains — add frame-src https://*.myshopify.com https://*.shopify.com; to your CSP header; (2) the website is HTTP and Shopify checkout is HTTPS — browsers block loading HTTPS iframes in HTTP pages; (3) the Buy Button embed code is outdated and references old Shopify checkout architecture after a Checkout Extensibility migration — regenerate the embed code in Admin → Sales Channels → Buy Button. If you can't modify server headers, switch the Buy Button to "Link" mode in the settings to open checkout in a new tab instead of an iframe.
Why is the discount code in my Shopify cart permalink not applying?
Discount codes embedded in cart permalink URLs (?checkout[discount_code]=CODE) silently fail when the code is expired, has reached its maximum use count, or has eligibility restrictions the current customer doesn't meet (first-order-only, specific customer tags, minimum order value). The checkout loads normally at full price — there's no error in the URL or on the page. Check the discount code in Admin → Discounts: verify the end date, usage count vs. limit, and eligibility settings. For campaigns you can't update after sharing, use automatic discounts instead — they apply to all eligible carts without a URL parameter.
