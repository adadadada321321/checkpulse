Most merchants try to fix the problem in Shopify Admin and find nothing wrong.

System Architecture

Three Different Systems Merchants Call "Affiliate Code" — Each Fails Differently

The most common debugging mistake: treating affiliate codes, referral codes, and ambassador codes as the same thing. They are three distinct systems with different failure modes. Diagnosing the wrong one wastes an hour.

The core misunderstanding

Shopify Admin → Discounts shows every discount code — but affiliate attribution tracking is NOT a discount code. A customer can complete a checkout that correctly attributes a sale to an affiliate without any discount code being applied at all. Conversely, a referral discount code can be applied at checkout without any affiliate tracking firing. They are separate systems that happen to sound similar.

Silent Failure Deep Dives

Three Failures That Never Generate an Error Message

These failures generate support tickets — not error logs. By the time you investigate, the customer has already paid full price or abandoned the checkout entirely.

Silent Failure 1

The Cookie Death Problem — Affiliate Attribution Lost Between Sessions

Most affiliate tracking apps write a first-party cookie when the affiliate link is clicked. If the customer completes the purchase in the same browser session, the attribution is captured correctly. But if the customer clicks the link on mobile, later completes the purchase on desktop (or even just reopens the browser the next day), the tracking cookie may be gone — and the affiliate receives no commission credit.

Safari's Intelligent Tracking Prevention (ITP) caps first-party cookies set by JavaScript at 7 days and aggressively partitions third-party cookies. On iOS and macOS Safari, any affiliate cookie older than 7 days — or any cookie set by a cross-site request — is deleted. Firefox Enhanced Tracking Protection does similar. Chrome's Privacy Sandbox changes (post-third-party cookie deprecation) have made this worse for many affiliate setups.

The failure is completely silent: the customer sees no error, the merchant sees a completed order with a "direct" traffic source, and the affiliate sees their dashboard showing a click that never converted — even though it did. The fix is migrating to code-based affiliate tracking: each affiliate gets a custom shared discount code (SARAH15, MIKE10). The code enters the checkout URL and survives any browser, any device, any session. Commission is attributed when the code is used at checkout, not when a cookie fires.

Silent Failure 2

The Single-Use Referral Code Leak — One Burned Code Blocks All Future Customers

Referral apps generate a unique discount code for each new customer a referrer sends. That code is emailed to the new customer (the referee) and is typically set to single-use. This model has a structural exploit: the code is often delivered in a plain-text email that the recipient can forward, screenshot, or paste into a coupon community forum without realizing it invalidates the code for everyone else.

Coupon browser extensions (Honey, Capital One Shopping, RetailMeNot, PayPal Honey) automatically harvest referral codes from public forums and email screenshots and apply them to checkouts across all users. The first person to use the code via a coupon extension burns it. Every subsequent use returns "This discount code isn't valid" or "This discount has already been used." The original referee is left confused — they received a code, they're trying to use it, and it says it's already been used when they've never used it before.

To mitigate: set referral codes to a usage limit of 2-3 (one buffer above the intended use) so the original recipient still gets to use it even if a scraper burns the first use. Add a "new customer only" restriction — most coupon extension users are existing customers browsing non-incognito, so the restriction blocks the most common scraper profile. For high-value referral codes ($25+), consider one-time-use codes delivered via SMS rather than email to reduce the interception surface.

Silent Failure 3

The One-Code Limit — Referral Code Silently Blocks Every Other Promotion

Standard Shopify plans (Basic, Shopify, Advanced) enforce a hard limit of one discount code per checkout. This limit is platform-level — no app or theme setting can override it on standard plans. When a customer applies a referral code and then tries to apply a sale discount code, one of them is rejected. The rejection is usually a soft inline message ("This discount can't be combined with other discounts") that customers interpret as a checkout error rather than a deliberate restriction.

The support ticket pattern: "Your checkout is broken. I had a discount but it wouldn't let me apply a sale code." Most merchants investigate Shopify Admin, find both codes are valid and active, and can't reproduce the problem — because they test each code separately and both work fine in isolation. The conflict only appears when a customer has both at the same time.

The most common occurrence: a store-wide 20% off sale is running. A customer was referred by a friend last week and received a $15 off referral code. They add both to checkout. Whichever they entered first stays; whichever they entered second is rejected. They assume the store is broken and may abandon rather than complete at a reduced discount. Shopify Plus merchants can enable Discount Combine in Admin → Discounts and allow codes to stack. Standard plan merchants must use automatic discounts for sitewide promotions — automatic discounts apply without entering a code and do not count against the one-code limit.

Diagnosis Checklist

6-Step Affiliate and Referral Code Diagnosis

Start at Step 1 and stop as soon as you find the cause. Skipping to Step 4 before completing Step 1 is the most common time-wasting mistake.

  • 1
    Identify what type of code or tracking is failing Ask: is the customer reporting a discount code that won't apply (referral/ambassador), or an affiliate reporting that a sale they drove isn't showing in their dashboard (affiliate tracking)? These are different systems with different fixes. Do not investigate discount codes when the problem is tracking attribution, and vice versa.
  • 2
    For discount code failures: check the code in Shopify Admin → Discounts Search for the exact code string. Check: Times used counter, Active dates (start and end), Usage limits (total uses, per customer), Customer eligibility (all customers vs specific segments vs new customers only), and Minimum purchase requirements. One of these fields almost always contains the answer.
  • 3
    Check for the one-discount-code conflict Ask the customer: did you try to apply a second discount code at the same checkout? If yes, this is the one-code limit — not a bug. On standard plans, only one code per checkout. Confirm by asking what happened: if the referral code applied fine when entered alone but "stopped working" when they entered a second code, this is the cause.
  • 4
    For affiliate tracking failures: check your affiliate app dashboard Find the order in your affiliate app dashboard. If the order appears there but is unattributed (no affiliate linked), look at the traffic source in Shopify Analytics → Orders → [order] → Sessions. "Direct" traffic source with no referrer strongly suggests cookie loss. If the order doesn't appear in the affiliate app dashboard at all, the tracking script didn't fire — check for script blockers or app theme integration issues.
  • 5
    Check for coupon site code burn (for influencer / high-value referral codes) Search for the code string in Google: `"YOURCODE" site:retailmenot.com` or `"YOURCODE" coupon`. If you find it, a scraper has published it and Honey users are burning it. The code has been compromised. Issue the affected customer a new code directly and update your code generation strategy for future campaigns.
  • 6
    Run an end-to-end test from the affiliate link or referral email Use an incognito window. Click the exact affiliate link or use the exact referral code from the customer's email. Complete a test checkout. Verify: (a) if code-based — the discount appeared and applied, (b) if tracking-based — the test order appears attributed in your affiliate app dashboard within 15 minutes. If both work in your test but not for the customer, the customer's specific session has an unusual state (ad blocker, logged-in account conflict, or currency mismatch).

FAQ

Shopify Affiliate and Referral Codes — Frequently Asked Questions

Why is my Shopify referral code not working at checkout?
The most common reason a referral code fails is that it was a single-use code that was already redeemed — either by the intended customer, or by an unintended user who received or found the code via a coupon site or forwarded email. Go to Admin → Discounts, search for the code, and check the Times used counter. If it shows 1 use and your customer says they never used it, the code was burned by someone else — issue a new code directly to the customer. If Times used is 0 but the code still fails, check Customer eligibility: most referral apps restrict codes to new customers only, so returning customers see an "invalid" error even if they've never touched that specific code.
Why isn't my Shopify affiliate tracking working at checkout?
Affiliate tracking failures at checkout are almost always caused by cookie loss. Affiliate apps track attribution by writing a cookie when the customer clicks the affiliate link. If the customer clicks the link, browses, then completes checkout in a different browser session or on a different device — the cookie is gone. Safari ITP aggressively deletes tracking cookies, sometimes within hours. The result: the customer buys, the merchant sees a completed order with a "direct" traffic source, and the affiliate sees their click never converted. The fix is switching to code-based affiliate tracking: each affiliate gets a custom discount code (SARAH15). The code survives any device, any session, any browser — because it's typed at checkout, not stored in a cookie.
Can a referral discount code be combined with another discount code at Shopify checkout?
On standard Shopify plans (Basic, Shopify, Advanced), only one discount code is allowed per checkout — hard platform limit. If a customer has a referral code and a promotional code, one of them will fail. The practical workaround: switch to automatic discounts for store-wide sales. Automatic discounts apply without entering a code and don't count against the one-code limit. A customer can then apply their referral code AND benefit from your sitewide sale with no conflict. Shopify Plus merchants can enable Discount Combine in Admin → Discounts to allow codes to stack. Standard plan merchants cannot stack codes regardless of any app or setting.
Why does my influencer promo code say "code has already been used" when I just issued it?
If an influencer code is set to a single-use limit and is shared publicly (Instagram, YouTube, TikTok), coupon browser extensions harvest it and apply it to random checkouts. The first person to trigger the extension at your store burns the code. The influencer's actual audience then sees "already been used." Solution: set usage limits proportionally to your expected audience size — if the influencer has 5,000 followers, a limit of 50-200 uses is more appropriate than 1. Add a new-customer-only restriction to block most coupon extension users who are existing customers. Check if the code has been published on coupon sites by searching: `"YOURCODE" discount` in Google.
How do I test my Shopify affiliate or referral program end-to-end?
Run four tests: (1) Same-session affiliate test — click your affiliate tracking link in incognito, add to cart, checkout in the same session, verify attribution in your affiliate app dashboard within 15 minutes. (2) Cross-session cookie loss test — click affiliate link, close incognito, open new incognito, navigate to your store URL directly (not the affiliate link), complete checkout. Verify what your affiliate app shows — this simulates the most common real-world attribution failure. (3) Referral code test — issue a test code from your referral app, paste it at checkout in incognito as a new customer, verify the discount applies and the referrer gets credit. (4) Code conflict test — apply your referral code, then try adding a second promo code. Verify the error message is clear and expected, not a generic checkout error that customers might interpret as a system failure.
