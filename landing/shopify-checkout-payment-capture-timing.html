Shopify Payment Capture Timing — Authorize vs Capture, Manual Capture Not Working, Auth Expired

Orders stuck in "Authorized" status with no payment collected. Capture attempts failing days after fulfillment. Customers charged immediately on pre-orders. Confusion between voiding and refunding. Shopify's two-step payment system (authorization → capture) fails silently when authorization windows expire, manual capture is forgotten, or cards decline between the two steps.

Updated June 2026 · Covers Shopify Basic, Shopify, Advanced, Plus · Shopify Payments, third-party gateways, pre-orders, manual capture
The core issue: Shopify splits payment into two separate steps: authorization (bank holds funds, confirms card is valid) and capture (merchant actually collects money). In Shopify's default automatic mode, both happen instantly at order placement — invisible to merchants. When manual capture is enabled, authorization happens at checkout but capture is deferred until the merchant acts. Authorization holds expire after 7 days (most cards) to 30 days (Mastercard, Shopify Payments) — after that window, capture fails silently. The order stays "Authorized" in Shopify with no revenue collected and no automatic alert.
Architecture

Four Capture Modes in Shopify

Shopify supports four distinct payment collection patterns. Most merchants use only the default (automatic) and don't discover the others until they cause a problem.

Mode 1 — Default

Automatic Capture

Authorization and capture happen simultaneously at order placement. Customer's card is charged immediately. No merchant action required. Orders show "Paid" instantly. Available on all plans.

Mode 2 — Manual

Manual Capture

Authorization occurs at checkout, capture requires an explicit merchant action. Orders show "Authorized" until captured. Risk: auth window expires if capture is delayed more than 7–30 days depending on card network.

Mode 3 — On Fulfillment

Capture at Fulfillment

Some Shopify Plus apps and third-party gateways trigger capture automatically when fulfillment is marked. This creates risk if the fulfillment-to-capture trigger fails silently or if fulfillment is delayed past the auth window.

Mode 4 — Deferred

Deferred / Deposit Payment

Shopify Plus B2B net terms, deposit apps, or subscription apps collect partial payment at checkout and the remainder later. Each payment leg has its own authorization. Deferred capture requires a new payment method request if the original card is no longer valid.

Why "the customer paid" is not the same as "funds are in your account"

A successful authorization confirmation email to the merchant does not mean the money has moved. Shopify's payment status shows "Authorized" (hold placed, not collected) vs "Paid" (captured, funds in transit to your bank). Check the payment status in Shopify Admin → Orders → [Order] → Payment section. If it shows "Authorized" with a "Capture payment" button, the funds have not been collected.

Root Causes

Five Reasons Shopify Payment Capture Fails

Root Cause 1

Manual capture enabled but merchant forgets to capture — orders sit "Authorized" indefinitely

Manual capture is often enabled for a legitimate reason — fraud review, pre-order workflows, high-value orders — but without a process to ensure every authorized order gets captured within the auth window, orders accumulate in "Authorized" status. Shopify does not send any automatic notification when an order's authorization is about to expire. Merchants typically discover the problem when they attempt to capture an order after fulfillment and the capture fails, or when accounting reconciliation finds "Authorized" orders with no corresponding payment. Fix: Go to Shopify Admin → Orders → filter by Payment Status = "Authorized". This shows all uncaptured orders. Set a daily process to capture or void authorized orders before the auth window closes. Consider disabling manual capture unless you have a specific workflow that requires it — most fraud review use cases can be served by Shopify's automatic fraud analysis with order cancellation as the safety valve.

Root Cause 2

Authorization window expired before capture — capture fails silently, order shows "Authorized" with no payment

When a payment authorization expires, the bank releases the hold on the customer's funds. Any subsequent capture attempt returns an error from the payment gateway. Shopify may or may not surface this error prominently — in many cases, the order remains in "Authorized" status with no visible change, and the merchant doesn't know the capture failed unless they check the order Timeline. This is the most common cause of unrealized revenue for merchants using manual capture or fulfillment-triggered capture with long lead times. Fix: Check the order Timeline section in Shopify Admin for capture failure events. To collect payment after an auth expiry: navigate to the order, click the "..." menu, select "Resend invoice" to send the customer a payment link, or use a draft order to create a new payment request. You cannot recapture an expired authorization — a new authorization is required.

Root Cause 3

Card declined at capture time — valid at authorization but insufficient funds or card status changed by fulfillment

Authorization only verifies that the card exists, is not blocked, and has sufficient available credit at that moment. By the time capture is attempted — days or weeks later — the card's status can change: (1) debit card with exactly the right balance at authorization gets a purchase between auth and capture; (2) credit card reaches its limit from other purchases; (3) card is reported lost or stolen and reissued with a new number; (4) prepaid card balance depleted; (5) card expired (common for pre-orders with months-long fulfillment times). The capture attempt fails and the order stays in "Authorized" status. Unlike authorization failures which happen instantly at checkout, capture failures are asynchronous and easy to miss. Fix: Set up an order status automation or use Shopify Flow to monitor for capture failures and trigger a customer notification requesting updated payment details. For pre-orders with long timelines, consider charging a deposit at order time (which can be captured immediately) and collecting the balance at shipping.

Root Cause 4

Fulfillment-triggered capture workflow fails silently — order fulfilled but no capture event fires

Some merchants configure third-party apps or Shopify Flow workflows to automatically capture payment when an order is marked as fulfilled. This is a common pattern for dropshippers and print-on-demand merchants who want to avoid charging customers until the order ships. When the fulfillment event fires but the capture API call fails — due to a gateway timeout, app webhook failure, or Shopify rate limit — the order shows "Fulfilled" but payment status remains "Authorized". No alert fires. Revenue is not collected. This failure is particularly silent because both systems (fulfillment and payment) look correct in isolation. Fix: In Shopify Admin → Orders → filter by Fulfillment Status = "Fulfilled" AND Payment Status = "Authorized". Any orders in this state have a fulfillment without a capture — investigate the capture trigger. If you use Shopify Flow, check the Flow run history for the workflow. For critical capture workflows, add a monitoring step: Shopify Flow can trigger an email alert if payment status is still "Authorized" 24 hours after fulfillment.

Root Cause 5

Pre-order authorization expires during long fulfillment lead time — customer charged months after ordering

Pre-orders create a fundamental conflict with authorization-based payment: authorization windows are 7–30 days, but pre-order fulfillment may be 30, 60, or 90 days away. Two failure modes emerge: (1) if automatic capture is used, the customer is charged at order placement — months before receiving the product, creating support issues and chargebacks; (2) if manual capture is used, the authorization expires before fulfillment and capture fails, requiring merchants to contact customers for new payment. Neither mode matches customer expectations for pre-orders. Fix: Avoid using Shopify's native payment flow for pre-orders with long lead times. Instead: use a deposit app (Bold Deposits, Prepay) that captures a small authorization immediately and creates a separate charge request at fulfillment; or use Shopify's "draft order" flow where the invoice link is sent only when the order is ready to ship; or implement a clear-communication automatic capture (charge now, ship later) strategy with explicit customer consent at checkout.

Silent Failures

Seven Ways Payment Capture Fails Without Any Alert

These failures leave orders in a mismatched state — fulfilled but unpaid, or authorized but never captured — without surfacing an error visible to the merchant or customer.

Order fulfilled, payment never captured

Fulfillment-triggered capture workflow fails silently. Order shows Fulfilled + Authorized. No notification. Goods shipped, payment never collected.

Auth expiry — Shopify shows "Authorized" but nothing is collectible

Authorization expired days ago. "Capture payment" button still appears in Shopify admin but clicking it returns an error. No status change visible without checking Timeline.

Capture error buried in Timeline only

Gateway returns capture failure. Shopify logs it in the order Timeline but sends no email alert, no dashboard flag. Merchant must manually inspect each order to find it.

Void not possible after auth expiry — customer thinks charge was cancelled

Merchant tries to void an auth that already expired. Void fails silently. Bank already released the hold. No refund needed, but merchant and customer are confused about the status.

Card declined at capture — debit card, prepaid, or balance depleted

Auth succeeded on a debit or prepaid card. Funds depleted before capture. Capture fails. Order stays "Authorized" with no customer notification. Goods at risk of being shipped.

Third-party gateway shorter auth window than Shopify expects

Some international gateways or bank-issued cards use 5-day auth windows. Shopify's admin shows 7-day window advice. Merchant waits 6 days before capturing — already expired on the gateway side.

Shopify Payments account hold blocks capture

Shopify Payments account placed under compliance review between authorization and capture. Capture blocked at the platform level. Order Timeline shows failure; no email to merchant explaining the hold.

Diagnostics

How to Diagnose Payment Capture Issues

For Merchants

1

Filter orders by "Authorized" payment status

Shopify Admin → Orders → click "Payment status" filter → select "Authorized". Every order in this list has not been captured. Review auth dates and act before windows close. This should be a zero-count list if manual capture is not intentionally in use.

2

Check order Timeline for capture failure events

Open the problematic order → scroll to Timeline at the bottom. Look for "Payment capture failed" events with gateway error codes. This is the only place Shopify surfaces capture failure details. Common codes: "authorization_already_voided", "card_declined", "expired_card", "insufficient_funds".

3

Find fulfilled-but-unpaid orders

Shopify Admin → Orders → set two filters: Fulfillment Status = "Fulfilled" AND Payment Status = "Authorized". Any results indicate fulfilled orders where capture failed or was never attempted. These are your highest-priority revenue leaks — goods already shipped, payment not collected.

4

Verify your capture mode setting

Shopify Admin → Settings → Payments → scroll to "Payment capture" section. Confirm whether "Automatically capture payment" or "Manually capture payment" is selected. If manual capture was enabled accidentally (e.g., during a gateway change), this is the root cause of all "Authorized" orders.

5

Calculate auth window remaining per order

Auth window starts at the order creation timestamp. For Shopify Payments: 7 days for most card types, up to 30 days for some Mastercard. For third-party gateways: check the gateway's documentation or admin panel — windows can be 5–30 days. Prioritize capture on oldest "Authorized" orders first.

6

For expired auths: request new payment from customer

If the auth window has passed and capture fails: go to the order → click "..." menu → "Resend invoice" (sends a payment link via email) or use "Create draft order" to issue a new payment request. You cannot recapture an expired authorization — a fresh auth is required from the customer.

Deep Dives

Authorization Windows, Void vs Refund, and Capture via API

Deep Dive A

Authorization Window by Card Network

Authorization windows are set by card networks and issuing banks — Shopify has no control over them. The window starts at the moment of authorization (order placement), not at fulfillment or capture attempt.

Card Type / Network Typical Auth Window Notes
Visa credit card 7 days Shopify Payments follows this for most Visa cards
Mastercard credit card 30 days Longest window among major networks — good for pre-orders with short delays
American Express 7 days Amex can vary by issuing bank; some corporate cards are shorter
Discover 7 days Follows Visa-similar rules in most markets
Visa / Mastercard debit 5–7 days Banks vary widely; many issue only 5-day holds on debit. Most conservative window for manual capture workflows.
Prepaid cards 1–5 days Prepaid card programs set their own hold durations, often shorter than bank cards. Do not use manual capture for prepaid card orders.
International / local cards 3–7 days European, Asian, and Latin American issuers vary significantly. If your merchant gateway is international, verify windows in gateway docs, not Shopify docs.
The safest assumption: 5-day window

Because debit cards (which are widely used) can have 5-day windows and many international cards are shorter, any manual capture workflow should target capture within 5 days of authorization — not 7, not 30. The 30-day Mastercard window is the best case, not the rule to build your workflow around.

Deep Dive B

Void vs Refund — When Each Is Possible and How They Differ

Merchants frequently try to "refund" an authorized order that was never captured, or try to "void" a captured order. Understanding which is applicable prevents unnecessary gateway errors and customer confusion.

Action Payment Status Required Money Movement Customer Impact Timeline
Void "Authorized" (auth still active) No money moved — hold released Card hold removed; available balance restored immediately or within hours Instant to 24h for bank processing
Refund "Paid" / "Captured" Money transferred back to customer Credit appears on statement; takes 5–10 business days 5–10 business days
Void (expired auth) "Authorized" (auth expired) No action possible — hold already released by bank No impact — hold already removed by bank Not applicable
Refund on uncaptured order "Authorized" Some gateways process a $0 refund (no-op); others convert to void automatically Varies by gateway — Shopify Payments will void instead of refund if uncaptured Gateway-dependent
Shopify's behavior for "Refund" on an authorized order

When you click "Refund" on an order with "Authorized" payment status in Shopify admin, Shopify Payments automatically converts this to a void (since no capture occurred). The button still says "Refund" in the UI — but the underlying gateway call is a void. Third-party gateways may not support this automatic conversion; in those cases, "Refund" on an uncaptured order may fail with a gateway error or be treated as a zero-amount refund with no effect.

Deep Dive C

Capturing Payment via the Shopify Admin and API

Shopify offers three ways to capture an authorized payment: the admin UI, Shopify Flow, and the Admin API. Each has different use cases for merchants managing manual capture workflows at scale.

Admin UI (simplest, one order at a time)

Shopify Admin → Orders → open order → Payment section → click "Capture payment". Enter the amount (can be partial — useful for split shipments). Note: there is no bulk capture button in the UI. Each order must be captured individually unless you use Flow or API.

Shopify Flow (automated, event-triggered)

Trigger: "Order fulfilled" or "Fulfillment created". Action: "Capture payment". Condition: optionally filter on payment status = "Authorized". This creates a fulfillment-triggered capture without requiring custom code. Shopify Flow is available on all plans (Basic and above). The workflow runs asynchronously — check Flow run history for failures.

Admin API (bulk or conditional capture)

Use the POST /admin/api/2024-01/orders/{order_id}/transactions.json endpoint with {"transaction": {"kind": "capture", "amount": "XX.XX"}}. Requires write_orders scope. Supports partial capture (capture less than the authorized amount). Check the transaction response for status: "success" — a gateway-level failure returns status error with a message in the response body that Shopify admin may not surface clearly.

For merchants with high order volumes using manual capture, a daily automated sweep via the API or Shopify Flow is the only reliable way to ensure no authorized order ages past the auth window.
