Shopify Checkout Tip Not Working — Tipping Option Not Showing or Not Saving

Shopify tipping is off by default, only appears in standard checkout (not express), and requires Shopify Payments for reliable capture. Here is every failure mode — from the widget not appearing at all to tips silently disappearing from orders.

📅 Updated June 2026 ⏱ 7 min read 💡 Applies to: Shopify, Advanced, Plus (not Basic on legacy checkout)
Most Common Cause
Shopify tipping is disabled by default. Go to Shopify Admin → Settings → Checkout → scroll to "Tipping" → toggle ON. Without this step, no tip field appears at checkout regardless of plan or payment gateway.

How Shopify Tipping Works (and Where It Breaks)

Shopify's native tipping feature adds an optional tip widget to the payment step of checkout. Customers see preset percentage buttons (you configure these) plus a "Custom amount" field. The tip is added to the order total before payment is processed and stored separately on the order as tip_payment_details.

The feature must be manually enabled per store and has several constraints that aren't obvious from the admin UI:

  • Only appears in the standard checkout flow — express checkout buttons skip it entirely
  • Requires Shopify Payments for reliable tip capture (third-party gateways vary)
  • Available on Shopify plan and above — Basic plan stores on legacy checkout do not have it
  • Tip amounts are not included by default in order confirmation emails or packing slips

Failure Mode 1 — Tipping Not Enabled in Settings

The most common cause by far: the merchant never turned tipping on. The toggle lives in checkout settings, not in payment settings or theme settings.

How to enable Shopify tipping

  1. Go to Shopify Admin → Settings → Checkout
  2. Scroll down to the "Tipping" section
  3. Toggle "Show tipping at checkout" to ON
  4. Configure preset tip percentages (default: 10%, 15%, 20% — customize as needed)
  5. Optionally enable "Allow custom tip amounts" so customers can enter any value
  6. Click Save
  7. Test in a fresh incognito window using the standard "Check out" button
Tip
The tip widget appears on the payment step of checkout — the page where customers enter their card details. It does not appear on the information step (name/address) or shipping step. If you don't reach the payment step in testing, you won't see it.

Failure Mode 2 — Express Checkout Buttons Bypass Tipping

This is the most misunderstood behavior. All express checkout buttons skip the tip step. Shopify's tip widget lives on the checkout payment page — a step that express flows never show.

Checkout Method Shows Tip Option Notes
Standard "Check out" button (cart) Yes Full checkout — payment step includes tip widget
Shop Pay express No Skips payment step, goes directly to confirmation
Apple Pay No Payment sheet has no tip entry field
Google Pay No Same as Apple Pay — no tip support in payment sheet
"Buy now" product-page button No Routes to expedited checkout, skips payment page
PayPal express No PayPal-hosted flow has no Shopify tip widget
Shop Pay installments (BNPL) No Financing flow doesn't support additional tip amount

What to do if most customers use express checkout

  • Add messaging on the cart page: "Want to add a tip? Use 'Check out' — express pay buttons don't support tipping."
  • Consider disabling dynamic checkout buttons on product pages if tipping is important to your business model (e.g., service businesses, coffee shops with Shopify). Shopify Admin → Settings → Checkout → Dynamic checkout buttons.
  • For Shopify Plus: a checkout UI extension can add a tip-reminder banner visible even during express checkout flows, though the tip amount itself still can't be captured via express.

Failure Mode 3 — Tip Not Captured by Payment Gateway

Shopify Payments handles tip capture natively — when a customer adds a tip, the total (subtotal + shipping + tax + tip) is sent as one charge. Third-party gateways are more variable.

Payment Gateway Tip Capture Behavior
Shopify Payments (card) Reliable Tip added to total, single authorization
Shop Pay (accelerated) Skipped Express flow — tip step not shown
PayPal (standard) Varies Some PayPal integrations pass the full total including tip; others don't
Klarna / Afterpay / Affirm Not supported BNPL providers calculate repayment schedule on order total at session start — tip modifies total after this point
Stripe (via third-party app) App-dependent Depends on whether the Stripe app passes the modified total to Stripe
Manual / COD payments Order only Tip shows on order but no automatic capture — merchant must collect manually
BNPL Tip Gap
Klarna, Afterpay, and Affirm calculate repayment schedules at checkout session initialization. If a customer selects a BNPL option and then adds a tip, the tip modifies the order total after the BNPL authorization amount was set. Most BNPL providers reject the modified total or silently drop the tip to avoid recalculating installments. If you rely on tipping, BNPL payment methods create a structural conflict.

Failure Mode 4 — Tip Disappears from the Order

The customer added a tip, payment went through, but the tip amount doesn't appear in the order details. This is almost always caused by an app or automation that modifies the order total after checkout.

Common causes of tip stripping

  • Order editing apps: Apps that allow post-purchase order modification (add items, change quantities) may recalculate the order total and not account for tip_payment_details when re-summing.
  • Discount recalculation apps: Apps that apply or modify discounts after checkout (loyalty points redemption, post-purchase discount code application) can reset the order total calculation.
  • Checkout extensibility apps that modify pricing: Apps using the Cart Transform API or Checkout UI extensions that manipulate line items or totals may interfere with tip persistence.
  • Shopify Flow automations: If a Flow automation tags or edits the order immediately after creation, check whether the automation's actions touch the order financial data.
  • Third-party fulfillment apps: Some fulfillment integrations pull order data via API, process it, and push back a modified order that drops non-standard fields.

How to diagnose tip stripping

  1. Place a test order with a tip using Shopify's test payment method
  2. Immediately check Shopify Admin → Orders → the test order — is the tip visible?
  3. If yes initially: check again after 1–2 minutes to see if it disappears (indicating an async app ran)
  4. Check the order timeline at the bottom of the order page — any app activity logs appear here
  5. Disable recently installed apps one at a time and retest to isolate the culprit

Failure Mode 5 — Tip Not Showing in Order Confirmation or Packing Slip

Even when the tip is captured correctly and visible in the Shopify admin, it may not appear in customer-facing communications because the default templates don't include it.

Adding tip to the order confirmation email

  1. Go to Shopify Admin → Settings → Notifications
  2. Click "Order confirmation"
  3. Click "Edit code" (top right)
  4. Find the section that lists the order total breakdown (subtotal, shipping, taxes)
  5. Add a conditional block to display the tip when present:

The Liquid template variable is tip_payment_details on the order object. A common pattern:

{% if order.tip_payment_details %}
  <tr>
    <td>Tip</td>
    <td>{{ order.tip_payment_details.tip_amount | money }}</td>
  </tr>
{% endif %}
Third-Party Receipt Apps
Apps like Order Printer, Sufio, and Invoice Wizard use their own template systems. Not all of them expose tip_payment_details in their template variables. Check the app documentation or contact support to confirm. Some apps only surface the final order.total_price, which already includes the tip in the number even if it isn't labeled separately.

Failure Mode 6 — Tip Removed When Discount Code Applied

Native Shopify discount codes do not interact with the tip — the discount applies to the subtotal and the tip is a separate addition. However, if you use a third-party discount or pricing app, that app may recalculate totals in a way that drops the tip.

Diagnostic steps

  1. Test checkout: add items → proceed to checkout → add a tip → apply a discount code → observe if the tip persists
  2. If the tip disappears specifically when a discount code is applied: a discount app is recalculating and stripping it
  3. Test without discount apps active (temporarily disable in Shopify Admin → Apps)
  4. If tip persists without the app: contact the app developer to fix tip_payment_details preservation during discount recalculation
  5. If using Script Editor (deprecated): Scripts ran synchronously at checkout and could have stripped tip-related totals — migrate to Shopify Functions which have explicit access control over tip fields
Script Editor Deprecation Note
Shopify deprecated Script Editor in 2024. If your store was using a checkout script to modify pricing or discounts, it should have been migrated to Shopify Functions. Old Scripts that modified order totals may have had undocumented interactions with tip fields — if tip issues started around the time of a Script migration, the new Function implementation may need an explicit update to preserve tips.

Failure Mode 7 — Checkout UI Extension Conflict

Shopify Plus stores using checkout extensibility can install UI extensions that modify the payment page layout. The tip widget is rendered as a component in the payment section — extensions that inject content around the payment area can push the tip widget off-screen, hide it under overlapping elements, or trigger rendering errors that cause it to not render.

How to isolate a UI extension conflict

  1. Go to Shopify Admin → Settings → Checkout → Checkout customizations
  2. Note all active extensions
  3. Test checkout in incognito — confirm tip widget is missing
  4. Return to Checkout customizations and disable the first extension
  5. Retest checkout in a fresh incognito window
  6. Repeat until tip reappears — the last-disabled extension is the conflict
  7. Contact that app's support with the finding; they need to update their extension to not conflict with the native tip placement

Basic Plan Alternative
Stores on the Basic plan that don't have native tipping can use third-party tip apps from the Shopify App Store. These typically implement tipping as a special product that's added to the cart (e.g., a "$5 tip" product). This approach works across all plans but has trade-offs: the tip shows as a line item, tip reporting is more complex, and customers can remove it from the cart. Search the Shopify App Store for "tip" or "gratuity" for current options.

Where Tip Data Lives in Shopify

Understanding where tip data is stored helps when troubleshooting reports, custom email templates, or API integrations:

  • Shopify Admin: Shopify Admin → Orders → [order] → order total breakdown shows "Tip" as a separate line when present
  • Shopify API: order.tip_payment_details object — contains tip_amount and processing_method
  • Liquid (email templates): order.tip_payment_details.tip_amount — wrap in {% if order.tip_payment_details %}
  • Shopify Reports: Tip amounts are included in the "Sales by channel" and "Financial summary" reports under order financials
  • Exports: Shopify order exports (CSV) include a "Tips" column when tips are present on orders
Confirming a Tip Was Captured
After a tip is captured successfully, Shopify Admin → Orders → [order] shows the financial breakdown: Subtotal, Shipping, Taxes, Tip, and Total. If you see the Tip line with the correct amount, capture succeeded. If the Tip line is absent but the customer selected one, the tip was dropped somewhere in the payment or order-processing chain.
