Shopify Returns & Exchange Checkout Not Working — Store Credit, Loop Returns, Refund Portal

Shopify has no native exchange checkout. Exchanges run through three separate paths — refund + new order, store credit redemption, or Plus draft order — and each one fails differently. Here's how to diagnose and fix every scenario.

How Shopify Handles Returns and Exchanges

Shopify's return flow lives in two separate systems: refunds (money flowing back) and new orders (the exchange). There is no single "exchange checkout" built into standard Shopify. What most merchants call an exchange checkout is actually one of three separate mechanisms stitched together by the merchant or a third-party return app.

Key insight: When an exchange checkout fails, the failure is almost never in Shopify's core checkout — it's in how the return credit was issued (gift card vs. app credit vs. draft order) and how that credit is being redeemed. Each path has distinct failure modes.
Refunds

Money-back mechanism

Shopify Payments, PayPal, or third-party gateway reversal. Processed by Shopify admin; timeline controlled by the payment network (5–10 business days for cards).

Store Credit

Gift card or app credit

Shopify issues a gift card code (redeemable at any checkout) or routes through a store credit app like Rise.ai (redeemable only through that app's portal).

Exchange Order

New checkout or draft order

Standard: customer places a new order normally. Plus: admin creates a draft order with a negative return line item. Apps: return portal creates a new cart automatically.

Three Exchange Paths — and Where Each Breaks

Every exchange on Shopify uses one of these three mechanisms. Identifying which one your store uses is the first step in any diagnosis.

Path 1: Refund + New Order (Standard Shopify)

The merchant issues a refund (to card or as a Shopify gift card). The customer places a completely separate new order and applies the gift card code at checkout. This is the simplest path with the fewest failure modes — but customers find it confusing because the two steps are disconnected.

Most common failure: Customer receives the gift card email but the code doesn't work at checkout. Usually the code was already used, the customer is trying to stack it with a coupon that conflicts with an automatic discount, or the card was issued to a different email.

Path 2: Return App Exchange Portal (Loop, Aftership, Happy Returns)

A return app presents a branded portal where the customer selects their exchange item. The app creates a new Shopify cart and applies a discount code or gift card equivalent to the return credit. The customer checks out through a Shopify checkout session that the app controls.

Most common failure: The app-generated discount code conflicts with an active automatic discount on the store, blocking the customer from checking out with their exchange credit applied.

Path 3: Shopify Plus Draft Order Exchange

In the Shopify admin order view, Plus merchants see a "Create exchange" button. This creates a draft order with: (a) a positive line item for the exchange product, and (b) a negative line item for the returned product. Shopify generates a checkout URL for the customer to pay any balance difference. If the exchange value exceeds the original, the customer pays the difference. If it's less, the merchant issues a partial refund after the exchange is completed.

Most common failure: The draft order checkout URL expires (~30 days), or the negative line item makes the order total zero and Shopify won't process a $0 checkout through the draft order path — the merchant must manually create a free order instead.

Five Root Causes of Exchange Checkout Failure

Cause 1 — Most Common

Discount stacking conflict

Return app applies a discount code for the credit. If an automatic discount is active on the store, Shopify blocks the code: "you can only use one discount per order." Non-Plus only.

Cause 2

App store credit not native

Rise.ai, Gorgias Credits, and similar apps use internal ledgers, not Shopify gift cards. Their "store credit" only works in their portal — not at a regular Shopify checkout.

Cause 3

Draft order URL expired (Plus)

Shopify Plus exchange draft orders expire if the customer hasn't completed checkout in ~30 days. The URL returns a "checkout unavailable" error. Resend the invoice from the draft order.

Cause 4

Exchange item now out of stock

Return apps pre-reserve inventory when the customer selects an exchange item in the portal. If that reservation expires before checkout completes, the item is OOS and checkout fails.

Cause 5

Gift card code already used or expired

If the original return generated a Shopify gift card and the customer already used part of it, the remaining balance may be less than expected. Some third-party apps also set expiry dates on gift card codes.

Return App Exchange Checkout Errors

Loop Returns, Aftership Returns Center, Happy Returns, Returnly (now Loop), and similar apps all build on top of Shopify's cart and checkout APIs. Their exchange portals create new checkout sessions that can fail for app-specific reasons distinct from normal Shopify checkout errors.

Loop Returns

Loop creates a new cart with a generated discount code equal to the return credit. This discount code is single-use and tied to the specific exchange session.

  • Session expired: Loop exchange sessions are time-limited (~24 hours). If the customer doesn't complete checkout, the session and discount code both expire. Customer must start a new return request.
  • Automatic discount conflict: If your store has an active automatic discount (sitewide sale, seasonal promo), Loop's discount code is rejected at checkout. You must temporarily pause the automatic discount, or the customer must check out without the automatic discount (non-Plus can't stack codes).
  • Variant out of stock mid-session: Loop pre-reserves the exchange variant for a limited window. If another customer purchases that last unit before this checkout completes, Loop's cart becomes OOS. The customer must pick a different variant.
  • Loop portal blank page: Usually a cookie or third-party script blocking issue. Loop's portal runs on a Loop-owned domain; if the customer's browser blocks third-party cookies, the session token can't be passed to Shopify's checkout. Try incognito mode or a different browser.

Aftership Returns Center

Aftership issues a Shopify gift card (not a discount code) for the return credit. This is more robust than Loop's discount code approach because gift cards stack with discount codes and aren't blocked by automatic discounts — but it has its own failure modes:

  • Gift card issued to wrong email: Aftership sends the gift card to the email on the original order. If the customer changed their email after ordering, the gift card goes to the old address.
  • Gift card amount doesn't match return value: If a return has multiple items with different return reasons (one exchange, one refund), the gift card may reflect only the exchange portion. Check the Aftership return details vs. the gift card balance in Shopify admin → Gift cards.
  • Aftership exchange checkout link broken: Aftership generates a pre-filled cart URL for the exchange item. These cart URLs expire after the cart token expires (10 days by default in Shopify). If the customer clicks the link after 10 days, the cart is empty and the gift card still works — but they have to add the item manually.

Happy Returns

Happy Returns routes returns through physical drop-off locations and then issues refunds or exchanges back through the merchant's system. Exchange checkout issues here are typically Shopify integration sync delays — Happy Returns marks the exchange complete, but the new Shopify order takes up to 30 minutes to appear due to webhook processing queues.

General rule: If the return app's portal shows a success screen but the exchange order doesn't appear in Shopify, check the return app's dashboard for "failed order creation" errors. These are usually caused by out-of-stock items, a product that was deleted since the original purchase, or a variant ID that changed after a Shopify product re-import.

Store Credit Not Applying at Checkout

There are two fundamentally different types of "store credit" in Shopify's ecosystem, and they work completely differently at checkout. Mixing them up is the most common source of confusion.

Type A — Shopify Gift Card

Native. Works at any checkout.

Shopify issues a gift card code (e.g., ABCD-1234-EFGH-5678). Customer enters it in the gift card field at checkout. Works on any Shopify store checkout, stacks with discount codes, no account required.

Type B — App Store Credit

App-locked. Only works in that app's portal.

Rise.ai, Gorgias Credits, Smile.io store credit — these are balance entries in the app's own database. They are NOT Shopify gift cards. They can only be redeemed through that specific app's checkout extension or portal. A customer cannot enter a Rise.ai credit at a normal Shopify checkout — it won't show up in the gift card field.

Why Gift Card Codes Fail at Checkout

Check the gift card balance in Shopify admin

Go to Shopify admin → Gift cards → search by last 4 digits or customer email. Verify the balance matches what the customer expects. If it's $0, it's already been used.

Verify the exact code format

Shopify gift card codes are typically 16 characters in 4 groups of 4 (XXXX-XXXX-XXXX-XXXX). Some apps generate shorter or longer codes. The code is case-insensitive but must match exactly. Look for common transcription errors: 0 vs O, 1 vs I, 1 vs l.

Check if the gift card is disabled

In Shopify admin → Gift cards → click the specific card. It may show as "disabled" if a merchant or app disabled it after issuance. Re-enable it from the gift card detail page.

Check for automatic discount conflict

A gift card code is NOT a discount code — it doesn't conflict with automatic discounts. If the customer is also entering a discount/coupon code AND there's an active automatic discount, the coupon will be blocked, but the gift card will still apply.

Shopify Plus Draft Order Exchange Issues

The "Create exchange" feature in Shopify Plus admin generates a draft order. Understanding the draft order structure is essential for diagnosing checkout failures.

How it works: The draft order contains two line items: (1) the new exchange product at its current price, and (2) a negative line item for the returned product equal to its original paid price. The customer receives a checkout URL to pay any positive balance. If the exchange is equal in value, the checkout total is $0 — and Shopify cannot process a $0 draft order checkout (it requires at least one real payment).

Refund Not Appearing — Timeline by Payment Method

When a customer says "I haven't received my refund," the issue is rarely Shopify — it's the payment network timeline. Merchants issue the refund instantly in Shopify admin; the bank or payment processor controls when it appears on the customer's statement.

Payment Method Refund Processing Time Where Refund Appears
Shopify Payments — credit card 5–10 business days Customer's card statement (may show as a pending credit first)
Shopify Payments — debit card 2–5 business days Customer's bank account / debit card statement
PayPal Instant to PayPal balance; 3–5 days to linked bank Customer's PayPal account → they may need to transfer to bank
Shop Pay Installments 3–5 business days for current installments; future installments cancelled Affirm/provider account; remaining installment schedule is cancelled
Shopify gift card (issued instead of cash refund) Immediate Gift card code sent to customer email
Manual payment (bank transfer, check) Merchant-controlled; no Shopify automation Depends on merchant's process
Closed or expired card: If the customer's original payment card was closed, reported stolen, or expired since the purchase, the card network typically redirects the refund back to the store. Shopify admin will show the refund as "completed" — but the customer never receives it. In this case, you must issue a replacement refund outside Shopify (bank transfer, check, or Shopify gift card). There is no way to redirect a card refund to a different payment method through Shopify.
Refund limit: Shopify Payments can only refund up to the original amount charged on that specific order. You cannot refund more than the customer paid, even if the return value exceeds it (e.g., product price increased since purchase). For overage amounts, issue a Shopify gift card for the difference.

Diagnostic Checklist — Exchange Checkout Not Working

Identify which exchange path is being used

Is the customer: (A) trying to apply a gift card code at a regular checkout, (B) clicking a link from a return app's portal, or (C) using a Shopify Plus draft order checkout URL? Each path has a completely different set of causes.

For gift card codes — check admin

Shopify admin → Gift cards → search the code. Verify balance is correct, card is active (not disabled), and code hasn't already been fully redeemed. If balance is $0, the code was already used.

Check for automatic discount conflict

Shopify admin → Discounts → filter by "Active" and look for automatic discounts. If any automatic discount is active AND the return app is applying a discount code, non-Plus stores will see "you can only use one discount per order." Pause the automatic discount to test.

For app store credit — check if it's a native gift card

Is the "store credit" a 16-character Shopify gift card code (works anywhere) or an internal app balance (Rise.ai dashboard, Gorgias credit wallet)? App balances cannot be redeemed at a regular Shopify checkout — only through the app's own checkout UI extension.

For Plus draft orders — check URL validity

Open the draft order in Shopify admin and look at its status. If it shows "Draft" (not "Completed"), resend the invoice to generate a fresh checkout URL. Check that a shipping line is included and that the total is positive (not $0 or negative).

For return app failures — check the app's error log

Loop, Aftership, and similar apps have their own admin dashboards with return status and error details. Look for "order creation failed," "discount code invalid," or "variant unavailable" errors. These are app-side failures and require action in the return app's settings, not in Shopify admin.

For refund timing — communicate timeline proactively

Verify the refund was issued in Shopify admin (Orders → select order → Refund section shows "refunded" status). If yes, share the payment-method-specific timeline from the table above. If the customer's card was closed, you'll need to issue a replacement by alternative means.
