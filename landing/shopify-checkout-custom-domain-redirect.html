Root causes

6 Reasons Your Shopify Checkout Redirects to the Wrong Domain

Shopify checkout domain redirect failures fall into three groups: DNS misconfigurations (wrong records or expired verification), proxy/CDN conflicts (Cloudflare SSL mode, page rules), and post-migration issues (domain change left stale links or open sessions). Each has a specific fix. The six most common:

Shopify Admin Config
01

Custom domain is not set as the primary domain in Shopify

Shopify uses the primary domain to generate all checkout URLs. If your custom domain (e.g., www.mystore.com) is connected but not set as primary, checkout URLs are built from your .myshopify.com fallback domain instead. Customers reach the cart at your custom domain, then get redirected to your-store.myshopify.com/checkout — an unexpected domain that looks suspicious to customers and may trigger browser security warnings.

This is the most common cause after a new custom domain is added without completing the setup. The domain connects (DNS propagates, Shopify verifies it) but the "Set as primary" step is skipped.

Fix: Shopify Admin → Settings → Domains → Find your custom domain in the list → Click the three-dot menu (⋯) next to it → Select "Set as primary domain" → Confirm — checkout URLs will now use your custom domain → Test: add a product to cart → proceed to checkout → verify the URL bar shows your custom domain
DNS Issue
02

DNS records are no longer pointing to Shopify

Shopify periodically re-verifies that your custom domain's DNS records still point to its servers. If you changed DNS records (e.g., moved to a new DNS provider, modified A or CNAME records, or accidentally deleted the Shopify entry), Shopify's verification fails and the domain is flagged as disconnected. Checkout reverts to the .myshopify.com domain automatically — often without any warning in the admin.

Also occurs after domain transfers between registrars, after adding a page builder or custom server in front of your store (which can override the DNS), or after a Cloudflare proxy record change that misroutes the checkout subdomain.

Correct DNS settings for Shopify: → www subdomain: CNAME → shops.myshopify.com → Root domain (@): A record → 23.227.38.65 (Shopify's IP) OR: redirect root to www at your registrar, not via DNS A record After updating DNS: → Allow 24–48 hours for full propagation → Check status at Shopify Admin → Settings → Domains → A green checkmark means Shopify verified the connection
SSL / HTTPS Conflict
03

Cloudflare SSL mode set to "Flexible" — causes redirect loop

This is the #1 cause of ERR_TOO_MANY_REDIRECTS on Shopify checkout. When Cloudflare is in front of your Shopify store with SSL mode set to "Flexible", it connects to Shopify over HTTP while presenting HTTPS to the visitor's browser. Shopify's HTTPS enforcement detects the incoming HTTP request and redirects to HTTPS. Cloudflare forwards the redirect over HTTP again. The loop continues until the browser gives up after 10 hops.

Flexible SSL is Cloudflare's default for new domains added without an existing SSL certificate. It works for some sites — but not for Shopify, which hard-enforces HTTPS on all URLs including checkout.

Fix: → Cloudflare Dashboard → your domain → SSL/TLS → Overview → Change SSL/TLS encryption mode from "Flexible" to "Full (strict)" → "Full (strict)" means Cloudflare connects to Shopify over HTTPS and validates the certificate → Shopify has a valid SSL cert on shops.myshopify.com, so this works without additional setup → Clear browser cache and test checkout immediately after the change
DNS / SSL Issue
04

SSL certificate not yet issued — "Not Secure" warning after domain change

When you add or change a custom domain in Shopify, the platform automatically provisions an SSL certificate via Let's Encrypt. This takes between 15 minutes and 48 hours. During this window, your checkout may be accessible only over HTTP (no certificate), which causes browsers to show a "Not Secure" warning or a hard block on the checkout URL. Customers on Chrome or Safari with strict HTTPS settings will see an error page and be unable to proceed.

This failure is temporary, but it can cost significant revenue if it happens during a sale or after a scheduled domain migration that goes live during high-traffic hours.

Check and accelerate SSL provisioning: → Shopify Admin → Settings → Domains → click your domain → Look for SSL status: "Pending SSL" (still provisioning) or "SSL secured" (active) If pending for 48+ hours, common blockers: → CAA DNS record blocking Let's Encrypt: check for CAA records at your registrar Add: yourdomain.com. CAA 0 issue "letsencrypt.org" or remove the restricting CAA record entirely → DNS not fully propagated: verify with dns.google → enter domain → check all nameservers → Cloudflare proxy (orange cloud) can delay Let's Encrypt validation: Temporarily set the record to DNS-only (grey cloud) during provisioning
Post-Migration
05

Stale checkout links in emails, ads, or apps still point to the old domain

After a domain change, hardcoded checkout URLs across your Shopify setup continue pointing to the old domain. Shopify email notifications (abandoned cart, order confirmation), paid ad destination URLs, affiliate links, and checkout recovery links in SMS tools all generate checkout URLs based on the domain that was primary when they were created or last sent.

Customers who click a checkout recovery email sent before the domain change are routed to the old domain. Depending on whether the old domain still resolves, they may see a redirect (if you've set up forwarding), a 404, or a Shopify "this store doesn't exist" error page.

Post-domain-change audit checklist: → Shopify Admin → Settings → Notifications → Open every email template that contains a checkout or cart URL → Click "Preview" — verify the checkout link uses the new domain → Update any templates with hardcoded old-domain URLs → Google Ads / Meta Ads → check destination URLs on all active campaigns Update checkout-step destination URLs to new domain → Shopify Apps: checkout recovery (Klaviyo, Omnisend, Postscript) → Re-test flows with a real checkout to ensure links use the new domain → Affiliate/referral links: notify affiliates of domain change, provide updated link format
Shopify Plus Only
06

checkout.shopify.com redirect is intentional — but Shopify Plus stores can customize it

Since mid-2023, Shopify migrated all non-Plus checkout URLs to checkout.shopify.com. The custom checkout subdomain (e.g., checkout.mystore.com) is now a Shopify Plus-exclusive feature. If you are on a Basic, Shopify, or Advanced plan, your checkout will redirect to checkout.shopify.com/{token} and this is expected behavior — not a failure.

For Shopify Plus merchants who want to maintain checkout.yourdomain.com, the setup requires a CNAME record and domain configuration in Plus Admin. If the CNAME is missing or misconfigured, checkout falls back to the generic checkout.shopify.com domain.

For Shopify Plus — custom checkout domain setup: → Shopify Plus Admin → Settings → Domains → Checkout domain → Add checkout.yourdomain.com as the checkout domain → At your DNS provider, add: CNAME: checkout.yourdomain.com → checkout.shopify.com → Wait for Shopify to verify the CNAME (15 min – 24 hrs) → Test by going through checkout and verifying the URL bar shows checkout.yourdomain.com throughout the flow Note: Non-Plus plans cannot use a custom checkout domain. Upgrading to Plus is the only way to restore checkout.yourdomain.com.
Why monitors miss this

Why Your Uptime Monitor Won't Catch Checkout Domain Redirect Failures

Checkout domain redirect failures are invisible to standard uptime monitoring because monitors check whether a URL responds — they don't follow the full redirect chain from cart to payment confirmation. When a checkout domain redirect breaks, your store's homepage returns 200 OK. Your uptime monitor reports everything healthy. The failure only surfaces when a customer's browser follows the redirect chain and hits a loop, a certificate error, or an unexpected domain.

If any hop in the chain fails, the monitor should alert immediately — not after a customer complaint.

FAQ

Shopify Checkout Domain Redirect — Frequently Asked Questions

Why does my Shopify checkout redirect to checkout.shopify.com?
Since 2023, Shopify moved all checkout pages to checkout.shopify.com for non-Plus stores. This is intentional. Custom checkout subdomains (e.g., checkout.yourstore.com) are now a Shopify Plus-exclusive feature. If you are on a Basic, Shopify, or Advanced plan, the checkout.shopify.com redirect is expected and cannot be changed without upgrading to Plus.
Why does my Shopify checkout redirect to myshopify.com instead of my custom domain?
Your custom domain is likely connected to Shopify but not set as the primary domain. Go to Shopify Admin → Settings → Domains → find your custom domain → click the three-dot menu → "Set as primary domain". Shopify builds all checkout URLs from the primary domain. If your custom domain is not primary, checkout redirects to the .myshopify.com fallback.
Why is there a redirect loop on my Shopify checkout?
The most common cause is Cloudflare SSL mode set to "Flexible" instead of "Full (strict)". With Flexible SSL, Cloudflare connects to Shopify over HTTP while presenting HTTPS to the browser. Shopify redirects to HTTPS, Cloudflare forwards it over HTTP again — and the loop starts. Fix: Cloudflare → SSL/TLS → change to "Full (strict)".
Why does Shopify checkout show a "Not Secure" warning after I changed my domain?
Shopify automatically provisions an SSL certificate for new custom domains via Let's Encrypt. This takes 15 minutes to 48 hours. During this window, checkout may be served over HTTP, causing browser security warnings. Check status at Shopify Admin → Settings → Domains. If provisioning is stuck after 48 hours, look for a CAA DNS record blocking Let's Encrypt, or temporarily set the Cloudflare record to DNS-only during SSL issuance.
Does changing my Shopify domain break checkout for existing customers?
Yes, temporarily. Customers with open checkout sessions or who click old checkout links (in abandoned cart emails, ads, or affiliate links) will be routed to the old domain. After a domain change, audit your email notification templates, ad destination URLs, and any third-party checkout recovery apps — all need to be updated to use the new domain. Test checkout from a fresh browser session to confirm the new domain works end-to-end.
Can my uptime monitor detect a Shopify checkout domain redirect failure?
No. Standard uptime monitors check whether your homepage URL returns 200 OK — they don't follow the full checkout redirect chain. A redirect loop or SSL error on checkout is completely invisible to ping-based monitoring. You need a tool that simulates the complete checkout flow, follows every redirect hop, and verifies SSL at each step.
