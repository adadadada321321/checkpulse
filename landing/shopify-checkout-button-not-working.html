What Customers Actually See

When the checkout button is broken, the failure looks deceptively minor. There is no error message, no crash, no redirect failure. Customers see a checkout that appears fully functional — address fields, payment section, order summary — but the final action is blocked:

  • A greyed-out "Complete order" button that cannot be clicked regardless of what the customer fills in
  • A button that flashes enabled then immediately disables again — a form validation race condition
  • A button that looks active but does nothing on click — a JavaScript error absorbing the click event
  • A spinner on the button that never resolves — a silent API call failure during submission

In every variant: no order can be placed. Most customers assume your store is broken and leave. Very few will contact you to report it. The failure accumulates silently until you notice revenue has dropped.

How Shopify's Checkout Button Actually Works

Shopify's checkout button (Complete order or Place order) is disabled by default. JavaScript event listeners on the checkout form monitor field validity and enable the button only when all conditions are met:

  • All required fields are non-empty and valid (email, shipping address, billing address)
  • A payment method is selected (payment section rendered and a method chosen)
  • Any required consent checkboxes are checked (Terms & Conditions, age verification)
  • No form-level validation errors are present

If any one of these conditions fails — or if the JavaScript that monitors these conditions throws an error — the button stays grey. This is why a greyed-out button does not always mean the customer hasn't filled in the form correctly. It can mean the button's enable logic itself is broken.

# Simplified checkout button state logic
button.disabled = true // always starts disabled
allFieldsValid = checkRequiredFields() // false if any field empty/invalid
paymentSelected = checkPaymentMethod() // false if payment section empty
consentGiven = checkConsentBoxes() // false if T&C unchecked
if (allFieldsValid && paymentSelected && consentGiven) {
button.disabled = false // ← button enables here
}
// If any condition is broken, or if the JS above throws: button stays grey

All 8 Causes of a Greyed-Out Checkout Button

#1

Payment Methods Not Loaded

This is the most common non-obvious cause. If the payment section fails to load — due to a Shopify Payments suspension, gateway misconfiguration, or network error — the payment step renders blank. With no payment method selectable, Shopify's checkout logic cannot mark paymentSelected = true, so the button stays disabled regardless of how correctly the customer fills in every other field.

This is also why a "greyed out button" and a "no payment methods" error are often the same underlying problem seen from different angles.

FIX → Check Shopify Checkout No Payment Methods guide for all 9 payment loading failure causes. Start with Settings → Payments in your admin.
#2

Required Field Missing or Invalid

Shopify's checkout has required fields that vary by store configuration: email, phone number, shipping address (street, city, postcode), and sometimes billing address if it differs from shipping. If any of these fields are empty, contain an invalid format, or are hidden by a theme customization and auto-filled with invalid placeholder data, the button validation fails.

This is the most common cause for individual customers but rarely affects all customers simultaneously.

FIX → Test checkout in incognito, filling every field carefully. Open browser console (F12) and look for validation errors on submit attempt. Check your theme code for any fields being hidden or auto-filled with invalid values.
#3

Terms & Conditions Checkbox Not Checked

Many Shopify themes add a Terms and Conditions consent checkbox to checkout. If your theme includes this checkbox and a customer hasn't checked it, Shopify treats consent as absent and keeps the button disabled. The checkbox is sometimes rendered below the fold or in an easy-to-miss position on mobile.

This is a common cause of customer complaints — they fill everything in correctly but the button never enables, not realising there is an unchecked checkbox further down the page.

FIX → Test checkout on mobile. Scroll down to look for any unchecked consent checkbox. If you want to remove the T&C requirement: Online Store → Themes → Edit Code → search for "terms" in your theme files. Or contact your theme developer.
#4

JavaScript Error in Checkout

Shopify's checkout page loads several JavaScript files — Shopify's own checkout scripts, theme scripts, and any scripts injected by apps. A JavaScript error in any of these can prevent the button's event listeners from being set up correctly, meaning the button is never re-evaluated after fields are filled and stays disabled permanently.

JS errors in checkout are especially common after a theme update, a Shopify platform update, or a checkout app update that introduces a script conflict.

FIX → Open your checkout in Chrome, press F12, click "Console." Reproduce the issue and look for red error lines. Common culprits are uncaught TypeError or ReferenceError in a script from a checkout app. Disable checkout apps one by one to isolate the conflict.
#5

Checkout App or Extension Conflict

Shopify's checkout extensibility framework allows apps to inject UI components, event handlers, and validation logic into the checkout. A bug in a checkout app — particularly fraud detection apps, loyalty apps, upsell apps, or subscription billing apps — can override the button state and keep it permanently disabled, even when all native Shopify conditions are met.

This failure often appears suddenly after an app update, or when two apps both attempt to control the button state simultaneously.

FIX → Shopify Admin → Settings → Checkout → Checkout extensions. Disable extensions one at a time, test checkout after each. Identify the conflicting app and contact its developer or roll back to the previous version.
#6

Theme Customization Overriding Button Logic

Custom theme development sometimes adds JavaScript to the checkout to implement additional UX — custom field validation, order note requirements, upsell accept/decline logic. If this custom script contains a bug, it can prevent the button from enabling. This is most common with heavily customized themes and Shopify Plus accounts with checkout.liquid customizations.

FIX → Review any custom scripts in your theme's checkout.liquid (Shopify Plus) or in any theme Script Tags. Temporarily remove custom scripts to test whether the button enables correctly without them. Use the console to identify which script is throwing.
#7

Cart Changed During Checkout (Item OOS or Price Change)

If a cart item goes out of stock or its price changes while a customer is mid-checkout, Shopify detects the mismatch and may disable the button with an inline error about an "unavailable item" or "price change." The customer sees the button grayed out with a notice near the cart section — but on some themes, the error message is subtle and easy to miss.

FIX → Test checkout end-to-end immediately. Check your inventory levels for items currently in carts. If items are going OOS during checkout spikes, consider enabling "Continue selling when out of stock" or adding an inventory buffer.
#8

Customer Account Required but Not Logged In

If your Shopify store is configured to require customer accounts for checkout (Online Store → Preferences → Customer accounts set to "Required"), customers who are not logged in will be unable to complete checkout. Depending on your theme, this sometimes manifests as a disabled button rather than a clear login prompt — especially if the login redirect is broken.

FIX → Check Online Store → Preferences → Customer accounts. If account requirement is intentional, verify the login/register flow works correctly. If not intentional, switch to "Optional" or "Disabled."

Quick Diagnosis: Identify Your Cause in 5 Minutes

Run through these steps in order. Most causes become obvious at step 1 or 2.

01

Test in incognito on mobile

Open an incognito window on your phone. Add a product to cart, go to checkout, fill in every field carefully. Scroll all the way down before attempting to click. If the button enables in incognito: the issue is browser/session-specific (likely a conflicting browser extension). If it stays disabled: the issue affects all customers.

02

Check for a hidden T&C checkbox

On mobile, scroll down past the payment section and look for any unchecked checkbox. Terms & Conditions checkboxes are the most common cause of customers reporting "the button is greyed out and I've filled everything in." Check on both desktop and mobile — the checkbox position varies by theme.

03

Check the browser console for JS errors

Open Chrome on desktop, go to your checkout, press F12. Click the Console tab. Look for any red error lines. If you see errors from a script named after an app (e.g. a fraud app, a subscription app), that app's script is likely interfering with the button. Note the error message and source file.

04

Check Settings → Payments for the payment section

If the payment section itself is not rendering (blank payment step), the button will never enable because no payment method can be selected. Go to Settings → Payments and verify at least one gateway is active. See the No Payment Methods guide if the payment section is blank.

05

Disable checkout apps one by one

If steps 1–4 don't surface the cause, go to Settings → Checkout → Checkout extensions and disable each extension. Test after disabling each one. When the button starts working after disabling a specific extension, that extension is your cause. Contact the app developer with a bug report.

JavaScript and app-related button failures appear after changes — theme updates, app installs, checkout extension updates. Build the habit of running a full incognito checkout test immediately after any change that touches your storefront or checkout apps. Most JS-related button failures are caught within minutes of deployment if you test proactively.

Checklist for post-change testing:

  • Open incognito on desktop — add to cart — complete full checkout to confirmation page
  • Open incognito on mobile — repeat the same flow
  • Check browser console for new JS errors
  • Verify payment section renders and shows at least one selectable method
  • Confirm "Complete order" button enables after filling all fields

FAQ

Why is the Shopify checkout button greyed out?

The "Complete order" or "Place order" button in Shopify checkout is disabled by default and only enables when all required conditions are met: required fields filled, a payment method selected, and any consent checkboxes checked. If it stays greyed out even after filling everything in, the most likely causes are payment methods not loading (payment section blank), a JavaScript error preventing the enable logic from running, a Terms & Conditions checkbox not checked, or a checkout app conflict.

Why can customers see the checkout but not click "Complete order"?

Shopify's button stays disabled until all form conditions are met. If it never enables, the most likely issue is either the payment section hasn't loaded a selectable method (so paymentSelected stays false) or a JavaScript error broke the form-validation listener that would normally enable the button. Test in incognito, check the console for errors, and verify the payment section shows a selectable method.

Could a Shopify app cause the checkout button to stay greyed out?

Yes. Checkout extensibility apps can inject logic that controls the button state. A bug in a fraud detection app, subscription billing app, or upsell app can keep the button permanently disabled. If the issue appeared after installing or updating an app, disable it in Settings → Checkout → Checkout extensions and re-test.

My checkout button is greyed out even when all fields look filled in — what do I do?

First: open browser console (F12) and look for JavaScript errors on the checkout page — a JS error is the most common non-obvious cause. Second: check that the payment section is rendering and showing a selectable method (no payment loaded means button stays disabled). Third: scroll down past the payment section and check for a Terms & Conditions checkbox that requires checking. Fourth: try a completely different browser to rule out a browser extension conflict.

Will an uptime monitor alert me when the checkout button is broken?

No. Standard uptime monitors check whether a URL returns a non-error HTTP status. The checkout page returns HTTP 200 regardless of whether the button is usable. Only a tool that simulates the full checkout flow — like CheckPulse — can detect when payment methods fail to load and the button becomes permanently disabled for all customers.

How do I test if the checkout button issue is affecting all customers or just some?

Open an incognito window on a different browser and device. Add a product and complete the full checkout form. If the button enables in incognito but not in a regular session, the issue is browser/extension-specific. If it stays disabled in incognito on multiple devices, all customers are affected and you have an active revenue-blocking incident.
