HTTP Errors

HTTP Status Error Codes at Shopify Checkout

These are the HTTP status codes Shopify's checkout endpoint returns. Developers see these in network logs; standard merchants see a corresponding error page or message.

Code Name What It Means Common Causes
400 Bad Request Shopify couldn't parse the request. The data format is malformed or a required parameter is missing. Corrupted cart token, malformed checkout URL, invalid JSON in a Storefront API request body.
403 Forbidden Access to checkout is blocked. The request is valid but the server refuses to fulfill it. Store password protection active; geographic IP block; expired checkout session token; custom checkout.liquid access control on Plus plans.
404 Not Found The checkout or cart referenced in the URL no longer exists on Shopify's servers. Checkout session expired (abandoned carts expire after ~3 days); manually deleted cart; incorrect checkout token in URL.
422 Unprocessable Entity Shopify understood the request but the submitted data failed validation and cannot be processed. Empty cart; out-of-stock variant; invalid discount code; shipping address outside supported zones; required field missing; currency mismatch. Most common developer-reported checkout error.
429 Too Many Requests Your app or integration has hit Shopify's rate limit and is being throttled. Excessive Storefront API calls (burst limit: 50 req/s per app); aggressive polling of checkout status; multiple concurrent checkout mutations from the same app.
500 Internal Server Error Shopify's checkout service encountered an unexpected failure it could not handle. Payment gateway API timeout; Shopify platform incident; fatal error in third-party checkout app; deprecated checkout.liquid with unhandled exception.
503 Service Unavailable Shopify's checkout service is temporarily unavailable — either down for maintenance or under extreme load. Shopify platform incident (check status.shopify.com); extreme traffic spike on Shopify's infrastructure; store-level traffic overwhelm (rare on Shopify's multi-tenant platform).

How to diagnose HTTP errors

For standard Shopify stores (non-headless): open Chrome DevTools → Network tab → filter by "checkout" → reproduce the error and look at the status column. The response body often includes a JSON error description or an HTML error page with more detail.

For headless / Storefront API: HTTP errors are accompanied by a errors or userErrors array in the GraphQL response body. Always inspect both — a 200 response can still contain userErrors indicating a validation failure.

Storefront API / GraphQL

Storefront API Checkout Error Codes

These error codes appear in the userErrors array returned by Shopify Storefront API GraphQL mutations (checkoutCreate, checkoutLineItemsUpdate, checkoutCompleteWithTokenizedPaymentV3, etc.). They identify validation failures with precision — always inspect them before surfacing a generic message to the customer.

Inventory & Cart Errors

NOT_ENOUGH_IN_STOCK

Insufficient inventory

Requested quantity exceeds available stock. The field array points to the specific line item index. Reduce quantity or remove the item.

EMPTY

Cart is empty

A checkout mutation was attempted on a cart with no line items. Add at least one item before calling checkoutCreate.

LINE_ITEM_NOT_FOUND

Line item not found

An update or remove mutation referenced a line item ID that no longer exists in the checkout. Re-fetch the checkout to get current line item IDs.

VARIANTS_REQUIRE_SELECTING_OPTIONS

Options not fully selected

A product with multiple variants was added without fully selecting all required options (size + color, etc.). Surface option selectors before allowing add-to-cart.

ALREADY_COMPLETED

Checkout already completed

A mutation was called on a checkout that already has an order associated with it. Create a new checkout; do not reuse completed checkout tokens.

TOO_LONG

Field value too long

A string field (note, shipping address line) exceeds the maximum character limit. The field array identifies which field. Truncate or validate length client-side.

Discount & Gift Card Errors

DISCOUNT_NOT_FOUND

Discount code not found

The discount code entered does not exist in the store. Check for typos; codes are case-insensitive but must match exactly.

DISCOUNT_EXPIRED

Discount has expired

The code is valid but its end date has passed. The customer must use a different code.

DISCOUNT_DISABLED

Discount is disabled

The discount exists but has been deactivated in the merchant's admin. Reactivate it in Admin → Discounts if it should still be valid.

DISCOUNT_LIMIT_REACHED

Usage limit reached

The discount has been used the maximum number of times set by the merchant. No fix for the customer — the discount is exhausted.

CUSTOMER_ALREADY_USED_ONCE_PER_CUSTOMER_DISCOUNT_NOTICE

Already used (one-per-customer)

This code has a one-per-customer limit and the logged-in customer has already redeemed it. Display a clear "already redeemed" message.

DISCOUNT_NOT_APPLICABLE

Cart doesn't qualify

The discount has conditions (minimum order value, specific products, specific customer tags) that the current cart does not meet.

CART_DOES_NOT_MEET_DISCOUNT_REQUIREMENTS_NOTICE

Minimum requirements not met

The cart total or item count is below the discount's minimum requirement. Display the gap to the customer ("Add $X more to qualify").

UNABLE_TO_APPLY

Cannot apply discount

Generic application failure — often conflicts with another already-applied discount. Remove existing discounts before applying a new one.

GIFT_CARD_NOT_APPLICABLE

Gift card not applicable

The gift card cannot be used in this context — common when a gift card currency doesn't match the checkout currency.

GIFT_CARD_DEPLETED

Gift card balance is zero

The gift card has been fully used. The balance is $0. Customer must use a different payment method for the full amount.

GIFT_CARD_EXPIRED

Gift card has expired

The gift card's expiry date has passed. Contact the merchant — Shopify gift cards do not expire by default unless the merchant set an expiry date.

GIFT_CARD_CODE_INVALID

Gift card code invalid

The code entered doesn't match any gift card in the store. Verify the code; gift card codes are case-sensitive in some implementations.

GIFT_CARD_ALREADY_APPLIED

Gift card already applied

The same gift card has already been added to this checkout. Each gift card can only be applied once per checkout.

GIFT_CARD_CURRENCY_MISMATCH

Currency mismatch

The gift card was issued in a different currency than the current checkout. Shopify gift cards are currency-specific.

Address & Shipping Errors

INVALID_FOR_COUNTRY

Country not supported

The shipping address country is not in any of the merchant's shipping zones. The merchant must add a shipping rate for that country, or the customer must use a different address.

INVALID_FOR_COUNTRY_AND_PROVINCE

Province not supported

The country is supported but the specific province/state is excluded from shipping. Check shipping zones in Admin → Settings → Shipping and delivery.

INVALID_PROVINCE_IN_COUNTRY

Province invalid for country

The province code doesn't exist in Shopify's database for the given country. Common cause: using a non-ISO province code in a Storefront API mutation.

INVALID_REGION

Invalid region

Geographic region is not recognized. Verify the countryCode and provinceCode values against Shopify's supported regions list in the Storefront API schema.

INVALID_COUNTRY_AND_CURRENCY

Country / currency mismatch

The checkout currency doesn't match what's available for the billing or shipping country. Occurs when using multi-currency with country-specific currency rules.

Payment Errors

TOTAL_PRICE_MISMATCH

Price changed since last fetch

The total amount submitted in the payment mutation no longer matches the actual checkout total. Always re-fetch the checkout total immediately before submitting payment.

MISSING_PAYMENT_INPUT

Payment data missing

The payment mutation was called without valid payment data (card token, wallet payment token, etc.). Check that the payment gateway tokenization completed successfully before calling the mutation.

SHOPIFY_PAYMENTS_REQUIRED

Shopify Payments required

The checkout requires Shopify Payments but the store's account doesn't support it in the current country, or the plan doesn't include it. Surfaces when using certain payment methods that only work with Shopify Payments.

NOT_SUPPORTED

Feature not supported

The requested operation is not supported for this checkout configuration (plan level, store settings, or app context). Check the Storefront API changelog for feature availability by plan.

INVALID

Field value invalid

Generic validation failure — a field value doesn't pass format validation. The field array specifies which field. Common for email format, phone number format, or custom attribute values.

PRESENT

Already exists

Attempt to add something (gift card, discount) that is already applied to the checkout. Remove the existing one first or skip the add if idempotency is needed.

Customer-Facing Messages

Error Messages Your Customers See

These are the error messages Shopify surfaces to shoppers in the standard checkout UI. Customers don't see error codes — they see these plain-language messages. The right column explains what's actually happening and how to resolve it.

Customer Sees

"Your cart has been modified"

What's Happening & Fix

Inventory changed between add-to-cart and checkout. Another customer checked out the last unit, or the merchant changed stock. Customer must review cart and remove or reduce the affected item.

Customer Sees

"This item is no longer available in the quantity you requested"

What's Happening & Fix

Maps to NOT_ENOUGH_IN_STOCK. Available quantity dropped below the requested amount. Checkout forces a quantity reduction. Customer must accept the lower quantity or remove the item.

Customer Sees

"We had trouble processing your payment. Please try again."

What's Happening & Fix

Payment gateway declined the transaction or returned an error. Card details may be wrong, the issuing bank declined it, or the payment gateway is temporarily unavailable. Customer should retry with correct card details or a different payment method.

Customer Sees

"Your payment was declined"

What's Happening & Fix

The card issuer (bank) declined the charge — not Shopify. Common reasons: insufficient funds, suspected fraud, card not authorized for online purchases, or billing address mismatch. Customer should contact their bank or use a different card.

Customer Sees

"This payment method isn't available"

What's Happening & Fix

The checkout loaded with no payment methods shown. Maps to NO_PAYMENT_METHODS_AVAILABLE. Causes: Shopify Payments suspended, all payment gateways have their country restrictions applied to this customer's country, or the store is in test mode. See our no payment methods guide.

Customer Sees

"Your session has expired. Please go back to the store and try again."

What's Happening & Fix

The checkout session token has expired (typically after ~3 days of inactivity). Shopify cannot recover an expired checkout. Customer must start checkout again from the store. See our checkout session expired guide.

Customer Sees

"Sorry, this discount code isn't valid"

What's Happening & Fix

Maps to DISCOUNT_NOT_FOUND, DISCOUNT_EXPIRED, or DISCOUNT_DISABLED. Check the discount in Admin → Discounts: is it active, not expired, not at the usage limit, and applicable to the items in cart?

Customer Sees

"This discount has already been used"

What's Happening & Fix

Maps to CUSTOMER_ALREADY_USED_ONCE_PER_CUSTOMER_DISCOUNT_NOTICE. The one-per-customer limit applies and this customer has already redeemed the code once.

Customer Sees

"No shipping options available for this address"

What's Happening & Fix

No shipping rate in the merchant's shipping zones covers the customer's address. Merchant must add a shipping rate for that country/region, or the customer must use a different address. See our no shipping options guide.

Customer Sees

"Address validation failed"

What's Happening & Fix

Shopify's address validator (or a third-party validator app) rejected the address as non-deliverable. Customer must verify street, city, postcode, and country. See our address validation guide.

Customer Sees

"Checkout is not available"

What's Happening & Fix

Store is password-protected or inaccessible. Maps to HTTP 403. For development stores: only staff and collaborators can access checkout. For live stores with password protection: the customer needs the store password. See our password protected store guide.

Customer Sees

"You must agree to the terms of service to complete your order"

What's Happening & Fix

The store has "Require customers to agree to your terms of service and refund policy before purchase" enabled and the customer hasn't checked the box. A UX issue — the checkbox is often easy to miss on mobile.

Customer Sees

"There was a problem with your order. Please contact us."

What's Happening & Fix

Generic fallback error — Shopify couldn't give a specific message. Usually indicates a server-side failure (HTTP 500/503), a payment gateway error, or an unhandled exception in a checkout app. Check Admin → Orders for any partial orders, and Shopify status for incidents.

Error Detection

How Automated Monitoring Catches These Errors

Standard uptime monitors send an HTTP request to a URL and record the status code. This catches HTTP 500 and 503 errors when the server returns them on the first request — but misses the vast majority of checkout errors.

Most Shopify checkout errors are not server errors. They are application-level errors that appear during the checkout flow — at the cart step, the address step, the shipping step, or the payment step. An uptime monitor that pings the checkout URL never reaches any of these steps.

FAQ

Frequently Asked Questions

What does Shopify checkout error 422 mean?

HTTP 422 (Unprocessable Entity) means Shopify understood the request but the data failed validation. Common causes: empty cart, out-of-stock variant, invalid discount code, shipping address outside supported zones, or a required field missing. For headless stores using the Storefront API, the GraphQL userErrors array will contain a specific error code (NOT_ENOUGH_IN_STOCK, DISCOUNT_NOT_FOUND, etc.) alongside the 422 status code that pinpoints the exact failure.

What causes Shopify checkout error 500?

HTTP 500 indicates a server-side failure. Common causes: payment gateway API timeout, Shopify platform incident (check status.shopify.com), a third-party app injecting checkout code that throws an unhandled exception, or deprecated checkout.liquid with a fatal Liquid error. If status.shopify.com shows no incident, disable all checkout apps one at a time to isolate the cause.

What is Shopify error code NOT_ENOUGH_IN_STOCK?

NOT_ENOUGH_IN_STOCK appears in the Storefront API GraphQL userErrors array when a line item quantity exceeds available inventory. It includes a field array pointing to the specific line item. Standard Shopify checkout shows customers "This item is no longer available in the quantity you requested." Fix: reduce the quantity to available stock or remove the item.

Why does Shopify show "Your cart has been modified"?

This notice appears when inventory changes between add-to-cart and checkout. Another customer checked out the last unit while this customer was browsing, or the merchant changed availability. It's a cart notice, not an error — Shopify automatically adjusts the cart and asks the customer to review before continuing.

What does TOTAL_PRICE_MISMATCH mean in Shopify's Storefront API?

TOTAL_PRICE_MISMATCH is returned when a checkout payment mutation is called with a total amount that no longer matches the current checkout total. This happens when prices change between the display step and payment submission (sale ends, discount applied/removed, currency rate updates). Always re-fetch the checkout object immediately before displaying the final price and submitting the payment mutation. Never cache the total amount.

How do I find which Shopify checkout error is causing problems?

For standard stores: open Chrome DevTools (F12) → Network tab → reproduce the error → look for requests to checkout.shopify.com and inspect the response body. For headless/Storefront API: log the full userErrors array from every GraphQL mutation response — never assume a 200 response means success. For production monitoring without manual testing: use CheckPulse's automated checkout simulation, which detects errors at each step and sends alerts with the specific step that failed.
