8 Causes of Shopify Checkout Not Loading

Checkout loading failures split into two categories: platform-side (Shopify or its CDN is down) and store-side (something specific to your store is breaking checkout). Platform outages are rare and short-lived. Store-side failures are far more common and can last for hours or days without any automatic alert.

01

Shopify Platform or CDN Outage

Shopify delivers checkout pages through a global CDN (Cloudflare and their own infrastructure). A CDN edge node failure or platform incident can cause checkout to return 503 or timeout for customers in specific regions — even when your store homepage loads fine from a different edge location. This is the first thing to rule out because the fix requires no action on your part.

FIX → Check https://www.shopifystatus.com and the Shopify Status Twitter for active incidents. If an incident is declared, wait for resolution.
02

Corrupted Browser Cache or Stale Checkout Session

Shopify checkout uses a session token stored in the browser. If that token is corrupted, expired in an unusual way, or mismatched against the cart token, the checkout page can load a blank state or loop on an initializing spinner. This is the second most common cause of customer-reported checkout loading failures and is invisible to you as the store owner — your own browser works fine because your session is fresh.

FIX → Tell customers: clear browser cache + cookies for your domain, or open checkout in an incognito window. If incognito works, the issue is session/cache.
03

Browser Extension Blocking Checkout Scripts

Ad blockers, privacy extensions, VPN browser extensions, and anti-tracking tools frequently intercept Shopify checkout resources. Shopify loads several first-party and third-party scripts during checkout (payment SDKs, risk assessment, analytics). If any of these are blocked, the checkout UI either doesn't render at all (blank page) or renders partially then freezes. This is the #1 cause of customer-specific checkout failures — the customer can't checkout but you can, because you don't have the same extension.

FIX → Test in incognito (no extensions). If it loads: the issue is a browser extension on the customer's side. Add a note to your checkout instructions advising customers to disable ad blockers if they see a loading error.
04

JavaScript Error from a Checkout App or Extension

Shopify's checkout extensibility platform lets apps inject JavaScript directly into the checkout page. An uncaught JavaScript error thrown before Shopify's checkout bundle completes execution will halt page rendering entirely, leaving customers with a blank white screen. This happens after app installs, app updates, and sometimes after Shopify's own checkout bundle version changes. The error is real but invisible — checkout returns HTTP 200, so uptime monitors see no problem.

FIX → Open checkout in a browser, press F12 → Console. Any red JS error is your culprit. Identify which app or script produced the error (check the file name in the stack trace), then disable that app via Settings → Checkout → Checkout extensions.
05

Custom Theme Script Breaking Checkout Render

Theme customizations that inject scripts into checkout (via checkout.liquid in older themes, or via Script Tags API) can throw errors that prevent checkout from rendering. This is especially common after theme updates — the theme update changes a dependency or class name that a custom script relies on, causing a runtime error. Unlike app extensions, theme script errors are harder to trace because they may not show a clear error message in the console.

FIX → Temporarily switch to Shopify's default Dawn theme and test checkout. If it works on Dawn, the issue is your custom theme or a script tag added to checkout. Check the most recent theme changes and revert the last edit.
06

Third-Party Script Timeout Blocking Page Render

Chat widgets, fraud prevention services, analytics platforms, and ad pixels are often loaded synchronously or with insufficient timeout handling during checkout. If one of these services is slow or unresponsive, the checkout page waits indefinitely for that script to complete before rendering — causing an infinite spinner. The underlying Shopify checkout is fine; it's a third-party dependency that's hanging. Check the Network tab in developer tools for requests stuck in "pending" for more than 3–5 seconds.

FIX → Open F12 → Network tab → Reload checkout → Sort by "Time" — any request stuck pending for 5+ seconds is likely your culprit. Either remove the script, add a timeout, or switch to async loading.
07

Content Security Policy (CSP) Blocking Checkout Resources

If you or a Shopify app has added a custom Content-Security-Policy header to your storefront, it may block checkout-specific scripts served from Shopify's CDN subdomains. Shopify checkout loads resources from multiple cdn.shopify.com, checkoutapi.shopify.com, and payment partner domains. A CSP that doesn't include these as allowed sources will silently block them — causing a partial or blank checkout render without any obvious error on the page.

FIX → Check F12 → Console for CSP violation errors (they appear in red with "Content Security Policy" prefix). If found, update your CSP to allow Shopify's checkout CDN domains, or remove the custom CSP if it was added by an app you no longer use.
08

DNS or SSL Certificate Issue

If you use a custom domain for your Shopify store, a DNS propagation issue, expired SSL certificate, or misconfigured CNAME record can cause checkout to fail for some customers while working for others (depending on which DNS resolver they use). Shopify's checkout URL typically redirects to checkout.shopify.com — but if the initial domain resolution fails, customers never reach the redirect. Check Shopify Admin → Settings → Domains for any SSL or DNS warnings.

FIX → Shopify Admin → Settings → Domains → look for any yellow warning icons. If your SSL certificate shows "pending" or "error", follow Shopify's reconnect steps. For DNS issues, check that your CNAME points to shops.myshopify.com.

5-Step Diagnosis Checklist

Work through these in order. Each step narrows down whether the issue is platform-wide, browser-specific, or store-specific.

1

Check Shopify Status first (30 seconds)

Go to shopifystatus.com. If there is an active incident involving Checkout, nothing you do will fix it — only Shopify can. Wait for resolution before diagnosing store-side issues.

2

Test in incognito on a different device (2 minutes)

Open a fresh incognito/private window on a different device or browser (ideally a real mobile device on cellular, not WiFi). Add a product and go to checkout. If it loads: the issue is either browser-cache, browser extension, or network-specific to the original device.

3

Open browser developer tools and check Console (3 minutes)

Press F12 (or Cmd+Option+I on Mac) → Console tab. Reload the checkout page. Any red JavaScript errors are your most likely cause. Note the filename in the error stack trace — it usually identifies the offending app or script.

4

Check Network tab for pending requests (2 minutes)

F12 → Network tab → reload checkout → wait 10 seconds → filter by "Pending" or sort by Time descending. Any request still in pending for 5+ seconds is blocking checkout render. The URL of the pending request identifies the third-party service responsible.

5

Disable checkout apps and extensions one by one (5–10 minutes)

Go to Settings → Checkout → Checkout extensions. Disable apps starting with the most recently installed or updated. Re-test checkout after each disable. When checkout loads again, you have identified the offending extension.

# "ERR_BLOCKED_BY_CLIENT" = browser extension blocking the script
# "TypeError: Cannot read properties of undefined" = app JS error
# Both cause a blank or frozen checkout page

How to Prevent This from Happening Again

The most dangerous aspect of checkout loading failures is that they are completely invisible to standard monitoring. Your homepage returns 200. Your synthetic uptime check returns 200. You have no alert. The only indicator is a drop in conversion rate — and that signal arrives hours or days after the failure, after significant revenue has already been lost.

1. Automated Checkout Simulation (Catches Platform and App Failures)

A tool that runs the full checkout flow end-to-end on a schedule — creates a cart, initiates a checkout session, fetches payment methods, verifies the checkout URL is reachable — and alerts you immediately when any step fails. This is the only approach that catches checkout failures in real time, regardless of whether the cause is a Shopify outage, an app conflict, or a payment gateway failure.

2. Post-Change Checkout Tests (Catches App and Theme Failures)

The vast majority of store-side checkout loading failures appear immediately after a change: an app install, a theme update, a Script Tag addition. Building the habit of running a full checkout test in incognito immediately after any storefront change catches these failures in seconds, before they affect real customers.

Post-change checkout checklist:

  • Open incognito on desktop — add product to cart — load checkout — confirm page renders
  • Open F12 → Console — confirm no new JavaScript errors on the checkout page
  • Open F12 → Network — confirm no requests pending >3 seconds
  • Repeat on a real mobile device (not just desktop emulation)
  • Complete a test order through to the order confirmation page

3. Stay Subscribed to Shopify's Status Page

Shopify publishes real-time status at shopifystatus.com. Subscribe to email or RSS alerts so you are notified the moment Shopify declares a platform incident. This separates platform outages (nothing you can do) from store-specific failures (something you must fix) in under 60 seconds.

FAQ

Why is my Shopify checkout page not loading?

Shopify checkout fails to load for several distinct reasons: a Shopify platform or CDN outage (check shopifystatus.com), a corrupted browser session or stale checkout cache, a JavaScript error in a checkout app or theme script, a third-party script hanging and blocking page render, or a browser extension intercepting checkout resources. The fastest first check: open checkout in a fresh incognito window. If it loads there but not in a regular browser, the issue is browser-side (cache, extension, or stale session).

Why does Shopify checkout show a blank white page?

A blank white checkout page means the HTML was delivered but the JavaScript that renders the UI crashed before it could run. Open F12 → Console and look for red JavaScript errors — there is almost always a visible error. The filename in the error stack trace usually points to the offending app or script. Disable checkout apps one by one (Settings → Checkout → Checkout extensions) until the page loads again.

Is Shopify checkout down right now?

Check shopifystatus.com in real time. If an active incident is listed for Checkout, it's a platform issue — wait for Shopify's resolution. If the status page shows all-green but your checkout still doesn't load, the failure is specific to your store (app conflict, theme script, DNS issue). The status page only reflects platform-wide failures, not store-specific problems.

Why does Shopify checkout keep spinning / never finish loading?

An infinite spinner usually means a third-party script is hanging. Open F12 → Network tab → reload checkout → wait 10 seconds → sort by Time. Look for requests in "pending" state for 5+ seconds. The URL of the hanging request identifies the service. Common culprits: chat widgets, fraud prevention services, subscription app SDKs. Either remove the script from checkout or ensure it loads asynchronously with a timeout.

Can a browser extension cause Shopify checkout not to load?

Yes — this is the most common cause of customer-specific loading failures. Ad blockers and privacy extensions frequently block Shopify checkout scripts or payment SDK resources. Test in incognito (which disables extensions by default). If it loads there, the customer's extension is the cause. Add a note to your checkout error messaging advising customers to try incognito if they see a loading issue.

Will an uptime monitor tell me when checkout is not loading?

No. Standard uptime monitors check whether a URL returns HTTP 200. Shopify checkout returns 200 even when the page is blank or stuck — because the server delivered the HTML correctly. The rendering failure is client-side when JS executes. Only a tool that runs the full checkout simulation end-to-end, like CheckPulse, can reliably detect these failures. CheckPulse verifies that checkout creates a valid session and returns accessible payment methods, not just that the URL is reachable.
