Shopify Checkout Phone Number Troubleshooting

Shopify Checkout Rejects Valid Phone Numbers — Format Errors, Country Code Conflicts, E.164 Validation

Shopify checkout validates phone numbers differently depending on your market settings, active payment methods, and whether checkout extensions are installed. A number that is perfectly valid in one context gets rejected in another. The phone field can be Required, Optional, or Hidden per market — a wrong setting blocks every customer who doesn't have a phone. Shop Pay auto-fill inserts the saved account number with the original country code regardless of where the customer is checking out. And SMS order notifications can fail silently even when checkout completes successfully. This guide covers every way the phone number field breaks, and the exact paths to fix each one.

Updated June 2026 Affects: international stores, Markets users, Shop Pay, SMS notifications, checkout extensions
Shopify does not enforce a single global phone number format. Validation depends on: market settings (Required vs Optional), active payment method (Shop Pay enforces E.164 more strictly), checkout extensions installed, and whether the customer's phone is used for SMS notifications. A number that passes checkout can still fail SMS dispatch.
Root Causes

Five Reasons Shopify Checkout Rejects a Phone Number

Phone number rejection in Shopify checkout comes from five distinct sources. The error message is usually the same — "Enter a valid phone number" or "Phone number is invalid" — but the fix path is completely different depending on which of these is triggering it.

Cause 1 — E.164 Format
Missing or Wrong Country Code
International markets expect E.164 format: +[country code][subscriber number] with no spaces, dashes, or parentheses. A US customer entering (555) 123-4567 passes domestic checkout but fails for stores using Shopify Markets with international settings or Shop Pay as primary payment method.
Cause 2 — Required Field
Phone Field Set to Required
The phone field is configurable per market as Required, Optional, or Hidden. If set to Required and a customer skips it or Shop Pay auto-fill fails to populate it, checkout is blocked at the contact step with no clear explanation to the customer.
Cause 3 — Double Prefix
Country Code Entered Twice
When Shopify displays a country code flag selector next to the phone field, the dialing code is prepended automatically. If the customer also types the country code (e.g., types "1" when the +1 flag is already selected), the result is +11XXXXXXXXXX — an invalid E.164 string that Shopify's validator rejects.
Cause 4 — Shop Pay Auto-fill
Wrong Country Code Auto-filled
Shop Pay stores the phone number from the customer's account profile. If the account was created in a different country, Shop Pay auto-fills the phone field with the original country code regardless of where the checkout is happening — triggering format validation failure in the current market.
Cause 5 — App Conflict
Checkout Extension Overriding Validation
Third-party checkout apps (Rebuy, Bold, Zipify, custom Checkout UI Extensions) can attach custom phone validation on top of Shopify's native validation. The app's regex may be stricter or region-specific, rejecting numbers that Shopify itself accepts. The customer sees the app's error message, not Shopify's.

Deep Dive A

E.164 Format — What It Is and When Shopify Enforces It

E.164 is the international phone numbering format defined by the ITU. It consists of a + sign, followed by a country calling code (1–3 digits), followed by the subscriber number (up to 15 digits total), with no spaces, dashes, or parentheses.

Deep Dive A

What E.164 Looks Like vs. Local Format

A US phone number in E.164 is +15551234567. The same number in common local formats — (555) 123-4567, 555-123-4567, 555.123.4567 — all become +15551234567 in E.164. A UK number in E.164 is +447700900123; the same number in UK local format is 07700 900123 (note the leading 0 which is dropped in E.164 and replaced by the +44 country code).

E.164: +1 555 123 4567 → +15551234567 ✓ E.164: +44 7700 900123 → +447700900123 ✓ Local: (555) 123-4567 → rejected in international markets ✗ Local: 07700 900123 → rejected; leading 0 is UK local format ✗ Double: +1 1-555-1234567 → +115551234567 ✗ (double country code)

Shopify enforces E.164 in three specific contexts: (1) when the checkout market has international payment methods active, (2) when Shop Pay is the active payment method — Shop Pay's backend validates against E.164 strictly for account linking, (3) when a Checkout UI Extension or app has added custom phone validation that calls Shopify's libphonenumber-compatible validator. In domestic-only checkout with no extensions and no Shop Pay, Shopify often accepts local formats — this is why the same number works on one store but not another.

To test whether your store enforces E.164: go through checkout as a customer and enter a local-format number. If it passes, your store is in lenient mode. If it fails with "Enter a valid phone number," E.164 is being enforced — either by market settings, Shop Pay, or an extension.

Deep Dive A — Fix Path

How to Configure Phone Field Per Market

Admin → Settings → Markets → [select market] → Checkout and accounts → Phone number in checkout. The three options are: Required (customer cannot complete checkout without a valid phone), Optional (customer can skip the phone field entirely — recommended for most stores), Hidden (phone field does not appear — use only if you have no post-purchase SMS or WhatsApp workflow).

For international stores: set phone to Optional in all international markets unless you have a specific, mandatory reason to require it. Requiring phone creates the highest checkout abandonment in markets where customers are unaccustomed to sharing phone at checkout (Germany, Netherlands, France, Japan). Optional means customers who want SMS updates will enter their number; customers who don't can proceed normally.

Admin → Settings → Markets → Europe (or individual market) → Checkout and accounts → Phone number in checkout → Optional ← recommended for international

Deep Dive C

Checkout Extension Phone Validation Conflicts

Third-party apps that touch the checkout (Rebuy, Bold Checkout, Zipify Pages, custom Checkout UI Extensions, and post-purchase apps) can add JavaScript-side phone validation on top of Shopify's native validation. These layers can conflict, producing errors that Shopify's own validator would never show.

Deep Dive C — Diagnosis

Identifying Whether an Extension Is Causing the Phone Error

The fastest diagnostic: open checkout in a browser with all extensions disabled (Chrome → Settings → More tools → Extensions → disable all; or use a clean private window on a different browser). Go through checkout and enter the same phone number that was failing. If it now passes, a browser extension is not the issue — a Shopify checkout app is. If it still fails, Shopify's native validation is the source.

To identify which checkout app: Admin → Apps → look for any app with "checkout," "upsell," "extension," or "post-purchase" in the name. Temporarily disable each one and retest checkout with a phone number. The one whose removal fixes the error is the conflicting app.

Diagnostic steps: 1. Test checkout in clean browser → phone fails → native Shopify validation 2. Test checkout in clean browser → phone passes → extension/app conflict 3. Apps → disable checkout apps one by one → retest after each 4. Identify conflicting app → contact developer for regex fix or disable phone validation

Common app-side phone validation mistakes: regex patterns that don't account for E.164 format (they expect local format only), patterns that require exactly 10 digits (fails for countries with 11–12 digit numbers), patterns that reject the + prefix (treating it as an invalid character instead of a country code indicator).

Deep Dive C — Fix

Shopify Plus: Enforce E.164 at Input via Checkout Editor

Shopify Plus merchants can customize the checkout phone field directly in the Checkout Editor without a custom extension. The editor allows: changing the field label, adding placeholder text (e.g., "Include country code: +1 555 123 4567"), and marking the field as required or optional per market. This is the lowest-friction way to guide customers to enter correctly formatted numbers.

For real-time E.164 format enforcement (showing an error as the customer types, not after they try to submit), a Checkout UI Extension is required. The extension can use the libphonenumber-js library to validate the entered number against the detected country code in real time, and display a country-specific format hint. This requires Shopify Plus and a deployed extension — not possible with Checkout Editor alone.

Shopify Plus Checkout Editor: Admin → Sales channels → Online Store → Customize → Checkout → Contact section → Phone field → Edit label: "Phone number (with country code)" → Edit placeholder: "+1 555 123 4567" → Required: Optional (recommended)
Diagnosis Checklist

Phone Number Fix Checklist — Start Here

  • 1
    Check per-market phone field setting Admin → Settings → Markets → [market] → Checkout and accounts → Phone number in checkout. If set to Required and causing abandonment, change to Optional. This is the fastest fix for "phone blocking checkout" reports.
  • 2
    Check for double-prefix issue Does your checkout show a flag/country code selector next to the phone field? If yes, instruct customers to enter only the subscriber number — not the country code — in the field. Customize the placeholder text to make this clear (Checkout Editor for Plus, Checkout UI Extension for non-Plus).
  • 3
    Test with Shop Pay disabled If the error appears specifically when Shop Pay is the selected payment method, the issue is Shop Pay's stricter E.164 enforcement or auto-fill conflict. Test the same phone number with a different payment method (credit card) — if it passes, the issue is Shop Pay-specific.
  • 4
    Audit checkout extensions for custom phone validation Admin → Apps → identify any apps touching the checkout. Disable them one by one and retest. If the phone error disappears after disabling an app, contact that app's developer to fix their phone validation regex or configure it to accept E.164 format.
  • 5
    Verify SMS notification phone normalization If checkout passes but SMS notifications fail: check stored phone format in recent orders (Admin → Orders → customer details). If numbers are stored in local format (no country code), configure your SMS app to normalize to E.164 before dispatch, or enforce E.164 at checkout input.
  • 6
    For international stores: audit per-country format expectations Check the country table above for the specific format customers expect to enter vs. what Shopify requires. Germany, UK, Australia, Japan, and France all have leading-0 local formats that conflict with E.164. Consider adding country-specific format hints in your checkout or post-purchase instructions.
FAQ

Shopify Phone Number Validation — Frequently Asked Questions

Why does Shopify checkout say my phone number is invalid even though it's correct?
Shopify validates phone numbers against E.164 international format for certain markets and payment methods. E.164 requires a country code prefix (e.g., +1 for US/Canada, +44 for UK) followed by the subscriber number with no spaces, dashes, or parentheses. If you're in an international market or using Shop Pay, enter your number with the country code: +1 555 123 4567 instead of (555) 123-4567. If your number passes domestic checkout but not international — this is why.
Why is the phone number field required and blocking my customers from checking out?
The phone field Required/Optional/Hidden setting is configurable per market. To check: Admin → Settings → Markets → select the relevant market → Checkout and accounts → Phone number in checkout → change from Required to Optional. Some checkout extensibility apps also set the phone field to required — audit installed apps that touch checkout and disable their phone validation if it conflicts.
Why does Shop Pay insert the wrong country code in the phone field?
Shop Pay stores the customer's phone from their account profile, created in their home country. If they're checking out from a store with different market settings, the auto-filled number may have the wrong country prefix. Merchants can't control Shop Pay auto-fill — the mitigation is setting phone to Optional so auto-fill failures don't block checkout. Customers must manually correct the pre-filled number.
Why are SMS order notifications not being sent after checkout?
A phone number can pass Shopify's checkout validation but fail SMS dispatch if it's in local format (no country code). SMS providers (Klaviyo, Postscript, Attentive) require E.164 for dispatch. Configure your SMS app to normalize phone numbers to E.164 before sending — most major SMS apps have a built-in normalization setting. Alternatively, enforce E.164 at checkout input using a Checkout UI Extension.
Why does Shopify reject my phone number when I add the country code?
This is the double-prefix problem. Shopify's flag selector prepends the dialing code automatically. If you also type the country code (e.g., the flag shows +1 and you type 1-555-123-4567), the result is +11-555-123-4567 — invalid E.164. Always enter only the subscriber number in the field when a flag selector is visible. For clarity, add placeholder text: "Enter number without country code."
