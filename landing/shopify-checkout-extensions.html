Shopify Checkout Guide

Shopify Checkout Extensions & Extensibility

What you can customize on each plan, the three pillars of Checkout Extensibility, and why extensions are the top cause of silent checkout failures.

Updated June 2026 13 min read All Shopify plans
The Misconception

Most Merchants Misunderstand What Requires Plus

Ask most Shopify merchants what checkout customization requires Shopify Plus and you get one of two wrong answers. Half say "everything" — they believe the checkout is completely locked unless they upgrade to Plus. The other half say "nothing" — they installed a few apps and assume they have full checkout control.

Both are wrong. Checkout Extensibility has three distinct components with completely different access models. Understanding which pillar you need — and whether your use case actually requires Plus — is the decision that determines whether you spend an extra $2,000/month on Plus or spend that on ads.

What Actually Requires Plus

Building your own extensions from scratch. If you want to write custom React code that deploys into the checkout as a UI Extension, you need Shopify Plus. App partners who publish extensions to the app store need Plus developer access — but you as a merchant installing their app do not.

checkout.liquid access. Shopify Plus merchants who built on checkout.liquid (the legacy HTML/CSS/JS system) retained that access for existing stores, but it is frozen — no new features, no new Plus stores can use it. Migration to Checkout Extensibility is required.

Everything else merchants commonly want — upsell widgets, loyalty point displays, custom delivery notes, shipping method filtering, conditional payment options, post-purchase offers — is available through app-built extensions on all plans.

Pillar 1

Checkout UI Extensions — Visible Components in the Checkout Page

Checkout UI Extensions are React-based components that render inside the checkout page at defined positions. They can display banners, form fields, loyalty points, offer widgets, compliance notices, subscription add-ons, gift message fields, delivery instructions, and any other UI that makes sense at checkout.

Where extensions render

Extension points are specific named positions in the checkout layout where approved components can mount:

  • purchase.checkout.contact.render-after — after the email/phone field
  • purchase.checkout.shipping-option-list.render-after — below shipping method selection
  • purchase.checkout.payment-method-list.render-after — below payment method selection
  • purchase.checkout.order-summary.block.render — in the order summary panel
  • purchase.checkout.reductions.render-after — below discount code entry
  • purchase.checkout.actions.render-before — above the "Pay now" button

Extensions render in a sandboxed iframe-like environment. They cannot access the parent checkout page's DOM, cannot inject arbitrary CSS, and cannot fire uncontrolled network requests. This sandbox is the reason most extension failures are contained — a broken extension in its slot doesn't crash the rest of the checkout.

What app-based extensions deliver (all plans)

The following are all available through app partners without Shopify Plus:

  • Post-purchase upsell widgets (ReConvert, AfterSell, Rebuy, Zipify)
  • Loyalty point display and redemption at checkout (Smile.io, LoyaltyLion, Yotpo)
  • Custom checkout fields — delivery instructions, gift messages, PO numbers (various apps)
  • Subscription add-ons at checkout (Recharge, Seal Subscriptions, Skio)
  • Custom trust badge banners and checkout messaging
  • Free gift selection or threshold trackers (Rebuy, Gift Box)
  • Age verification or compliance notices

What requires building your own extension (Plus only)

If you have a highly specific use case that no app partner covers — a proprietary loyalty engine, a bespoke configuration tool, deep integration with your own backend systems — you need Shopify Plus to develop and deploy custom extension code. You also need a Shopify Partner account to build and test in a development environment before deploying to your production store.

Pillar 2

Shopify Functions — Server-Side Logic That Changes Checkout Behavior

Shopify Functions are backend programs that run inside Shopify's infrastructure to customize checkout computations. Unlike UI Extensions, they don't change what the page looks like — they change what the page calculates. Functions run in a WebAssembly sandbox with strict execution time limits (typically under 10 milliseconds wall clock).

What Functions can customize

  • Discount logic — custom automatic discount rules beyond Shopify's native discount types: tiered pricing, volume discounts, BOGO, bundle pricing, loyalty point discounts, B2B contract pricing
  • Shipping rate customization — hide, rename, sort, or filter shipping methods based on cart contents, customer tags, order value, delivery address, or product types
  • Payment method customization — conditionally hide or rename payment options (e.g., hide "Cash on Delivery" for international orders; rename "Credit card" to your brand name; block specific payment methods for specific products)
  • Order routing and fulfillment — custom logic for fulfillment location selection and split shipments (requires apps built on Functions for fulfillment)
  • Cart transforms — bundle line items, split products into components, or merge items for display and pricing purposes
Functions vs Script Editor

Script Editor was the old Plus-only way to run custom checkout logic. Shopify Functions replaced Script Editor and made most of the same capabilities available to all plans through app implementations. Existing Script Editor scripts on Plus continue to run, but Shopify has deprecated the tool — new checkout logic should use Functions.

The key difference: Script Editor ran arbitrary Ruby code on Shopify's servers. Functions run compiled WebAssembly with strict memory and time limits. Functions are faster, more reliable, and sandboxed — a crashing Function fails gracefully (the feature it provides doesn't apply; the checkout proceeds normally).

Apps that use Shopify Functions (all plans)

  • Discounts — Automatic Discounts, Bold Discounts, BOGO+, Rebuy (smart coupons)
  • Shipping — Shipping Rates Calculator, Parcelify, Advanced Shipping Rules
  • Payment customization — Payment Rules, various conditional payment apps
  • B2B pricing — Wholesale Gorilla, B2B Ninja (for non-Plus stores needing basic wholesale)
  • Bundles — Shopify Bundles (native, free), Bundle Builder apps
Pillar 3

Post-Purchase Extensions — Upsells After Payment, All Plans

Post-purchase extensions are the extension point that appears between the payment confirmation and the final order confirmation page. The customer has already paid. The extension shows an offer — a product add-on, an upgrade, a subscription — that the customer can accept with one tap without re-entering payment details.

Why this matters

Post-purchase is the safest stage for upsell experimentation because payment is already captured. A failed post-purchase extension means the customer doesn't see the offer — the original order is unaffected. This is fundamentally different from in-checkout extensions, where failures can block or degrade the payment flow.

Post-purchase extensions are available on every Shopify plan. They do not require Shopify Plus. The leading post-purchase apps — ReConvert, AfterSell, Zipify One Click Upsell — all support this extension point on standard plans.

Order Status Page

The order confirmation / order status page is customizable on all plans via the order status page script editor in Shopify admin. This is separate from post-purchase extensions — it's the page that shows after the post-purchase offer. Common uses: custom HTML/JavaScript for pixels and tracking, loyalty program enrollment, referral program invitations, and custom thank-you messaging.

Legacy System

checkout.liquid — Frozen, Not Removed

checkout.liquid was the HTML/CSS/JavaScript template file that Plus merchants used to fully control the checkout page layout. It is now frozen — Shopify stopped accepting new checkout.liquid customizations for Plus stores onboarded after June 2024, and announced migration requirements for existing checkout.liquid stores.

What frozen means

  • Existing checkout.liquid code on qualifying Plus stores continues to run
  • Shopify no longer adds new features to checkout.liquid — performance improvements, new checkout capabilities (accelerated checkouts, Shop Pay updates, BNPL improvements) are Extensibility-only
  • New Plus stores cannot use checkout.liquid at all — Checkout Extensibility is required
  • Shopify has signaled a long-term sunset — stores that rely on checkout.liquid will need to migrate to maintain checkout functionality as Shopify evolves

Migration from checkout.liquid

Most checkout.liquid customizations have Extensibility equivalents. The migration assessment typically involves: identifying which customizations exist in checkout.liquid; mapping each to an Extensibility equivalent (UI Extension via app, custom UI Extension development, or Function); and building or installing the replacement. Shopify provides migration documentation, and most major app partners have already published Extensibility versions of their checkout features.

Migration Risk

checkout.liquid stores often have undocumented customizations — old code written by previous agencies that no one remembers. Before migrating to Extensibility, audit every customization in your checkout.liquid file against what it does at checkout today. Running a checkout health monitor before, during, and after migration catches regressions that manual testing misses.

Failure Modes

Why Checkout Extensions Fail Silently

Custom extensions are the fastest-growing cause of checkout failures in 2025–2026. As more merchants install Checkout UI Extension apps, the exposure to extension-related failures increases. The specific failure modes differ from traditional checkout errors — they are harder to see, harder to attribute, and often persist for hours before anyone notices.

1
Silent app update
An app publisher pushes an extension update at 2am. Your checkout now runs the new code with no notification. If the update introduced a rendering bug or an API call timeout, your checkout is degraded before you wake up. No Shopify admin alert. No email from the app. Just fewer conversions.
2
Extension conflict
Two extensions installed from different apps try to render in the same extension point or make conflicting API calls. The result varies: one extension doesn't load, the section renders blank, or both extensions partially render in a broken layout. Neither app's support team knows the other app exists.
3
Function timeout
A Shopify Function that filters shipping rates or applies discounts exceeds its execution time limit. Shopify fails the Function gracefully — the checkout proceeds, but the shipping filter or discount doesn't apply. Customers see full shipping rates or no automatic discount. No error is shown. Merchants notice only when looking at orders.
4
Performance degradation
Multiple extensions, each individually fast, combine to add 1–2 seconds to the payment section load time on mobile. No single app is obviously the problem. Mobile conversion drops gradually. Without a before-and-after baseline, it's nearly impossible to attribute the drop to extension overhead without instrumentation.
5
Extension renders for some customers, not others
An extension renders correctly in Chrome on desktop, fails silently in Safari on iPhone. The developer tested on Chrome. The failure affects mobile users — your highest-abandonment segment. Extension sandbox failures on specific browser versions or device types are a known challenge and require real cross-device testing to detect.
The Detection Problem

Most of these failures are invisible in Shopify analytics. The checkout reports "completed" or "abandoned" — it doesn't tell you whether the payment section loaded correctly, whether the shipping filter applied, or whether the loyalty widget rendered. You need real checkout tests — not synthetically, but as a real customer on a real device — to detect extension failures. CheckPulse runs these tests continuously so failures surface in minutes, not after your next customer complaint.

Monitoring

Monitoring Checkout Extensions After Installation

Every Checkout UI Extension app installation increases your checkout's exposure to silent failures. This is not a reason to avoid extensions — the conversion lifts are real. But it makes monitoring non-optional.

After installing any extension app

  • Run a real checkout test immediately — place a test order from end to end, on mobile Safari and desktop Chrome
  • Verify the extension renders correctly, the payment button is responsive, and checkout completes
  • Check load time with and without the extension installed (use browser DevTools Network tab during checkout)
  • Set a baseline conversion rate — watch for drops over the following 3–7 days

After any app update

  • App publishers don't notify merchants of extension updates — subscribe to app changelogs where available
  • If your checkout monitoring alerts on a degradation, cross-reference the timing with recent app update activity in your Shopify admin Apps log
  • Test in an incognito/private window on mobile to bypass browser caches

FAQ

Shopify Checkout Extensions — Common Questions

Do I need Shopify Plus to customize the checkout page?

It depends on what "customize" means. You do not need Plus to install app-built Checkout UI Extensions (widgets, loyalty points, custom fields, upsell banners), use Shopify Functions via app partners (discount logic, shipping filtering, payment customization), or add post-purchase upsell pages. You do need Plus to write your own custom UI Extension code rather than installing a partner app, and to access checkout.liquid (which is frozen anyway). For the vast majority of merchants, app-based extensions cover every common customization need without Plus.

What are Shopify Functions and what can they do at checkout?

Shopify Functions are server-side programs that run inside Shopify's infrastructure to customize how checkout computes prices, discounts, shipping options, and payment methods. They run in a WebAssembly sandbox with under-10ms execution limits. Functions can: apply custom discount logic (tiered, volume, bundle, loyalty), filter or rename shipping methods based on cart conditions, conditionally show or hide payment options, and customize order routing. Apps implement Functions — you install and configure the app. Functions are available on all plans via app partners, not Plus-gated.

What is the difference between checkout.liquid and Checkout Extensibility?

checkout.liquid was the legacy Plus system for modifying checkout HTML/CSS/JavaScript. It is frozen — Shopify stopped adding features to it and new Plus stores cannot use it. Checkout Extensibility is the replacement: a sandboxed React component system (UI Extensions), WebAssembly server-side logic (Functions), and post-purchase pages. Extensibility is safer (sandboxed, no arbitrary JS injection), enables automatic Shopify platform upgrades, and makes most customizations available to all plans via app partners. Stores on checkout.liquid should assess and plan migration — the longer they wait, the more Shopify platform improvements they miss.

Why do checkout extensions cause checkout to break or slow down?

Extensions add code that runs during the checkout flow. Silent failures include: app publishers pushing extension updates that introduce bugs without notifying merchants; two extensions conflicting at the same render position; Functions timing out silently and dropping discount or shipping logic; cumulative load time from multiple extensions on mobile connections; and browser-specific rendering failures that only affect Safari on iPhone but not desktop Chrome. Most of these failures produce no visible error — the checkout proceeds or the feature simply doesn't appear. Continuous checkout monitoring detects them before they accumulate into measurable revenue loss.

What checkout customizations are available without Shopify Plus?

More than most merchants know: all app-based Checkout UI Extensions (upsell widgets, loyalty point display, custom checkout fields, trust banners, subscription add-ons); all Shopify Functions via apps (discount customization, shipping filtering, payment method customization); post-purchase extension pages (one-click upsells after payment); order status page customization (via scripts in admin); and Shopify Markets for international checkout currency and language routing. What requires Plus: writing your own custom UI Extension code, checkout.liquid (frozen), Script Editor (deprecated), and B2B checkout features like net terms and purchase order workflows at the checkout level.
