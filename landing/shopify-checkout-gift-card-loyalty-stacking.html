Shopify Gift Card and Loyalty Points at Checkout — Why Stacking Breaks

Gift cards and loyalty points look identical to customers — both reduce the amount owed. But inside Shopify's checkout, they live in completely different layers. One is a payment instrument. The other is a discount. When you add a promotional code to the mix, the discount code limit silently drops one of them — and the customer gets no explanation.

The silent failure pattern: Customer has Smile.io points redeemed (discount code applied automatically) + enters a promotional code they got in an email + applies a gift card. Three things compete. Shopify's discount code limit on standard plans drops one. The customer sees the gift card applied but the loyalty points have vanished. They contact support asking why the gift card "broke" their rewards — but the gift card did nothing. The promotional code is the conflict. Shopify reports the checkout as 100% healthy. No error is shown.
The Root Cause

Gift Cards and Loyalty Points Are Different Checkout Instruments

Every stacking conflict at Shopify checkout stems from one misunderstanding: gift cards and loyalty points are not the same type of thing. They are processed in different steps, by different Shopify systems, and they interact with the discount code limit in completely different ways.

Payment Instrument
Gift Cards
  • Applied at the payment step, not the discount step
  • Do NOT count against the discount code limit
  • Processed by Shopify.checkout.gift_cards
  • Stored in Admin → Products → Gift cards
  • Third-party gift card apps (Rise.ai, GiftUp, Govalo) may override native behavior
  • Currency must match the store's checkout currency
Discount Instrument
Loyalty Points
  • Applied at the discount step via one of two methods
  • Method A: Shopify discount code (generated per redemption)
  • Method B: Checkout Extension widget (renders in checkout UI)
  • Method A codes count against the discount code limit
  • Method B extensions do not count against the discount limit
  • Used by Smile.io, LoyaltyLion, Yotpo, Bon Loyalty, Rise.ai loyalty

The key insight: gift cards can always coexist with loyalty points — unless loyalty uses discount codes AND the store has already hit the discount code limit from a promotional code. The gift card is not the culprit. The conflicting promotional code is.

Combination Reference

What Can Be Stacked at Shopify Checkout

Not all combinations fail. The outcome depends on your Shopify plan, your loyalty app's architecture, and how many discount codes are already active. Use this matrix to identify which combination your customer is attempting and whether it should work.

Combination Standard Plan Shopify Plus Root Cause When Failing
Gift card only Works Works Gift card balance, wrong store, deactivated card
Loyalty (extension) only Works Works Extension render failure, session expiry
Loyalty (discount code) only Works Works Code generation failure, code expiry
Gift card + loyalty (extension) Works Works Extension render failure on mobile/Safari
Gift card + loyalty (discount code) Works Works Gift card currency mismatch; code expiry
Promo code + loyalty (discount code) Fails Depends Discount code limit (1 on standard). Plus: needs Combine enabled
Gift card + promo code + loyalty (discount code) Fails Depends Two discount codes on standard plan. Plus: Combine setting must allow it
Bundle discount + loyalty (discount code) Fails Depends Bundle app discount code + loyalty discount code = two codes

Shopify Plus "Depends" means stacking works only when Admin → Discounts → [code] → Combinations has "Can be combined with other discounts" enabled. This setting must be configured per discount code — it is not global.

Failure Analysis

5 Stacking Failure Modes at Shopify Checkout

Failure Mode 1
Discount Code Limit Silently Drops Loyalty Points

A customer has loyalty points redeemed. The loyalty app (Smile.io, LoyaltyLion, Bon Loyalty) applied a Shopify discount code to the cart automatically — silently, without the customer manually entering anything. The customer then types a promotional discount code from an email. Shopify's standard plan allows one discount code per checkout. The promotional code was entered last and takes precedence — or Shopify replaces the promotional code with the loyalty code, confusing the customer further. Either way, one is dropped.

Silent aspect: The customer was never told the loyalty code existed. They only see it's gone after the promotional code is applied. They contact support saying "the gift card broke my points" — but there was no gift card involved. The promotional code was the conflict.

Fix: Upgrade to Shopify Plus and enable discount combining on each relevant code. On standard plans, design the loyalty UX to prevent simultaneous promotional code entry — show a warning in the cart if a promo code is active when points are redeemed.
Failure Mode 2
Third-Party Gift Card App Conflicts with Loyalty Discount Code

Some merchants use third-party gift card apps (Rise.ai, GiftUp, Govalo) instead of Shopify's native gift card system. These apps sometimes implement gift cards as discount codes — not as payment instruments — because it gives them more flexibility. When they do, the "gift card" is actually a Shopify discount code.

Now the customer has three discount codes: the third-party gift card code, the loyalty redemption code, and a promotional code. Even on Shopify Plus with combining enabled, the specific combination setting on each code may not allow all three to coexist.

Fix: Check whether your gift card app uses Shopify's native gift card system (Admin → Products → Gift cards) or implements gift cards as discount codes. Contact the app developer and ask directly. If it uses discount codes, treat it as a discount when diagnosing stacking conflicts.
Failure Mode 3
Cart-to-Checkout Session Clears Loyalty Code

The customer redeems loyalty points in the cart. The loyalty app applies a discount code. The customer adds a gift card, which triggers a cart recalculation. During the recalculation, the cart session is refreshed — and the loyalty app's discount code is not re-applied in the new session. The gift card survives (it's a payment instrument). The loyalty code does not.

Why this happens: The loyalty app listens for cart events to know when to apply or re-apply its discount code. If the gift card addition event fires before the loyalty app's re-apply listener, the code is gone. This is a timing and event ordering issue in the cart JavaScript layer.

Fix: Reproduce by: (1) redeem loyalty points, (2) wait 30 seconds, (3) apply gift card — observe whether the loyalty code survives. If it does not, report the exact reproduction steps to your loyalty app support. This is a known edge case most loyalty apps have patched in recent versions.
Failure Mode 4
Gift Card Currency Mismatch Makes Points Apply Incorrectly

A store uses Shopify Markets or a currency app to offer checkout in multiple currencies. A customer with a gift card issued in USD tries to redeem loyalty points (as a discount code, also generated in the store's base currency) while checking out in EUR. The gift card conversion may apply an unexpected amount. The loyalty discount, also in a different denomination, may apply incorrectly or vanish after the gift card triggers a currency recalculation.

Fix: Test the full combination in each currency your store supports. Shopify gift cards are issued in the store's primary currency — the conversion happens at checkout. Loyalty discount codes are also typically in the base currency. If both need conversion at checkout, test that the combined order total after both are applied is correct.
Failure Mode 5
Bundle Discount + Loyalty Code = Silent Drop on Standard Plans

Bundle apps that use discount code architecture apply a Shopify discount code to represent bundle savings (e.g., "10% off when buying items A + B together"). If the customer also has loyalty points redeemed (another discount code), the standard plan's one-discount limit silently drops one of the two. Neither the bundle discount nor the loyalty points shows an error. One just disappears.

This is the three-way version: bundle discount code + loyalty discount code + promotional code = three codes competing for one slot. Gift cards are not involved. The customer blames the most visible recent action — usually the gift card entry, even if they are not using one.

Fix: If your store uses both a bundle app and a loyalty app, verify which architecture each uses (see the bundle guide for how to identify bundle app architecture). If both use discount codes, upgrading to Plus and enabling combining is the only solution that fully resolves the conflict.
Diagnosis Checklist

7 Steps to Diagnose a Gift Card + Loyalty Stacking Failure

1
Identify your loyalty app's architecture. Log in to your loyalty app admin and search for "integration method" or "how points are redeemed at checkout." If it shows "discount code," your loyalty app creates a Shopify discount code per redemption — and it counts against Shopify's discount code limit. If it shows "checkout extension" or "checkout widget," it does not count against the limit.
2
Identify whether your gift card app is native or third-party. Go to Admin → Products → Gift cards. If your gift cards appear here with a "Remaining balance," they use Shopify's native gift card system — which is a payment instrument, not a discount code. If this page is empty and you use a third-party app, that app may implement gift cards as discount codes — which counts against the limit.
3
Check your Shopify plan's discount code limit. Go to Admin → Settings → Plan. Standard plans (Basic, Shopify, Advanced) allow one discount code per checkout. Shopify Plus allows multiple — but only if the Combine setting is enabled per code in Admin → Discounts → [code] → Combinations. If you are on a standard plan, any combination of two discount codes will fail — one will always be dropped.
4
Reproduce the exact failure scenario. Open an incognito window. Log in as a test customer with redeemable loyalty points. Add items to cart. Redeem loyalty points. Apply the gift card. Apply the promotional code (if relevant). Observe which elements survive and in what order. The sequence matters — each step may clear what was applied before.
5
Remove the promotional code and re-test. If removing the promotional code allows both gift card and loyalty to coexist, the failure is a discount code limit conflict — not a gift card conflict. The promotional code is the variable. Communicate this clearly to support and to your loyalty app vendor.
6
Check for bundle app conflicts. If you use a bundle app (Bold Bundles, Bundler, Rebuy bundle mode), verify whether it applies a Shopify discount code for bundle pricing. If it does, the same discount code limit applies — and loyalty + bundle discount + promotional code = three codes competing on a one-code plan.
7
Test on a real mobile device. If loyalty uses a Checkout Extension (not a discount code), mobile Safari is a common failure point — extension blocks may fail to render or fail to apply the loyalty discount on iOS Safari even when they work on desktop Chrome. Reproduce on a real iPhone, not a simulator. If the failure is mobile-only, it is a Checkout Extension rendering issue, not a stacking conflict.

FAQ

Frequently Asked Questions

Why does my Shopify loyalty discount disappear when I apply a gift card?

Loyalty discounts disappearing when a gift card is applied is almost never caused by the gift card itself. Gift cards are payment instruments — they are processed at the payment step and do not interact with the discount code layer. What causes loyalty points to vanish is a discount code limit triggered by a cart recalculation.

When you apply a gift card, Shopify recalculates the cart. During that recalculation, the discount code limit is re-enforced. If a promotional code was also active alongside the loyalty code, one of the two codes gets dropped. Because the loyalty app applied its code automatically (the customer never typed it), the customer notices only that the loyalty points are gone — not that the promotional code was the real conflict.

Test by removing the promotional code, re-applying the loyalty points, then applying the gift card. If the loyalty points survive, the promotional code was the conflict — not the gift card.

Can Shopify gift cards and loyalty points be used together at checkout?

Yes — gift cards and loyalty points can be used together at Shopify checkout in most configurations. The two systems are designed to coexist: gift cards are payment instruments (applied at the payment step), and loyalty points are either discount codes or checkout extensions (applied at the discount step). These two layers do not conflict by design.

The conflict happens when a third element is added: a promotional discount code. On standard Shopify plans, the discount code limit is one code per checkout. Loyalty (discount code) + promotional code = two codes = one gets dropped when the gift card triggers a cart refresh. On Shopify Plus, stacking is available if the Combine setting is enabled per discount code in Admin → Discounts → [code] → Combinations.

Why does my Smile.io points discount disappear when a gift card is entered at Shopify checkout?

Smile.io applies points redemptions as Shopify discount codes — one unique code per redemption, generated when the customer clicks "Redeem." This code is applied to the cart automatically. Shopify gift cards (native) are payment instruments that do not use the discount API, so a gift card alone cannot conflict with a Smile.io code.

The Smile.io code disappears when a gift card is applied for one of three reasons: (1) a promotional code was also active, making two discount codes total — the gift card triggered a cart refresh that enforced the one-code limit; (2) the cart session that held the Smile.io code expired between redemption and the gift card being applied; (3) the cart refresh event order caused Smile.io's code re-application to miss its window.

To isolate the cause: remove any promotional code and re-test. If the Smile.io code now survives the gift card application, the cause was a three-way conflict. If it still disappears, contact Smile.io support with your reproduction steps — they have specific guidance for this cart event timing issue.

Why can't my Shopify customer use both a gift card and a promotional discount code?

Shopify gift cards do not count against the discount code limit. A customer can always use a gift card plus one discount code together on a standard plan. The issue is when more than one discount code is active — which can happen without the customer realizing it if a loyalty app auto-applies a code.

If a customer reports that their promotional discount code is not working alongside a gift card, check whether the loyalty app has already applied a discount code to their cart. The loyalty code + promotional code = two codes = one is rejected. The gift card is not the cause.

Also check Admin → Discounts → [promotional code] → Combinations. If "Cannot be combined with other discounts" is shown, that code blocks any other discount — including loyalty codes — from being used simultaneously, regardless of plan.

How do I test whether gift card, loyalty points, and discount codes work together at Shopify checkout?

Run three test scenarios monthly and after every loyalty app update: Scenario 1 — gift card only: issue a test gift card in Admin → Products → Gift cards, complete a checkout using only the gift card, verify the correct amount was deducted. Scenario 2 — loyalty points only: log in as a test customer with redeemable points, redeem in cart, complete checkout, verify the correct discount was applied. Scenario 3 — all together: log in as the same test customer, add to cart, redeem loyalty points, apply a promotional code, apply the gift card, observe whether all three survive to the order confirmation.

Scenario 3 reveals exactly what your customers experience. If any element disappears during this test, note which step caused the drop and check the stacking matrix above to identify the root cause. Run this test sequence in an incognito window to prevent cached session state from masking failures. Document the results so that after the next Shopify update, loyalty app update, or discount rule change, you can re-run and compare.
