Shopify Checkout Duties & Import Fees Not Working — 8 Causes Explained

Duties not showing at checkout? Estimated duty amount wrong? DDP collection toggle greyed out? Customers getting surprise duty bills on delivery? Here are all 8 root causes of Shopify duty and import fee failures — for merchants and developers.

Updated June 2026 8 failure modes DDP vs DDU · HS Codes · Shopify Payments Required · De Minimis · Country of Origin · Exchange Rate · Headless

Shopify Markets introduced native duty and import fee calculation at checkout — but the feature has more silent failure modes than almost any other part of the checkout flow. When duties don't show, show the wrong amount, or aren't actually collected, there's rarely an error message to explain why.

The two most common causes merchants don't know about: (1) there is a fundamental difference between DDU mode (shows an estimate, collects nothing) and DDP mode (actually collects the duty at checkout) — most merchants enable Markets without realizing they're in DDU mode and their customers still get carrier duty bills on delivery; and (2) DDP mode is only available on Shopify Payments — third-party gateways cannot collect duties at checkout regardless of Markets settings.

Silent Failure Pattern

Shopify shows no error when duty calculation is incomplete. If your HS code is missing or your gateway doesn't support DDP, the checkout silently shows nothing (or a $0 estimate) with no indication that duties should have appeared. The first sign of a problem is often a customer complaint about an unexpected bill on delivery.

Failure Mode 1 — DDU vs DDP: "Estimated duties" is displayed but not collected

Shopify Markets has two distinct duty modes that most merchants confuse:

  • DDU (Delivered Duty Unpaid): Shopify shows an "Estimated duties and import fees" line at checkout as an informational notice. The customer does NOT pay at checkout. The carrier collects the actual duty from the customer on delivery. The actual amount often differs from the estimate (different exchange rate, carrier handling fees added).
  • DDP (Delivered Duty Paid): Shopify actually collects the calculated duty at checkout. The customer pays upfront. No duty bill on delivery. Requires Shopify Payments.

Most merchants who "enable Markets" land in DDU mode without realizing it. The checkout looks correct — it shows a duty line — but the duty isn't collected. Customers still get bills from DHL, FedEx, or UPS on delivery, which generates support tickets and chargebacks.

How to Check Your Mode

Admin → Markets → [Market] → Duties and import taxes. If you see "Include an estimate of duties and import taxes" selected, you're in DDU mode. To switch to DDP, select "Collect duties and import taxes at checkout" — this option only appears if Shopify Payments is your active gateway.

Failure Mode 2 — Shopify Payments required for DDP duty collection

DDP duty collection at checkout is exclusively available to stores using Shopify Payments as their primary payment gateway. This is not documented prominently in the Markets setup flow.

If your store uses any third-party gateway — Stripe (via direct integration, not Shopify Payments with Stripe backend), PayPal, Authorize.net, Braintree, Square, Adyen, or any other processor — the "Collect duties and import taxes at checkout" (DDP) toggle is hidden or greyed out in Markets settings. You can only offer DDU estimates.

Note: In some countries, Shopify Payments is not available. Merchants in those regions cannot use DDP mode at all. Check Shopify's country availability list for Shopify Payments coverage.

Failure Mode 3 — HS code missing on products: silent $0 duty

Shopify uses the Harmonized System (HS) code on each product to look up the applicable duty rate for the destination country. If the HS code is missing, Shopify cannot calculate a duty and shows nothing — no estimate, no error, just a blank where the duty line should be.

The HS code field is located at: Admin → Products → [Product] → Shipping → Customs information → HS (Harmonized System) code

Common situations where HS codes are missing:

  • Products imported from a CSV that didn't include HS codes
  • Products added before the merchant enabled Markets / duty calculation
  • Products created via third-party inventory sync apps that don't map HS codes
  • Products added via the Shopify mobile app (HS code field less prominent)
  • Large catalogs where only some products have HS codes — duty shows for some, not others
Bulk HS Code Update

Use the Products CSV export/import to update HS codes in bulk. The column is "Variant HS Code". GS1 and the WCO HS Nomenclature database are free references. For US imports, use the HTS (Harmonized Tariff Schedule) code — the first 6 digits are internationally standardized and work for Shopify's lookup.

Failure Mode 4 — Country of origin missing: wrong duty rate or no duty

Alongside the HS code, Shopify also requires the country of origin for each product to calculate the correct duty rate. The same product with the same HS code has very different duty rates depending on where it was made — trade agreements (USMCA/CUSMA for North America, EU trade deals, GSP programs for developing nations) can reduce duty to 0% for eligible origins.

Country of origin is set at: Admin → Products → [Product] → Shipping → Customs information → Country/Region of origin

Impact of missing country of origin:

  • Shopify may fall back to the Most Favored Nation (MFN) rate — the standard tariff rate that applies without any trade agreement, which is typically the highest possible duty rate
  • For some product/country combinations, duty calculation is blocked entirely with no estimate shown
  • Merchants selling products made in China entering the US may see vastly different rates from merchants selling EU-made goods — country of origin determines whether Section 301 tariffs apply

Failure Mode 5 — De minimis thresholds not applied correctly

Every country has a de minimis threshold — an order value below which no duties or taxes are charged. When an order is below the threshold, duties should show as $0 at checkout. When above, full duties apply. Merchants are surprised when duty behavior changes at an invisible value boundary.

Common failures related to de minimis:

  • Canada shock: Merchants who sell into Canada successfully assume duty-free because US orders are duty-free. Canada's CAD $20 threshold means almost everything attracts duty — customers get large unexpected bills.
  • EU VAT misconception: The EU eliminated the €22 VAT exemption in July 2021. Now all EU imports attract VAT (no de minimis for VAT). The €150 threshold only applies to customs duty, not VAT. Merchants show €0 "duties" for sub-€150 orders but EU customers still owe VAT on delivery if the store doesn't collect it via IOSS.
  • Third-party app conflict: Some duty calculation apps have outdated threshold tables and show duties on US orders under $800 or fail to show duty on Canadian orders under CAD $20.

Failure Mode 6 — Third-party duty apps conflicting with Shopify Markets

Before Shopify Markets introduced native duty calculation, many merchants installed third-party duty and landed cost apps (Zonos, DutiCalc, Easyship, GlobalE, Flow Commerce). When Shopify Markets is later enabled with duty calculation, both systems try to calculate and display duties simultaneously, causing:

  • Double duty line items: Two separate duty charges appear at checkout — one from the Shopify Markets engine and one from the third-party app
  • Conflicting amounts: The two systems use different rate databases and exchange rates, producing different numbers that confuse customers
  • Cart total inconsistency: One system's duty amount is included in the cart total while the other is displayed as informational, causing the displayed total to not match what's actually charged
  • Third-party app overrides Shopify Markets: Some apps inject their own duty calculation layer that overrides Shopify's native calculation entirely, breaking DDP collection
Pick One System

You cannot reliably run both Shopify Markets native duty calculation and a third-party duty app simultaneously. Decide which system you want to use and disable or uninstall the other. For most merchants on Shopify Payments, the native Markets system is the simpler choice. For complex multi-carrier landed cost scenarios, a dedicated third-party solution with the native system disabled is the alternative.

Failure Mode 7 — Exchange rate timing: checkout estimate vs actual customs bill

Even when Shopify correctly calculates and collects duty in DDP mode, the amount collected at checkout may differ from the actual duty assessed by customs. This is structural and expected, but causes customer confusion and disputes.

The sequence:

  1. Customer checks out on Day 1. Shopify calculates duty using today's exchange rate between the destination currency and the product's origin currency.
  2. The package ships and clears customs on Day 10 (or Day 30 for surface shipping).
  3. Customs assesses duty using the exchange rate on Day 10 — a different rate than Day 1.
  4. In DDP mode, the carrier pays customs and reconciles against the amount collected at checkout. If the customs bill is higher than collected, the carrier may bill the merchant for the difference. If lower, the difference is typically absorbed.

Additional factors that cause checkout estimate vs actual bill differences:

  • Carrier handling fees: DHL, FedEx, UPS charge an "advancement fee" or "customs brokerage fee" for paying duty on the customer's behalf — typically $10–40 USD. This fee is not included in the Shopify duty estimate and appears as a separate charge even in DDP mode for DDU carriers.
  • HS code reclassification: Customs inspectors sometimes reclassify goods under a different HS code than declared, applying a different duty rate. The Shopify estimate is based on the declared HS code, not the inspected classification.
  • Declared value disputes: Some countries (Brazil, Russia, India) apply duty to a calculated import value that may differ from the invoice value, especially for high-value goods.

Failure Mode 8 — Headless storefronts: @inContext missing or duties not queried

Headless storefronts using the Shopify Storefront API have no automatic duty display. Duties must be explicitly requested and rendered. Two separate issues cause duties to silently not appear:

Issue A — Missing @inContext directive

Duty calculation in the Storefront API requires the @inContext(country: X) directive on all cart and product price queries. Without it, Shopify returns prices and costs in the store's primary currency with no duty calculation applied.

# WRONG — no country context, no duty calculation
query {
  cart(id: $cartId) {
    cost {
      totalAmount { amount currencyCode }
    }
  }
}

# CORRECT — country context enables duty calculation
query @inContext(country: CA) {
  cart(id: $cartId) {
    cost {
      totalAmount { amount currencyCode }
      dutiesAmount { amount currencyCode }   # Must query this field explicitly
      totalDutyAmount { amount currencyCode }
    }
  }
}

Issue B — Duties not in totalAmount in DDU mode

The behavior of cart.cost.totalAmount differs by duty mode:

  • DDP mode: totalAmount includes duties. The customer pays the displayed total and owes nothing on delivery.
  • DDU mode: totalAmount does NOT include duties. Duties appear in dutiesAmount as an informational field. If your headless storefront only renders totalAmount, duties are invisible to customers in DDU mode.

Most common headless duty mistakes:

  • Querying totalAmount only and not displaying dutiesAmount separately in DDU mode
  • Using Storefront API token with @inContext for product prices but forgetting to pass it for cart queries
  • Country selector updates the display locale but doesn't call cartBuyerIdentityUpdate with the new country — cart currency and duty calculation stay stale
  • Passing an incorrect country code format — must be ISO 3166-1 alpha-2 (e.g., CA, not CAN or Canada)
# cartBuyerIdentityUpdate must fire when country changes
mutation updateBuyerIdentity($cartId: ID!, $country: CountryCode!) @inContext(country: $country) {
  cartBuyerIdentityUpdate(
    cartId: $cartId
    buyerIdentity: { countryCode: $country }
  ) {
    cart {
      cost {
        totalAmount { amount currencyCode }
        dutiesAmount { amount currencyCode }
      }
    }
  }
}

Quick Diagnosis Checklist

  • Go to Admin → Markets → [Market] → Duties and import taxes — confirm whether you're in DDU or DDP mode
  • Confirm Shopify Payments is your active gateway (required for DDP)
  • Check one affected product: Admin → Products → [Product] → Shipping — verify HS code AND country of origin are both filled in
  • Run a test order to a duty-applicable country (Germany, Australia, Canada) with a product that has HS codes set — verify whether a duty line appears
  • Check whether you have any third-party duty calculation apps active — if so, test with them disabled
  • For headless storefronts: verify @inContext(country: X) is present on cart queries and dutiesAmount is being queried and rendered
  • For EU/UK: confirm IOSS (EU VAT One-Stop Shop) or UK VAT registration if applicable — failure to collect VAT is separate from duty calculation but often confused with it
