Shopify Spotii Not Working — Not Showing, Declined, Zip Integration Issues

Updated June 2026 · 7 failure modes · UAE and Saudi Arabia merchants

Spotii is a UAE-founded BNPL platform — now owned by Zip Co. — operating in the UAE and Saudi Arabia with a Pay-in-4 bi-weekly model. Its Shopify integration fails in ways tied to its two-market footprint, its post-Zip-acquisition infrastructure changes, and its opaque spending limits that customers cannot see. Understanding the difference between a merchant-side configuration failure (fixable) and a customer-level risk decline (Spotii's decision) is the fastest path to resolution. This guide covers all 7 Spotii failure modes with exact diagnostic steps for UAE and Saudi Arabia merchants.

How Spotii Differs From Tabby and Tamara

Spotii is the most geographically limited of the three main MENA BNPL platforms on Shopify. Spotii covers only UAE (AED) and Saudi Arabia (SAR). Tabby covers UAE, KSA, Egypt, Kuwait, and Qatar. Tamara covers UAE, KSA, Kuwait, and Bahrain. Merchants whose customers span the full GCC — especially Egypt, Kuwait, Qatar, or Bahrain — cannot rely on Spotii alone and need a second BNPL provider for those markets. Merchants who installed Spotii when it was the primary UAE BNPL option often discover this limitation only when customers from Egypt or Kuwait report that the BNPL option is missing.

Spotii is owned by Zip Co., an Australian company. Zip acquired Spotii in 2021 as part of its MENA market expansion. The acquisition has been largely positive for infrastructure and risk modelling, but it created a transition period in which merchant portal URLs changed, API endpoint documentation was consolidated under Zip's global developer docs, and some legacy Spotii-branded integrations stopped receiving updates. Merchants who set up Spotii before mid-2021 and have not audited their integration since may be referencing discontinued infrastructure. This is a uniquely Spotii failure mode with no Tabby or Tamara equivalent.

Spotii uses a Pay-in-4 bi-weekly model. The customer pays in four equal installments, one every two weeks — so the total purchase cost is spread over six weeks, not four months (Tabby's monthly Pay-in-4) or two months (Tamara's Pay-in-3 at 30-day intervals). The bi-weekly cadence means customers pay off Spotii plans faster than Tabby plans, which theoretically frees their spending capacity sooner — but also means payment due dates arrive more quickly and missed payments accumulate faster if a customer is in financial difficulty.

Spotii's spending limit is opaque — customers cannot see it. Unlike Tamara, which shows customers their available credit limit in the app, Spotii does not disclose the customer's spending capacity. Customers discover the limit only when they are declined. This opacity creates support tickets where customers report "Spotii is broken at checkout" when what actually happened is that Spotii silently declined them due to a limit they had no way to know about. Tabby has the same opacity; Tamara is the exception that shows the limit.

7 Failure Modes Covered

  4. Emirates ID or National ID verification failure

1

Spotii Not Showing at Shopify Checkout

Merchant-side

Spotii is a third-party alternative payment provider on Shopify, which means its visibility at checkout requires two separate actions that many merchants conflate. Installing the Spotii app from the Shopify App Store is step one — but Spotii will not appear at checkout until it is also manually enabled as an alternative payment provider in Shopify Admin → Settings → Payments → Alternative payment providers. Merchants frequently install the app, see it in their app list, and assume the checkout activation is complete. It is not.

Currency is the hard gate for Spotii. Spotii operates exclusively in UAE (AED) and Saudi Arabia (SAR). If your store's checkout currency for UAE customers is USD, EUR, or any non-AED currency — even if AED is displayed as a runtime conversion from USD — Spotii will not appear. The checkout must present a fixed AED transaction for UAE customers and a fixed SAR transaction for Saudi customers. Multi-currency stores that use Shopify Markets with USD as the base currency and runtime conversion to AED will suppress Spotii for all UAE customers regardless of configuration. To fix this, set AED as the fixed checkout currency for the UAE market in Shopify Admin → Settings → Markets → UAE.

Merchant account onboarding is a common silent blocker. Spotii (under Zip) requires completed KYC and business verification before going live. This includes commercial registration documents, bank account verification, and a review period that typically takes 3–7 business days. Merchants who went through this process before the Zip acquisition may need to re-verify if Zip migrated their account data and flagged it for re-review. The Spotii/Zip merchant portal will show pending status if onboarding is incomplete — but this status is not surfaced at Shopify checkout, so merchants may not know until they test a full checkout flow.

Additional suppression causes: test/sandbox API credentials are configured instead of live production keys (a very common cause of Spotii appearing in test mode but not at live checkout); the cart contains only digital products or gift cards (Spotii does not support non-physical goods); or the Shopify app version is outdated and needs updating from the App Store.

How to Diagnose

  1. Check two-step activation: Shopify Admin → Apps → confirm the Spotii app is listed as installed; then Shopify Admin → Settings → Payments → Alternative payment providers → confirm Spotii is listed and the toggle is enabled. Both must be true simultaneously.
  2. Check checkout currency per market: Shopify Admin → Settings → Markets → UAE → confirm AED is set as the fixed checkout currency, not a runtime conversion. Repeat for Saudi Arabia (SAR). Currency conversion from USD is the most common suppression cause for multi-market stores.
  3. Check Spotii/Zip merchant account status: Log into the Spotii/Zip merchant portal and verify your account shows "Live" or "Active" status. Accounts pending review, KYC documents, or risk assessment will suppress Spotii at checkout with no error message to the merchant.
  4. Verify API credential mode: In the Spotii app settings in Shopify, confirm live/production API credentials are configured — not sandbox or test credentials. Test credentials suppress Spotii for real customers while allowing test checkouts to proceed, which creates confusion about when the issue started.
  5. Test with a physical product above minimum: Add physical products totalling above 50 AED (UAE) or 200 SAR (Saudi Arabia) to cart and checkout with a UAE or Saudi Arabia shipping address. Spotii should appear for valid carts.
  6. Check the Spotii app version: Shopify Admin → Apps → Spotii → check if an update is available. Post-acquisition app updates from Zip have occasionally required merchants to re-authorize with updated API scope permissions.
Two-market scope note: Spotii appears only for UAE (AED) and Saudi Arabia (SAR) checkouts. If customers from Kuwait, Bahrain, Egypt, or Qatar report Spotii is missing — it is not a configuration error. Spotii simply does not operate in those markets. Consider adding Tabby (covers Egypt, Kuwait, Qatar) or Tamara (covers Kuwait, Bahrain) for broader GCC coverage.
2

Per-Transaction Risk Declined — "Not Available for This Order"

Customer-side

Like Tabby and Tamara, Spotii runs an independent risk assessment on every transaction — past approval does not guarantee future approval. Each checkout triggers a real-time decision against Spotii's risk model, which evaluates the customer's payment history across all Spotii/Zip merchants, the number of currently open Spotii plans, the customer's total outstanding Spotii balance relative to the requested cart value, the customer's phone verification status, and pattern signals from the specific transaction (merchant category, cart composition, session behavior).

First-time Spotii customers face the highest risk of decline. The Spotii risk model starts conservatively for new users with no payment history. First-time purchases are scored against a limited data set, which can result in a decline or approval for a smaller amount than the customer expected. Customers who have built a 6–12 month track record of on-time Spotii payments generally experience better approval rates and higher available spending capacity over time.

Spotii's decline messages are opaque to customers. Because Spotii does not show customers their spending limit (unlike Tamara), a declined customer at checkout sees a message that Spotii is "not available for this order" — with no explanation of whether this is a risk decision, a limit issue, or a verification problem. This opacity means nearly every Spotii decline generates a customer support ticket to the merchant asking "why doesn't Spotii work?" when in fact the issue is entirely between Spotii and the customer's account.

Merchants cannot override Spotii risk decisions, access the decline reason, or appeal on behalf of customers. The most effective merchant response is to ensure that alternative payment methods (Tabby, credit card, Apple Pay, Google Pay) are visibly available at checkout so customers have a fallback when Spotii declines — and to include a brief checkout message noting that BNPL availability depends on the individual customer's account status.

Fix Steps for Customers

  1. Try a smaller cart value: Spotii may approve a smaller purchase even if a larger one was declined. If your cart is close to your Spotii spending capacity, reducing the total may flip the decision — you can complete the remainder in a separate order later.
  2. Wait for existing Spotii plans to progress: Each bi-weekly payment you make on open plans frees spending capacity. After one or two payments on existing plans, retry the declined purchase — your available Spotii balance will be higher.
  3. Verify your Spotii account is fully set up: Log into the Spotii app and confirm your Emirates ID (UAE) or National ID (KSA) is verified and your phone number is confirmed. An incomplete account profile is a silent decline trigger.
  4. Contact Spotii/Zip customer support: Spotii's in-app support can provide more context on a specific decline — they cannot always reverse it, but they can indicate whether it is a limit, verification, or risk issue and what you can do to improve approval chances.
  5. Use Tabby or a credit card as an alternative: If Spotii declines you on a specific purchase, Tabby (if offered by the merchant) provides a parallel MENA BNPL path, or complete the purchase with a credit or debit card.
Merchant note: Sporadic Spotii declines affecting specific customers are not integration errors — they are customer-level risk decisions. If all customers are being declined simultaneously, that is a merchant account or configuration issue (check failure mode #7 for post-acquisition integration problems). If only some customers see declines, your Spotii integration is working correctly and the issue is on the customer side.
3

Spending Limit Reached (Opaque — Customers Cannot See It)

Customer-side

Spotii assigns each customer a maximum outstanding balance — the total they can carry across all open Spotii plans simultaneously. When a new purchase would take their outstanding balance above this limit, the transaction is declined. Unlike Tamara — which shows customers their available credit limit clearly in the app — Spotii does not make this limit visible. Customers never know their Spotii spending limit until they hit it, and when they do, they receive only a generic "not available for this order" message with no indication of why.

The bi-weekly payment model makes limit exhaustion patterns different from Tabby and Tamara. Spotii's four payments occur every two weeks, so a plan started this week has one payment due in two weeks, another in four weeks, another in six weeks. A customer with multiple concurrent Spotii plans (across multiple merchants) has payments due every two weeks to multiple merchants simultaneously. This two-week cadence means spending capacity refreshes faster than with Tabby's monthly model — but it also means customers who miss a payment face more frequent due dates and can exhaust their limit faster during peak shopping periods.

Seasonal spending events create predictable limit exhaustion. During Ramadan and the White Friday sales period, UAE and KSA consumers open multiple Spotii plans across different merchants in a short window. By the middle of a major sale event, customers who started the season with available Spotii capacity may have consumed it entirely across five or six open plans — even at the bi-weekly payment pace. These customers then arrive at your checkout with zero available Spotii balance and experience a decline that they attribute to your store or your Spotii integration, not to their own spending pattern.

Fix Steps for Customers

  1. Check your open Spotii plans: Log into the Spotii app → My Orders or Plans → review how many open plans you have and what their remaining balances are. Total outstanding balance across all plans is what counts against your limit.
  2. Make a payment on an existing plan: Spotii app → select an open plan → Make a payment. Even a partial early payment on one plan reduces your outstanding balance and frees spending capacity immediately. You do not need to close a plan entirely to free limit.
  3. Wait for the next scheduled bi-weekly payment: Spotii payments occur every two weeks automatically. After your next scheduled payment on any open plan, your outstanding balance decreases and your available spending capacity increases — retry the declined purchase after that payment clears.
  4. Try a lower cart value: If your available Spotii capacity is partially consumed, a smaller cart value may fit within your remaining limit. Split a larger order across two purchases if possible.
  5. Contact Spotii support for a limit review: If you have a long history of on-time Spotii payments, contact Spotii support to request a spending limit review. Customers with 12+ months of clean payment history are more likely to receive a limit increase.
Merchant note: Because Spotii's limit is opaque, customers who hit their limit have no self-diagnostic path the way Tamara customers do. The most useful thing you can do as a merchant is add a visible note at checkout (via Shopify's checkout editor or an app) indicating that BNPL availability depends on individual account status and offering an alternative payment method clearly. This reduces support volume from limit-declined customers who otherwise assume your store is broken.
4

Emirates ID or Saudi National ID Verification Failure

Customer-side

Spotii requires identity verification as part of account setup and risk management. UAE customers must verify with their Emirates ID; Saudi Arabia customers must verify with their National ID (Saudi Huwiyya for nationals, or Iqama residence permit for expats). This verification happens at the Spotii account level — not at every checkout — but unverified or partially verified accounts are blocked from completing purchases, which surfaces at checkout as a decline with no explanation about the verification issue.

Emirates ID verification failures are the most common identity issue for UAE customers. Emirates ID verification fails when: the ID number entered does not match records; the Emirates ID is expired (UAE residents are required to renew their Emirates ID periodically, and an expired ID is not accepted for identity verification); the name on the Emirates ID does not match the name registered on the Spotii account; or the customer registered Spotii before completing their Emirates ID renewal and the Spotii account has outdated ID data that needs to be updated.

Expatriate identity verification is a consistent friction point in both UAE and KSA. The UAE has a very large expatriate workforce — many UAE residents do not hold an Emirates ID (for example, visitors on short-term visas) and cannot meet Spotii's UAE customer identity requirements. Similarly, in Saudi Arabia, expatriate residents with Iqama face the same verification challenges: an Iqama that has recently been renewed, an Iqama number that has changed, or a name transliteration that differs between the Iqama and the Spotii account can all block verification. Merchants targeting tourist or visitor segments in UAE are likely to see higher Spotii decline rates than merchants whose customer base is UAE resident nationals.

Customers cannot complete identity re-verification at the Shopify checkout page. The fix always happens inside the Spotii app before attempting another purchase. Customers must re-submit identity documents through the Spotii/Zip app and wait for re-verification, which can take from a few hours to several business days depending on document backlog.

Fix Steps for Customers

  1. Open the Spotii app and check your verification status: Spotii app → Account or Profile → check if your Emirates ID (UAE) or National ID/Iqama (Saudi Arabia) shows as verified. An unverified or pending status blocks all purchases.
  2. Update your ID if recently renewed: If you recently renewed your Emirates ID or Iqama, your Spotii account may still reference your old ID data. Spotii app → Account → Update ID → submit your new ID details and front/back ID scan.
  3. Ensure name consistency: The name on your ID must match the name on your Spotii account exactly. For expatriates, name transliteration from Arabic to English can differ between the Iqama and the Spotii account — contact Spotii support to manually match the name if they differ.
  4. Submit clear ID images: If Spotii's automated verification rejected your document scan, resubmit with a higher quality image — well-lit, flat surface, all four corners of the ID visible, no glare across the chip or barcode.
  5. Wait for re-verification to complete: After resubmitting identity documents, verification typically completes within a few business hours during working hours (UAE/KSA business days). Submissions made on weekends or UAE public holidays may take longer.
  6. Contact Spotii support directly: If automated verification has failed more than twice, manual review by Spotii's compliance team is the only remaining path. Contact Spotii/Zip customer support with your account details and explain which verification step failed.
UAE visitor note: Short-term visa holders and tourists in the UAE typically cannot verify a Spotii account because Emirates ID is required and they do not hold one. Spotii is designed for UAE and KSA residents, not for tourist or visitor commerce. Merchants whose stores attract significant UAE visitor traffic should not rely on Spotii as a primary BNPL option — and should ensure credit card, Apple Pay, and Google Pay are offered clearly.
5

Phone Number Verification / OTP Failure

Customer-side

Spotii uses SMS OTP (one-time password) verification during account setup and, in some flows, at checkout confirmation. The OTP is sent to the mobile number registered on the customer's Spotii account. OTP delivery fails in several specific scenarios that are especially common in the UAE and Saudi Arabia due to regional SIM and telecom patterns.

Non-UAE/KSA SIM cards are the primary OTP failure cause. Spotii's OTP delivery is optimized for UAE (+971) and Saudi Arabia (+966) mobile numbers. Customers who registered Spotii with a UAE or KSA number but are now using a foreign SIM (roaming or a SIM swap) may not receive OTPs reliably — international roaming can delay or drop SMS messages from UAE-based telecom gateways. Customers who moved away from the UAE or Saudi Arabia but still have a Spotii account on their old number will face OTP failure on every checkout until they update their phone number.

Dual-SIM phones are a frequent OTP confusion source. Many UAE and KSA residents carry dual-SIM phones — often one UAE/KSA local SIM and one home-country SIM. OTPs intended for the UAE number sometimes arrive on the second SIM unexpectedly (if the UAE number is assigned to SIM slot 2), or the customer is checking the wrong SIM for the OTP. When the customer reports the OTP "didn't arrive," the SMS was often delivered — but to the SIM they are not actively checking.

Blocked or filtered SMS is also a cause. Some UAE and KSA mobile carriers filter short-code or promotional SMS aggressively; Spotii OTPs sent via certain telecom gateways may be routed to filtered or spam SMS folders on some carrier networks. This affects a minority of customers but creates unexplained, intermittent OTP failures that look like a Spotii system problem.

Fix Steps for Customers

  1. Confirm which number your Spotii account is registered to: Spotii app → Account → Phone Number — verify this is your current, active UAE or KSA number. An old number from a previous SIM is the most common OTP failure cause.
  2. Check both SIMs on a dual-SIM phone: If you use a dual-SIM device, check the SMS inbox on both SIMs — the OTP may have been delivered to the SIM you are not currently using for data.
  3. Check SMS spam/filtered folders: Some UAE and KSA carriers route short-code SMS to filtered or promotional SMS folders. Check your messages app for a filtered or spam folder if the OTP did not arrive in your main inbox.
  4. Request a new OTP: OTPs expire after a short window (typically 3–5 minutes). If you took too long entering it, request a new one rather than retrying the expired code.
  5. Update your phone number if it has changed: Spotii app → Account → Update Phone Number — enter your current active UAE or KSA number and complete re-verification before attempting another checkout. This change requires re-verification of the new number.
  6. Try over a stable mobile connection: OTP SMS delivery can be affected by intermittent network connectivity. If you are on a weak signal or WiFi calling, switch to a strong 4G/5G mobile data connection and request a new OTP.

Order Value Below Spotii Minimum

Merchant + Customer

Spotii has a minimum order value requirement — typically 50 AED in the UAE and 200 SAR in Saudi Arabia. Carts below these minimums will not display Spotii at checkout, regardless of the customer's account status or the merchant's configuration. This is a hard threshold that cannot be overridden, and it is applied before Spotii's risk assessment runs — so a below-minimum cart never reaches the point of showing Spotii as an option.

The minimum is applied to the Shopify checkout transaction value, not the catalog price. Discount codes, automatic discounts, free-shipping thresholds, or order-level promotions that reduce the checkout total below the Spotii minimum will suppress Spotii — even if the cart contained items priced above the minimum before discounts. A customer who adds a large discount code that brings the order total below 50 AED will no longer see Spotii at checkout, even if Spotii was visible earlier in the checkout flow without the discount applied.

Spotii minimum thresholds differ from Tabby and Tamara. Tabby's minimum is 1 AED (effectively no minimum for UAE) and 10 SAR for Saudi Arabia. Tamara's minimum is 100 SAR for Saudi Arabia and 100 AED for UAE — which is higher than Spotii's AED minimum and the same as Spotii's SAR minimum. For low-value average-order-value stores (cosmetics, accessories, small home goods), Spotii's 50 AED minimum means a meaningful portion of carts will naturally fall below it, while Tabby's near-zero minimum would capture those carts.

Fix Steps

  1. Verify cart value before troubleshooting further: Add up items in the cart after discounts are applied — if the total is below 50 AED (UAE) or 200 SAR (Saudi Arabia), Spotii not showing is expected behavior, not a bug.
  2. Check if a discount code is bringing the cart below minimum: A large discount code or automatic promotion applied at checkout may push the order total below the Spotii threshold. Test the checkout flow with and without the discount to confirm.
  3. Compare Tabby and Spotii minimum thresholds: If Spotii suppression due to below-minimum carts is affecting a significant portion of your orders, consider whether Tabby (lower minimum, broader market coverage) better fits your product catalog and average order value.
  4. Consider minimum order incentives: If Spotii is a significant BNPL option for your customers, adding a small incentive (free shipping threshold, bundle pricing) that naturally pushes orders above 50 AED may both increase Spotii eligibility and raise your average order value.

Post-Zip-Acquisition Integration Breaks

Merchant-side

Zip Co. (ASX: ZIP) acquired Spotii in 2021 to enter the MENA BNPL market. The acquisition migrated Spotii's infrastructure into Zip's global platform over subsequent months. For merchants who integrated Spotii before the acquisition — or who set up their integration using pre-acquisition documentation — several elements of the integration may now be pointing to deprecated Spotii-branded infrastructure. This is the most uniquely "Spotii" failure mode: Tabby and Tamara have no equivalent, as both have remained under their original ownership without major infrastructure transitions.

Merchant portal URL changes. The original Spotii merchant portal was at merchant.spotii.me. Following the Zip acquisition, merchant management migrated to the Zip merchant portal. Merchants still bookmarking or being linked to the old Spotii URL may be accessing a deprecated interface — changes made there may not propagate to the live Spotii/Zip checkout system, or may not propagate at all if the old portal has been decommissioned. Verify you are accessing the current Zip-operated merchant portal by checking Spotii's official documentation.

API endpoint and webhook URL changes. Spotii's pre-acquisition API used spotii.me-hosted endpoints. Post-acquisition, Zip consolidated API infrastructure. Merchants with hardcoded Spotii API base URLs in their configuration (especially those who customized the Shopify app settings manually) may be making API calls to endpoints that no longer respond or that redirect incorrectly. Webhook URLs for order notifications also changed, which can cause a merchant's backend to stop receiving Spotii order confirmations even when checkouts are completing successfully from the customer's perspective.

API credentials may have expired or been re-issued. As part of the Zip migration, some merchants had their API credentials re-issued under Zip's key management system. Merchants who did not receive or apply the new credentials continue to see Spotii appear at checkout (the app itself still loads) but transactions fail at the payment processing stage, often with a generic "payment could not be completed" error. Re-issuing and applying new API credentials from the current Zip merchant portal typically resolves this.

The Spotii Shopify app itself has been updated post-acquisition — but auto-updates to the app do not automatically update the underlying API credentials or webhook URLs if they were manually customized outside the default app configuration. Merchants who customized their Spotii integration should review each customized element against current documentation.

How to Diagnose and Fix

  1. Access the current merchant portal: Go to the current Spotii/Zip merchant portal (not merchant.spotii.me if that URL no longer resolves correctly) and log in. If your login credentials no longer work or the portal shows an error, your merchant account may need to be migrated — contact Spotii/Zip merchant support directly.
  2. Regenerate API credentials from the current portal: In the Zip merchant portal, navigate to API settings and regenerate your public and secret API keys. Apply the new keys to the Spotii app settings in Shopify Admin → Apps → Spotii → Configuration. Old pre-acquisition keys may no longer be valid.
  3. Verify webhook URLs: In the Zip merchant portal under notifications or webhooks, confirm the webhook destination URL matches your current Shopify store or backend endpoint. If the URL still references the old Spotii-era format, update it.
  4. Run a test transaction: After updating credentials and webhook URLs, use Spotii's test mode (switch to test credentials in the Shopify app settings) to run a test checkout end-to-end and confirm order confirmation is received. Then switch back to live credentials and run a small live test order.
  5. Update the Spotii Shopify app: Shopify Admin → Apps → check for available updates to the Spotii app. Post-acquisition updates include compatibility fixes for Shopify's checkout extensibility updates.
  6. Contact Spotii/Zip merchant support: If you integrated before 2022 and have not audited the integration since, the most reliable fix is to contact Zip's MENA merchant support team and ask them to walk through your account migration status and verify all integration endpoints are current.
Critical diagnostic question: Did your Spotii integration stop working without any changes on your side? If yes and the integration was set up before mid-2021, the post-acquisition infrastructure changes are the most likely cause. Start with API credential regeneration and merchant portal verification before investigating any Shopify-side configuration.
