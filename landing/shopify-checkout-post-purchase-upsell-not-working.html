Shopify Post-Purchase Upsell Not Working — Thank You Page Missing, One-Click Charge Failing

Post-purchase upsells fail in two completely different ways: the upsell page doesn't show at all, or it shows but the one-click charge is declined. Each cause has a different fix — and most merchants are debugging the wrong problem.

How Shopify Post-Purchase Upsell Actually Works

Shopify's post-purchase upsell system uses an official API called the Post-Purchase Checkout Extension. It runs inside Shopify's checkout environment — between the moment the customer's payment is confirmed and the moment they see the order confirmation (thank you) page.

Timing matters: The post-purchase page appears after the customer's card has been charged for the original order, but before they see the order confirmation page. The customer's browser is still inside Shopify's checkout domain. This is why one-click upsell is possible — Shopify still has the payment session active.
Step 1 — Checkout complete

Payment confirmed

Customer clicks "Pay now." Shopify processes the charge. The original order is created. Payment succeeds.

Step 2 — Extension fires

Post-purchase page loads

Shopify checks if a post-purchase extension is registered for this store. If yes, it renders the extension's UI before showing the thank you page. This is where the upsell appears.

Step 3 — Acceptance

One-click recharge

Customer clicks "Add to my order." Shopify attempts to recharge the same payment method for the upsell item. If the payment method supports recharging, it succeeds without re-entering card details.

Step 4 — Confirmation

Thank you page

After the upsell is accepted (or declined), Shopify shows the order confirmation page. If the customer accepted the upsell, the original order is edited to include the upsell item, or a second order is created.

Key architectural facts that cause most confusion:

  • Only one app can be assigned to the post-purchase extension slot at a time. If two apps compete for this slot, only the one currently assigned in Shopify Admin → Settings → Checkout → Post-purchase page will show.
  • The post-purchase extension runs inside Shopify's checkout domain, not your theme. Theme customizations, JavaScript, and Liquid do not affect it.
  • The Post-Purchase Checkout Extension API is available on all Shopify plans, not just Plus. However, some specific upsell apps have their own Plus requirements.
  • The "thank you page" in this context means the page that appears immediately after checkout — not the order status page a customer can revisit via their confirmation email. These are different pages with different customization mechanisms.

Why the Upsell Page Is Not Showing After Checkout

If customers complete checkout and go straight to the order confirmation page without seeing any upsell, there are six common causes:

Extension not assigned in Shopify Admin

The most common cause. Go to Shopify Admin → Settings → Checkout → Post-purchase page. There should be a dropdown where you select which app manages the post-purchase page. If no app is selected here, or the wrong app is selected, nothing will appear. Installing the app is not enough — it must be explicitly assigned in this settings panel.

Funnel not set to Active in the app

Apps like ReConvert and Zipify OCU require you to both build the funnel and then toggle it to "Active" or "Enabled." A funnel in draft or paused state will not display. Check the app dashboard for a status toggle or publish button.

Trigger conditions don't match the order

Most post-purchase apps let you set triggers: show this upsell only when the customer bought Product X, or bought from Collection Y, or spent over $Z. If the test order you're placing doesn't match the trigger conditions, the upsell page will be skipped. Temporarily set the trigger to "All orders" during testing, then refine the conditions.

A/B testing split

Many post-purchase apps include A/B testing features where only a percentage of customers see the upsell. If you've accidentally enabled a 50/50 split, roughly half of your test orders will go straight to the thank you page without seeing an upsell. Check the funnel settings for any A/B or split test configuration and confirm it's set to 100% for the upsell variant during testing.

Payment gateway blocks post-purchase extensions

Some payment gateways do not return the checkout session token that Shopify needs to display a post-purchase extension page. When this happens, Shopify skips the extension entirely and goes straight to the thank you page. This is particularly reported with certain manual payment methods and some third-party gateways that implement a non-standard redirect flow after payment confirmation.

Geographic targeting or customer segment filter

Advanced funnel configurations may restrict display to customers in specific countries, with specific customer tags, or who match a customer segment. If the test order comes from a customer who doesn't match the filter, the upsell is skipped. Check all audience and targeting settings in the funnel.

Why the One-Click Upsell Charge Is Failing

If the upsell page shows but the charge fails when the customer clicks "Add to my order," the root cause is almost always the original payment method. Post-purchase one-click charges work by recharging the payment instrument used for the original order — without asking the customer to re-enter any details. This requires Shopify to hold a reusable payment token from the original checkout.

Critical constraint: Many payment methods do not return a reusable token. When a customer pays with one of these methods, Shopify cannot issue a post-purchase recharge — the one-click mechanism is architecturally impossible regardless of app configuration.

✓ Supports one-click recharge

  • Shopify Payments — credit or debit card: Returns a reusable Stripe payment method token. One-click upsell works.
  • Shop Pay (saved card): The customer's card is stored in Shop Pay. Shopify can recharge it without re-entering details.
  • Some gateway credit cards: Third-party gateways that use Shopify's payment session API and return a compliant reusable token.

✗ Does not support one-click recharge

  • PayPal standard checkout: Single-use authorization token only. Cannot be recharged post-purchase without PayPal Billing Agreement.
  • Cash on delivery / manual payment: No payment instrument to recharge.
  • Bank transfer / wire transfer: Payment hasn't even been received yet — no token.
  • Buy Now Pay Later — most implementations: Klarna, Afterpay, Sezzle, and similar BNPL methods do not support post-purchase recharge in their standard Shopify integration.
  • 100% gift card orders: If the entire order is paid by Shopify gift card with no credit card on file, there is no reusable payment token.
What happens varies by app: Some apps (ReConvert, AfterSell) detect unsupported payment methods and hide the upsell page entirely for those customers — you may see fewer upsell impressions than expected. Other apps show the upsell page but display a fallback button ("Create a new order" instead of "Add to my order"). Others show the one-click button but the charge fails — this looks like a payment error to the customer.

App-Specific Failure Modes

ReConvert — Most Common Issues

ReConvert has two distinct products that merchants often confuse: the post-purchase funnel (uses Shopify's extension API, appears between checkout and thank you page) and the thank you page customization (customizes the order status page content, visible when the customer revisits via their confirmation email). These are configured separately and fail separately.

ReConvert issue 1

Extension not activated in Shopify

ReConvert's post-purchase funnel requires you to go to Shopify Admin → Settings → Checkout → Post-purchase page and select ReConvert from the dropdown. This step is separate from installing the app. Many merchants skip it.

ReConvert issue 2

Funnel is published but not set to primary

ReConvert allows multiple funnels. Only one can be active at a time. If you have multiple funnels, confirm the correct one is set as primary/active — others will be in "draft" or "paused" state and won't show.

ReConvert issue 3

A/B test splitting impressions

If the funnel has an A/B split configured (e.g., 50% see upsell A, 50% see upsell B, or 50% see the upsell and 50% see nothing), only a portion of customers will see the upsell. During testing, set the split to 100% for the upsell variant to confirm it works before re-enabling the split.

ReConvert issue 4

Trigger product is out of stock

If the upsell product or variant has zero inventory and inventory tracking is enabled in Shopify, ReConvert may skip showing the upsell for that product to avoid charging customers for an item that can't be fulfilled. Check the upsell product's inventory status.

Zipify OCU (One Click Upsell) — Most Common Issues

Zipify issue 1

Funnel page targeting

Zipify OCU lets you set which products or pages trigger each funnel. If the purchased product is not in the funnel's trigger list, the funnel won't appear. Check the funnel's "What products trigger this funnel?" settings.

Zipify issue 2

Funnel priority conflicts

If multiple Zipify funnels could match the same order, priority settings determine which one shows. A lower-priority funnel may be shadowed by a higher-priority one. Check the funnel priority ranking in the Zipify dashboard.

Zipify note

Shopify Plus requirement

Zipify OCU's full feature set (including some analytics and multi-step funnel capabilities) is marketed primarily at Shopify Plus merchants. Confirm your plan matches the features you're trying to use.

AfterSell Post Purchase Upsell

AfterSell is generally considered the easiest to set up because it provides a clearer onboarding checklist that includes the Shopify Admin extension assignment step. Common AfterSell issues are trigger mismatch and the upsell product being unavailable for purchase (hidden, deleted, or OOS).

CartHook Post-Purchase Upsell: Sunset and Migration

CartHook was one of the original Shopify post-purchase upsell apps and was widely adopted. Its post-purchase upsell feature was sunset in 2023 and is no longer functional.

If CartHook upsells have stopped working: This is expected. CartHook's post-purchase upsell relied on checkout.liquid customization — a deprecated Shopify mechanism. Shopify removed checkout.liquid support for new stores in August 2021 and began sunsetting it for existing stores in 2024. CartHook did not migrate to the new Post-Purchase Extension API before sunsetting its upsell product.

Migration path for former CartHook merchants:

Replacement option 1

ReConvert

Most feature-complete CartHook replacement. Offers similar funnel builder, product upsell and cross-sell, thank you page customization. Works via the official Shopify Post-Purchase Extension API.

Replacement option 2

AfterSell

Simpler setup than ReConvert. Good for merchants who need a straightforward single-upsell funnel without complex branching. Uses the official API.

Replacement option 3

Zipify OCU

Strong for stores with higher-volume testing needs. Multi-step funnels, A/B testing, and analytics. Uses the official API. Best fit for Shopify Plus merchants.

checkout.liquid Deprecation and Post-Purchase Upsell Breakage

Older post-purchase upsell implementations injected JavaScript or Liquid code directly into Shopify's checkout pages via a file called checkout.liquid. Shopify deprecated this approach as part of its Checkout Extensibility initiative.

Timeline What Changed Impact on Upsells
August 2021 New Shopify stores can no longer create checkout.liquid New stores unable to use checkout.liquid-based upsell apps from the start
2022–2023 checkout.liquid still works for existing stores; Post-Purchase Extension API available Existing implementations still working; new apps migrate to extension API
August 2024 Shopify begins requiring checkout extensibility for checkout customizations Stores upgrading to new checkout lose access to old checkout.liquid customizations
August 2025 checkout.liquid scripts no longer execute (Script Editor deprecated) Any remaining checkout.liquid upsell implementations stop working entirely
How to tell if this is your issue: If your post-purchase upsell was working and stopped, and the timing correlates with a Shopify plan upgrade or a notification about "upgrading to the new checkout," checkout.liquid deprecation is the likely cause. Check your app's documentation for migration instructions — reputable apps have migrated to the Post-Purchase Extension API.

Edge Cases That Cause Silent Failures

Digital product race condition

Customer gets their product and leaves

If the order triggers instant digital delivery (download link, license key, course access) via a fulfillment webhook, the customer may receive their email, open the product, and close the browser before the post-purchase page loads. The upsell was technically shown but the session ended. Consider adding a short fulfillment delay for digital orders with post-purchase upsell active.

Inventory race condition

Upsell product goes OOS between checkout and acceptance

The upsell page loads when the original order is confirmed, but the upsell product's inventory is not reserved at that point. If the upsell product has limited stock and another customer purchases it in the window between when the upsell page loads and when this customer accepts, the acceptance charge may fail with an inventory error. This is rare but occurs during flash sales.

Subscription product upsell

Cannot add subscription to existing order

Subscription products (managed by Recharge, Seal Subscriptions, etc.) generally cannot be added to an order via post-purchase one-click upsell. The subscription billing relationship needs to be set up separately. Offering a subscription product as a post-purchase upsell will either fail at the charge step or create a standalone subscription order rather than adding to the original.

Order editing conflict

Upsell accepted but creates separate order

Depending on the app configuration and whether the original order is editable (based on fulfillment status, payment status, and app settings), the upsell acceptance may either edit the original order (adding a line item) or create a second order. Tax calculations, shipping, and customer communications differ between these two approaches. Review how your app handles accepted upsells in its settings.

Frequently Asked Questions

Why is my Shopify post-purchase upsell not showing after checkout?

The most common reasons are: (1) the app is not assigned in Shopify Admin → Settings → Checkout → Post-purchase page; (2) the funnel is not set to Active; (3) trigger conditions don't match the order; (4) A/B testing is enabled and the customer was placed in the control group; (5) a payment gateway that doesn't pass the necessary checkout session token to Shopify's extension system.

Why is the one-click upsell charge failing?

One-click post-purchase charges require a reusable payment token from the original checkout. Many payment methods don't produce one: PayPal standard checkout, cash on delivery, bank transfer, most BNPL methods (Klarna, Afterpay), and orders paid entirely with gift cards. The one-click mechanism is architecturally impossible with these payment methods regardless of how the app is configured.

Does post-purchase upsell work with PayPal?

Standard PayPal checkout does not support post-purchase one-click upsell because it returns a single-use authorization token that cannot be recharged. PayPal Billing Agreements are a separate product that does support recharging, but it requires specific integration and is not available in all markets. Most Shopify merchants using standard PayPal checkout cannot offer one-click upsells to PayPal customers.

ReConvert is active but upsells aren't showing — what's wrong?

Beyond the app being installed and the funnel being published, ReConvert requires a separate step: go to Shopify Admin → Settings → Checkout → Post-purchase page and select ReConvert from the dropdown. Without this step, the funnel cannot appear. Also check: trigger filter mismatch (the purchased product must match the funnel trigger), A/B test configuration, and whether the upsell product is in stock.

My post-purchase upsell stopped working after a Shopify update — why?

If the app was using checkout.liquid scripts for its implementation, those scripts are no longer executed after Shopify's checkout extensibility migration. Shopify deprecated checkout.liquid for new stores in 2021 and completed its removal for existing stores by 2025. Apps that have migrated to the official Post-Purchase Checkout Extension API are not affected. If your app hasn't migrated, you need to switch to one that has (ReConvert, AfterSell, Zipify OCU).

CartHook post-purchase upsell stopped working — what's the replacement?

CartHook's post-purchase upsell was sunset in 2023 and is permanently non-functional. The main replacements are ReConvert (most feature-complete), AfterSell (simpler setup), and Zipify OCU (best for Shopify Plus merchants with high-volume testing). All three use Shopify's official Post-Purchase Checkout Extension API.
