Shopify Local Pickup Not Working — In-Store Pickup Not Showing at Checkout

Local pickup disappears from checkout when the location isn't enabled for pickup, the product has no stock there, or express checkout buttons bypass the selection entirely. Here is every failure mode and fix.

📅 Updated June 2026 ⏱ 8 min read 🏪 Applies to: Shopify Basic, Shopify, Advanced, Plus
Most Common Cause
Local pickup requires two things to be true simultaneously: (1) the pickup location must have the "Local pickup" toggle enabled in Shopify Admin → Settings → Shipping and delivery → Local pickup → Manage, and (2) the product must have stock assigned to that specific location. If either is false, Shopify silently hides the pickup option at checkout.

How Shopify Local Pickup Works (and Where It Breaks)

Shopify local pickup is a first-party feature that lets customers choose to collect their order from your store instead of having it shipped. It appears as a delivery method option at checkout, separate from shipping rates.

The feature has a specific activation path: Shopify Admin → Settings → Shipping and delivery → scroll to "Local pickup" → click "Manage" → toggle "Local pickup" ON for each location. This toggle is per-location — enabling it at your main store does not automatically enable it for a second warehouse or pop-up location.

Once enabled, Shopify checks two things in real time during checkout to decide whether to show the pickup option:

  1. Is the pickup toggle on for at least one active location?
  2. Does that location have stock available for every item in the cart?

If either check fails for all eligible locations, the pickup option simply does not appear — no error message, no explanation. The customer sees only shipping options.

Failure Mode 1 — Local Pickup Toggle Not Enabled

The most common cause: the pickup option was never activated, or it was accidentally turned off. The setup path is non-obvious — "Local pickup" is buried inside the Shipping and delivery settings, not in the Location settings.

How to enable local pickup

  1. Go to Shopify Admin → Settings → Shipping and delivery
  2. Scroll down past your shipping zones to the "Local pickup" section
  3. Click "Manage" next to the location you want to enable
  4. Toggle "Local pickup" to ON
  5. Optionally set an expected pickup time (e.g., "Usually ready in 2 hours") — this shows in checkout
  6. Click Save
  7. Repeat for each additional pickup location
Tip
After enabling, test by adding a product to cart and going through checkout in a fresh incognito window. If you're logged in as admin, some checkout behaviors differ. Test as a guest customer.

Failure Mode 2 — Product Has No Stock at the Pickup Location

Even with the toggle on, local pickup only shows if the product being ordered has available stock at the pickup location. If inventory is zero at that location — or if inventory tracking is on and the stock count is 0 — Shopify hides the pickup option.

How to check and fix inventory at a location

  1. Go to Shopify Admin → Products → [select the product]
  2. Scroll down to the "Inventory" section
  3. Look at "Inventory at each location" — find the pickup location row
  4. Check if the quantity is 0 or if the location is not listed
  5. If quantity is 0: update the stock count or disable inventory tracking for that product if stock tracking isn't needed
  6. If the location isn't listed: go to Shopify Admin → Settings → Locations → confirm the location is active, then return to the product and assign stock
Variant-Specific Issue
For products with multiple variants (size, color), each variant's inventory is tracked separately. If a customer selects a variant that has 0 stock at the pickup location, the pickup option won't show — even if other variants are in stock there. Check inventory for the specific variant being tested.

Failure Mode 3 — Express Checkout Buttons Bypass Local Pickup

This catches many merchants off guard. Shop Pay, Apple Pay, Google Pay, and other express checkout buttons do not show the local pickup option. These buttons launch a streamlined one-step checkout flow that skips the delivery method selection screen and defaults to shipping.

Checkout Method Shows Local Pickup Notes
Standard "Check out" button Yes Full checkout flow — pickup option shows if eligible
Shop Pay express No Skips delivery method screen, defaults to shipping
Apple Pay No Payment sheet doesn't include pickup selection
Google Pay No Same as Apple Pay — shipping only
"Buy now" product page button No Routes to expedited checkout without delivery choice
PayPal express No PayPal-hosted flow, no pickup support
Shop Pay installments No BNPL flow doesn't support pickup

Workarounds for express checkout bypass

  • Add messaging on cart/product pages: "Want to pick up in store? Use the 'Check out' button — express pay buttons don't support pickup."
  • Disable dynamic checkout buttons on product pages: In Shopify Admin → Settings → Checkout → Dynamic checkout → disable "Show dynamic checkout buttons on product pages" if pickup is a primary delivery method for your store.
  • Checkout UI extension: For Shopify Plus, a custom checkout UI extension can add messaging at checkout reminding customers that express methods skip pickup.

Failure Mode 4 — Local Pickup vs Local Delivery Confusion

Shopify has two distinct features that sound similar but are configured completely differently:

Feature What It Does Location in Admin Distance Filtering
Local pickup Customer collects order from your location Settings → Shipping and delivery → Local pickup None — any customer can select it
Local delivery You deliver to customer within a defined zone Settings → Shipping and delivery → Local delivery Yes — radius or postal codes

If you want customers nearby to get a "local pickup" option but not customers far away, Shopify's native local pickup doesn't support distance filtering. Any customer anywhere in the world can technically select local pickup if it's enabled (though ordering pickup from another country is obviously impractical).

To restrict who can choose pickup by location, you need a third-party app like Zapiet - Pickup + Delivery or Store Pickup + Delivery (BOPIS) from the Shopify App Store.

Common Mistake
Merchants trying to set up local pickup sometimes go to Settings → Locations instead of Settings → Shipping and delivery. The Locations page manages where your inventory is tracked — it does NOT have a "local pickup" toggle. The pickup activation is only in the Shipping and delivery section.

Failure Mode 5 — Pickup Order Gets Shipped Instead of Picked Up

The customer chose local pickup, the order came in correctly, but your team (or an automation) processed it as a shipment instead.

The correct pickup fulfillment flow

  1. Order arrives — it shows "Local pickup" as the delivery method in the order details
  2. Prepare the order in your store/warehouse
  3. In Shopify Admin → Orders → open the order
  4. Click "Mark as ready for pickup" (not "Fulfill" or "Create shipping label")
  5. Shopify sends the customer a "Ready for pickup" email with your store address and hours
  6. When customer collects the order, click "Mark as picked up" to close the order
Automation Risk
If you use Shopify Flow, a fulfillment app, or auto-fulfillment settings that automatically create shipping labels for all new orders, these may incorrectly process pickup orders as shipped. Add a condition to your automation: order.delivery_method = "local" → skip auto-fulfillment.

Failure Mode 6 — Pickup-Ready Email Not Sent

The "Ready for pickup" email is not automatic — it fires only when you click "Mark as ready for pickup" in the order. If the email isn't reaching customers:

  • Verify the notification is enabled: Shopify Admin → Settings → Notifications → find "Ready for pickup" and confirm it's active and the template exists.
  • Check the customer's email address on the order — typos during checkout are common.
  • Spam/junk folder: Shopify sends from a shared sending infrastructure; some corporate email servers aggressive-filter. Ask the customer to check spam.
  • Third-party email override: If you use Klaviyo, Omnisend, or another ESP to manage all Shopify transactional emails, check whether your integration is correctly routing the "Ready for pickup" trigger. Some setups only route order confirmation and shipping emails, missing the pickup-ready trigger.
  • You clicked "Fulfill" instead of "Mark as ready for pickup": This sends a shipping confirmation email with a tracking number, not a pickup-ready email. Refund the shipping if added and manually contact the customer.

Failure Mode 7 — App Conflict Hiding Local Pickup

Carrier-calculated shipping rate apps and multi-location fulfillment apps can override Shopify's native shipping rate calculation, which is also responsible for returning the local pickup option.

How to diagnose an app conflict

  1. Go to Shopify Admin → Apps → find recently installed or updated apps that relate to shipping, fulfillment, or inventory
  2. Temporarily disable the most recently changed app
  3. Test checkout in incognito — does pickup appear now?
  4. If yes, the disabled app is the cause — review that app's shipping settings to add a "pass through Shopify native rates" or "preserve local pickup" option
  5. Common offenders: ShipStation, EasyShip, Shipify, multi-location inventory apps that remap fulfillment locations
Shopify Plus — Checkout Extensibility
If you've built checkout UI extensions or added checkout customizations on Shopify Plus, a poorly configured extension can hide the delivery method selector or filter out pickup options. Check Shopify Admin → Settings → Checkout → Checkout customizations and disable each extension one at a time to isolate the conflict.

Failure Mode 8 — Multiple Locations, Wrong Location Fulfills

If you have multiple locations, Shopify uses fulfillment priority to decide which location ships an order. For pickup orders, the customer selects a specific location — but a fulfillment app may route the order to a different location for packing, causing inventory discrepancies and operational confusion.

Controlling fulfillment location for pickup orders

  • Shopify automatically assigns pickup orders to the location the customer selected — this is preserved in the order data as the "fulfillment location."
  • Manual fulfillment (your staff handling it): this is fine, just make sure staff fulfill from the correct location.
  • Automated fulfillment via an app: configure the app to honor the order's pre-assigned fulfillment location rather than re-routing based on its own priority rules.
  • For Shopify Plus: use Shopify Flow to add a condition — if fulfillment_type == local, skip the app's auto-routing and assign directly to the location already set on the order.
