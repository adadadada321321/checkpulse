Shopify Draft Order Not Working — Payment Link Broken, Discount Missing, Tax Wrong

Shopify draft orders are a completely separate checkout path from the regular storefront flow. The payment link breaks the moment a draft is archived or already paid. Customers cannot enter discount codes. Tax doesn't auto-recalculate. Carrier shipping rates don't populate. Here is the complete diagnostic for every draft order failure mode.

📅 Updated June 2026 ⏱ 9 min read 💡 Applies to: All Shopify plans
Critical Architecture Fact
Draft order checkout is not the regular Shopify checkout. It has different behavior for discounts, shipping, tax, inventory, and abandoned checkout. Understanding these differences prevents the majority of support tickets around draft orders.

How Shopify Draft Orders Work (and Where They Break)

A Shopify draft order is a manually created order — built by the merchant in Shopify Admin or via the Admin API rather than by a customer browsing the storefront. The merchant adds products, applies any discounts, sets shipping and tax, assigns a customer, and then sends an invoice email. The customer receives a link to pay.

What most merchants don't realize about draft orders:

  • The invoice link only works while the draft is in "Open" status — archiving or deleting it immediately breaks the link
  • Customers cannot apply discount codes at draft order checkout — discounts must be set by the merchant at draft creation
  • Tax is not live-calculated — it must be manually recalculated before sending the invoice
  • Carrier shipping rates do not auto-populate — the merchant must enter a flat shipping amount manually
  • Draft orders do not reserve inventory — stock can sell out between draft creation and customer payment
  • Abandoned draft order checkouts do not trigger Shopify's standard recovery emails

Failure Mode 1 — Payment Link Not Working or Broken

The draft order invoice link stops working in four scenarios. Each has a different resolution path.

Draft Status Invoice Link Status Resolution
Open Working Customer can pay via the link
Completed / Paid Broken — already paid Check the order in admin — it was paid. No action needed unless there's an error.
Archived Broken — archived Go to Orders → Drafts → find the archived draft → click "Unarchive"
Deleted 404 — permanently gone Create a new draft order and send a new invoice

How to find a draft order's current status

  1. Go to Shopify Admin → Orders → Drafts
  2. Look for the draft in the "Open" tab — if it's not there, check the "Archived" tab
  3. If the draft shows "Completed" it was already paid — find the resulting order in Orders → All orders
  4. If you can't find it anywhere, it was deleted — you'll need to create a new one
Resending the Invoice
You can resend the draft order invoice at any time from the draft order page. The resent email contains the same checkout link (as long as the draft is still Open). To resend: open the draft → click "Send invoice" → confirm. The customer receives a fresh email with the same payment link.

Customer can't pay — wrong account logged in

If a draft order is assigned to a specific customer email address, the payment link may behave differently depending on whether the customer is logged into Shopify with a matching account. The customer does not need to be logged in to pay — the link works for anyone who has it. However, if the customer is experiencing an error message about account access, have them try the link in a private/incognito browser window without being logged in to any Shopify/Shop account.

Failure Mode 2 — Customer Can't Apply a Discount Code

This is the most common confusion around draft order checkout. There is no discount code field in the draft order checkout flow. A customer who receives a draft order invoice link and tries to enter a promo code will find no place to do so.

By Design — No Customer-Side Discounts
Shopify intentionally does not allow customers to apply discount codes at draft order checkout. Automatic discounts configured in Shopify Admin → Discounts also do not apply to draft orders. The only discounts that work are those the merchant explicitly adds to the draft before sending the invoice.

How to add a discount to a draft order

  1. Open the draft order in Shopify Admin → Orders → Drafts
  2. To discount the whole order: click "Add discount" below the line items → choose percentage or fixed amount
  3. To discount a specific line item: click the line item → edit the price directly or click the discount icon next to the item
  4. After adding the discount, recalculate tax (see Failure Mode 3 below)
  5. Click "Save" — then resend the invoice so the customer gets the updated amount
Resend the Invoice After Editing
If you add a discount to an existing draft order, the original invoice link the customer received still shows the old amount in the email body. The link itself will load the updated (discounted) total — but to avoid confusion, always resend the invoice after making changes so the customer sees the correct amount in the email.

Failure Mode 3 — Tax Amount Is Wrong

Shopify calculates tax on a draft order at the time the draft is created or last saved. It does not automatically recalculate when you change line items, update the customer's shipping address, or when tax rates change on the Shopify platform.

When does tax go wrong on a draft order

  • You added or removed products after the initial draft creation
  • The customer's shipping address was updated or corrected
  • The draft was created for a customer in one tax jurisdiction but will ship to another
  • Tax rates changed (e.g., a US state rate update, a new EU VAT rate)
  • You applied a discount but didn't recalculate tax afterward
  • The draft was created before a nexus was added for a new state/country

How to recalculate tax on a draft order

  1. Open the draft order in Shopify Admin → Orders → Drafts
  2. Confirm the shipping address is correct — tax is calculated based on this address
  3. Scroll to the Taxes section
  4. Click "Recalculate" — Shopify will recalculate based on current rates and the shipping address
  5. If you don't see a Recalculate button, make any small edit (like adding/removing a note) and the option should appear
  6. Click "Save" and resend the invoice
Tax-Exempt Customers and Draft Orders
If a customer is marked as tax-exempt in Shopify Admin → Customers → [customer] → Tax exemptions, that exemption will apply when you create a draft order assigned to that customer. However, if you reassign the draft or the exemption is added after draft creation, you may need to manually toggle the "Charge taxes" option on the draft and recalculate.

Failure Mode 4 — Shipping Rate Not Showing or Wrong

Draft order checkout does not pull carrier-calculated shipping rates. The shipping amount is set by the merchant when creating the draft and is presented to the customer as a fixed line item — the customer cannot choose between shipping options at draft order checkout.

Checkout Type Shipping Selection Carrier Rates
Regular storefront checkout Customer selects Auto-calculated (if carrier rates enabled)
Draft order checkout Merchant sets Not available — flat rate only

How to add shipping to a draft order

  1. In the draft order, click "Add shipping" below the line items and tax section
  2. Enter a flat shipping rate manually (e.g., $9.99 for Standard Shipping)
  3. You can label it with the carrier and service name for clarity (e.g., "UPS Ground")
  4. Click "Apply" — the shipping amount appears as a line item in the draft
  5. Recalculate tax if your store charges tax on shipping, then save and resend the invoice
No Shipping = $0 Shipping Charged
If you forget to add a shipping line to a draft order, the customer will be charged $0 for shipping when they pay. Shopify does not warn you about this. Always check that the shipping line is present before sending the invoice.

Failure Mode 5 — Inventory Not Reserved

Creating a draft order does not decrement inventory or place a hold on stock. Between the time you create the draft and the time the customer pays, those products are freely available for other customers to purchase via the regular storefront.

Scenarios where this causes problems:

  • You create a custom draft for a B2B client with limited inventory — another customer buys out the stock before the client pays
  • Flash sale situation: a customer agrees to a price via email, you create a draft, but the item sells out before they click the link
  • Draft sits unpaid for several days — stock situation changes

Options for managing inventory with draft orders

Approach How Trade-off
Manual inventory adjustment Reduce available inventory in admin for the draft quantity before creating the draft Simple but requires remembering to restore if draft is cancelled
Process payment immediately For phone/in-person orders, mark draft as paid manually in admin right away Eliminates the window of exposure but requires synchronous payment
Draft order app with inventory hold Third-party apps can place temporary holds (varies by app) Adds app complexity and cost
Continue selling & oversell Accept that occasional oversells may happen; handle manually Acceptable for low-volume custom orders with reliable customers

Failure Mode 6 — No Abandoned Checkout Recovery for Draft Orders

Shopify's built-in abandoned checkout recovery system — which automatically emails customers who reach checkout but don't complete payment — does not apply to draft order checkouts. If a customer clicks the invoice link, enters their details, but closes the browser before paying, Shopify does not send an automatic follow-up email.

Options for following up on unpaid draft orders

  1. Monitor open draft orders in Shopify Admin → Orders → Drafts and follow up manually
  2. Resend the invoice email from the draft — customers receive a fresh notification with the payment link
  3. Use a Shopify Flow automation to notify your team when a draft order has been open for X days without payment
  4. Configure Klaviyo, Omnisend, or similar tools to send follow-up sequences for unpaid draft orders via the Admin API webhook on draft_orders/create and checking payment status
Draft Order Invoice Link Does Not Expire
Unlike some order management systems, Shopify draft order invoice links do not expire after a fixed time period. The link remains active indefinitely as long as the draft order stays in "Open" status. You can send a customer an invoice link today, and if they haven't paid in 30 days, you can resend it and the same link will still work (as long as you haven't archived the draft). The only way the link stops working is if you archive, complete, or delete the draft.

Failure Mode 7 — Draft Order Payment Methods Missing or Incomplete

Draft order checkout shows a subset of the payment methods available on the regular storefront. Key differences:

  • Shop Pay Installments / BNPL: Not reliably available at draft order checkout. Klarna, Afterpay, and Affirm integrations may not appear for draft order customers depending on the gateway configuration.
  • Manual payment methods: The merchant can mark a draft order as paid by cash, money order, bank transfer, or other manual methods directly in admin without the customer going through the checkout link at all. This is useful for phone orders and in-person sales.
  • Shop Pay wallet: Available when the customer's email matches a Shop account, but behavior can vary.
  • Standard card payments: Consistently available via Shopify Payments or whichever payment gateway is configured.

How to mark a draft order as paid manually (without customer checkout)

  1. Open the draft order in Shopify Admin → Orders → Drafts
  2. Click "Mark as paid" at the top of the page
  3. Select the payment method (cash, money order, credit card manually entered, etc.)
  4. Enter the transaction amount and any reference number
  5. Click "Collect payment" — the draft converts to a completed order immediately
Best Practice: Draft Order vs Regular Checkout
Use draft orders for: custom quotes, wholesale orders, phone orders, orders requiring manual discounts or custom pricing, or situations where you want to collect payment outside the normal storefront flow. Use the regular checkout for all standard storefront purchases — it has better payment method coverage, real-time carrier rates, abandoned checkout recovery, and inventory holds.
