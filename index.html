FAQ

Common questions

How does CheckPulse simulate a checkout without buying anything?
We replay the HTTP request sequence a real checkout follows — cart creation, checkout session init, and payment methods fetch. No inventory is reserved, no payment is processed, no order is created. We check that your store responds correctly at each step without completing a transaction.
Do I need to install an app or give API access?
No. CheckPulse works by sending standard HTTP requests to your store's public checkout endpoints — the same ones any shopper's browser hits. No app installs, no API tokens, no code changes. Just your Shopify domain and an email address.
How quickly will I get my first check?
Within 60 minutes of setup. Our monitor runs on the hour, every hour, 24/7. You'll get a confirmation email when monitoring starts, and an alert within the hour if anything is broken.
What kinds of failures does it actually catch?
Shopify Payments suspensions, 3DS redirect failures, cart-to-checkout redirect errors, payment gateway misconfigurations, checkout app conflicts (gift cards, discounts, local delivery restrictions), and any HTTP 4xx/5xx at any checkout step. If a shopper can't complete a purchase, CheckPulse will catch it.
Does it work with Shopify Plus?
Yes. CheckPulse checks the standard Shopify checkout flow, which is identical on Plus and non-Plus stores. Custom checkout UI (via Shopify Functions or Checkout Extensibility) is on our roadmap.
Is my store or customer data at risk?
No. CheckPulse only reads public checkout endpoints — the same HTTP calls any visitor makes. We never access your Shopify admin, customer records, or payment information. No credentials are ever stored.
