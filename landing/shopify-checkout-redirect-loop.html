Shopify Checkout Redirect Loop?
6 Causes & How to Fix Each

Updated June 2026  ·  8 min read  ·  By CheckPulse Team

A Shopify checkout redirect loop means every customer who tries to complete a purchase gets bounced back to an earlier step — or cycles between pages indefinitely. No order goes through. Unlike a slow checkout or a confusing layout, a redirect loop is a complete hard stop: zero conversions until it's fixed.

Redirect loops at checkout are more common than people expect, they often go undetected for hours, and they're caused by a specific set of identifiable problems — most of which can be fixed in under 30 minutes once you know which one you're dealing with.

⚠ Why redirect loops are dangerous

Standard uptime monitoring won't catch a checkout redirect loop — your store's homepage still loads fine and returns HTTP 200. The only way to detect a redirect loop before customers report it is to run an automated end-to-end checkout test that actually attempts the checkout flow.

CAUSE 01 Third-Party Cookie Blocking

Shopify's checkout runs on a separate subdomain (checkout.shopify.com or your store's custom checkout subdomain). When a customer browses your store on your main domain and then proceeds to checkout, Shopify needs to transfer the session using a cookie that spans domains. If the browser blocks third-party cookies — as Safari does by default via Intelligent Tracking Prevention (ITP), and as Chrome increasingly does — the session cannot be transferred and the customer gets looped back to the cart or the main storefront.

Symptoms:

  • Redirect loop occurs primarily for Safari users or users with enhanced privacy settings
  • Chrome in Incognito mode also reproduces it (third-party cookies are blocked by default in Incognito)
  • Checkout works fine in Firefox or a browser with cookies fully enabled
  • Loop sends customer back to cart page or store homepage, not an error page
▶ How to fix
  • Connect a custom domain to your Shopify store if you haven't already — this allows Shopify to set same-site cookies and eliminates the cross-domain cookie problem
  • In Shopify Admin → Online Store → Domains, ensure your custom domain is set as the primary domain
  • If using Shopify Plus: configure a custom checkout subdomain (e.g., checkout.yourdomain.com) so checkout and storefront share the same first-party domain context
  • Check that your Shopify theme doesn't include any scripts that modify the checkout URL or session token before redirect
  • Test immediately in Safari on iOS (Settings → Safari → Privacy → disable "Prevent Cross-Site Tracking") to confirm this is the issue
Impact scale

Safari holds ~37% of mobile browser market share globally. If cookie blocking is causing a redirect loop for Safari users, you may be losing over a third of your mobile checkout conversions silently.

CAUSE 02 Custom Domain or SSL Misconfiguration

If your custom domain is misconfigured — wrong DNS records, mixed HTTP/HTTPS, or a recently changed domain that hasn't fully propagated — Shopify may loop between the custom domain and the default myshopify.com domain while trying to resolve the correct URL. This is especially common after migrating domains, renewing SSL certificates, or changing domain registrars.

Symptoms:

  • Browser shows "too many redirects" error (ERR_TOO_MANY_REDIRECTS)
  • Redirect loop is reproducible for all users, not just specific browsers
  • Issue started recently after a domain or DNS change
  • Curl shows alternating redirects between custom domain and myshopify.com
// Browser console — redirect chain
GET https://yourdomain.com/checkout → 301
GET https://yourstore.myshopify.com/checkout → 301
GET https://yourdomain.com/checkout → 301
ERR_TOO_MANY_REDIRECTS
▶ How to fix
  • In Shopify Admin → Online Store → Domains, verify the primary domain is set correctly and shows "Connected" status with a green checkmark
  • Check DNS records with a tool like dnschecker.org — the A record for your domain should point to Shopify's IP (23.227.38.65), and the CNAME for www should point to shops.myshopify.com
  • Remove any redirect rules at your domain registrar or DNS host that conflict with Shopify's own redirects
  • If you recently changed domains: old domain redirects can conflict with new domain setup — remove 301 redirects from the old domain or from Cloudflare/registrar settings
  • Clear browser cache and test in an Incognito window — cached redirects can persist even after the underlying issue is fixed
  • If using Cloudflare: ensure "Flexible" SSL mode is NOT enabled (use "Full" or "Full Strict") — Flexible SSL with Shopify creates a redirect loop because Shopify forces HTTPS on its end
CAUSE 03 CDN or Reverse Proxy Redirect Rules

If your store is behind a CDN (Cloudflare, Fastly, Akamai) or a reverse proxy, redirect rules defined at the CDN layer can conflict with Shopify's own checkout redirects. This is a particularly insidious cause because the issue lives outside Shopify's control — it won't appear in Shopify logs, and Shopify support won't be able to identify it.

Symptoms:

  • Redirect loop for all users, all browsers
  • Shopify admin logs show no errors on the checkout steps
  • Issue exists on custom domain but not on .myshopify.com fallback URL
  • Recently added or modified a CDN, page rule, or redirect rule in Cloudflare/Fastly
▶ How to fix
  • In Cloudflare: check Page Rules, Redirect Rules, and Transform Rules for any rule matching /checkout* or your domain — disable them temporarily to test
  • Verify Cloudflare SSL mode is set to "Full" or "Full (strict)" — never "Flexible" (causes redirect loop with Shopify)
  • If using Cloudflare Workers: check for any worker assigned to your checkout path that modifies the response or issues a redirect
  • Test checkout directly on your yourstore.myshopify.com URL — if checkout works there but not on your custom domain, the problem is in your CDN/proxy layer, not Shopify
  • Review Cloudflare Security settings: "Under Attack Mode" or aggressive bot protection rules can block Shopify's checkout session handoff
Quick isolation test

Temporarily pause Cloudflare (set to "DNS only" instead of "Proxied" on the A record). If checkout works immediately after, the problem is in your Cloudflare configuration.

CAUSE 04 Checkout App Injecting Conflicting Redirects

Apps that modify or hook into the checkout flow — upsell apps, post-purchase offer apps, loyalty apps, checkout customization apps — can inject redirect logic that conflicts with Shopify's own checkout redirect chain. This is more common after Shopify's Checkout Extensibility migration, which deprecated the older ScriptTag and Liquid-based checkout customizations and broke many older apps in the process.

Symptoms:

  • Redirect loop started after installing a new app or after a Shopify platform update
  • Loop occurs at a specific checkout step (e.g., after entering shipping address, or at the payment step)
  • Disabling all checkout-modifying apps stops the loop
  • App changelog or review section mentions "redirect issues" after a recent update
▶ How to fix
  • Temporarily disable all apps that modify checkout — test checkout without them to confirm the loop stops
  • Re-enable apps one at a time to identify which specific app triggers the loop
  • Check if the offending app has been updated recently — if so, the developer may have introduced a bug; contact their support with your store URL and the loop timestamp
  • Verify apps have migrated to Shopify's Checkout UI Extensions (replacing old ScriptTag approach) — outdated apps using deprecated APIs can cause redirect issues on newer Shopify stores
  • In Shopify Admin → Settings → Checkout → Customizations, review all active checkout UI extensions and remove any that aren't in active use
CAUSE 05 Login-Required Setting Loop

Shopify has a setting that controls whether customers must create an account or log in to complete a purchase. When this setting is misconfigured — or when a password-protected store attempts to send customers to checkout before authentication — Shopify loops the customer between the login page and the checkout page indefinitely, because neither step can complete without the other.

Symptoms:

  • Customer is bounced between a login/account page and the checkout
  • Guest checkout is not offered despite the customer not being logged in
  • Store is in password-protected mode (development or pre-launch) and checkout links are being shared externally
  • Redirect loop only affects specific customer types (non-logged-in customers, or B2B customers with required accounts)
▶ How to fix
  • Go to Shopify Admin → Settings → Checkout → Customer accounts. Set to "Guest checkout" (or "Accounts are optional") unless you specifically require account login
  • If your store is password-protected: check Online Store → Preferences → Password protection — customers cannot access checkout when a password is active on the storefront
  • For B2B stores with required login: verify the customer has an active account and is properly authenticated before sending checkout links — unauthenticated B2B checkout links always loop
  • If using Shopify Plus B2B: ensure company contacts are properly assigned and the checkout permissions are configured for the relevant price list
CAUSE 06 Shopify Markets Region Redirect Loop

Shopify Markets automatically redirects customers to region-specific versions of your store based on IP geolocation. When the geolocation data conflicts with the market configuration — or when a customer is in a region that has inconsistent redirect rules — the storefront and checkout can loop between region URLs indefinitely. This became more common after the 2024 Shopify Markets expansion added more granular region controls.

Symptoms:

  • Redirect loop affects customers in specific countries or regions
  • Domestic customers (in your primary market) check out fine; international customers loop
  • URLs in the redirect chain contain country/language codes (e.g., /en-gb/, /fr/)
  • Issue started after configuring or updating Shopify Markets
▶ How to fix
  • In Shopify Admin → Markets, review the region assignments — ensure no region is assigned to both the primary market and a secondary market simultaneously
  • Check that the "auto-redirect" setting in Markets matches your intent — if you want customers to choose their region manually, disable auto-redirect and add a country/language selector to your theme
  • Review any URL redirect rules in Online Store → Navigation → URL Redirects for conflicting /en-*/ or country-prefix redirects that may conflict with Markets' own routing
  • Test with a VPN set to affected country — reproduce the loop exactly before and after making changes to confirm the fix
  • If using a third-party geolocation or currency app alongside Shopify Markets: check for conflicts — some apps and native Markets routing issue competing redirects

Quick Diagnostic Checklist

  • Does the loop affect Safari users more than Chrome? (→ cookie blocking)
  • Does the loop affect all browsers equally? (→ domain, CDN, or app issue)
  • Does checkout work on yourstore.myshopify.com? (→ CDN/proxy issue)
  • Did the loop start after a DNS, domain, or Cloudflare change? (→ domain mismatch)
  • Did the loop start after installing a new app? (→ app conflict)
  • Does the loop affect only international customers? (→ Markets region issue)
  • Are non-logged-in customers being looped to a login page? (→ login setting)

Frequently Asked Questions

Why does my Shopify checkout keep saying "too many redirects"?

ERR_TOO_MANY_REDIRECTS usually means two systems are issuing competing redirects — most commonly Cloudflare with "Flexible" SSL mode conflicting with Shopify's forced HTTPS, or a CDN redirect rule conflicting with Shopify's checkout routing. Set Cloudflare SSL to "Full" and check for any page rules targeting your checkout URL.

Why does checkout redirect loop happen only on Safari?

Safari's Intelligent Tracking Prevention (ITP) blocks third-party cookies by default. Shopify's checkout session transfer between your storefront domain and the checkout subdomain relies on cross-domain cookies. The fix is to connect a custom domain and ensure both storefront and checkout run under the same first-party domain context.

My checkout works on myshopify.com but not my custom domain — why?

This points to a CDN, DNS, or proxy configuration issue on your custom domain. The most common cause is Cloudflare's SSL mode set to "Flexible" instead of "Full." Check your Cloudflare SSL settings, then review any Page Rules or Redirect Rules that target your checkout path.

How do I stop Shopify Markets from causing a redirect loop?

Go to Shopify Admin → Markets and review region assignments. Ensure no country appears in more than one market. Then check Online Store → Navigation → URL Redirects for any manually-created redirects that conflict with Markets' automatic country routing. If the issue persists, temporarily disable auto-redirect in Markets settings.

How do I detect a checkout redirect loop before customers report it?

Standard uptime monitors can't detect checkout redirect loops — they only check if your homepage loads. You need an automated checkout monitor that actually attempts the checkout flow end-to-end: add to cart, enter address, attempt payment. If a redirect loop breaks any step, you'll get an alert within minutes instead of finding out hours later from an abandoned cart surge.

Can a Shopify app cause a redirect loop at checkout?

Yes. Apps that use checkout UI extensions, post-purchase flows, or custom redirect logic can inject conflicting redirects. If a redirect loop appeared after installing or updating an app, disable that app temporarily to confirm. The most common offenders are upsell apps, checkout customization apps, and loyalty integrations that haven't fully migrated to Shopify's Checkout Extensibility platform.
