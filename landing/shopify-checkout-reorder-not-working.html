Shopify Reorder Not Working — Button Missing, Wrong Items, No Discount Restored

Shopify's "Reorder" button silently skips out-of-stock items, ignores deleted or discontinued variants, and never re-applies discount codes from the original order. Guest orders have no reorder button at all. Here is the complete diagnostic for every reorder failure mode.

📅 Updated June 2026 ⏱ 9 min read 💡 Applies to: All Shopify plans
Silent Failure Warning
Shopify's reorder feature has no error state visible to customers. When items can't be added — because they're out of stock, discontinued, or deleted — the button completes silently and the customer lands on a cart with fewer items than expected and no explanation.

How Shopify Reorder Works (and Where It Breaks)

When a customer clicks the "Reorder" button on their account order history page, Shopify reads the variant IDs and quantities from the original order and calls cartAdd for each line item. That's it — no discount restoration, no stock check with a warning, no variant validation before the call.

What most merchants don't know about Shopify reorder:

  • Reorder only exists on the customer account order history page — guest orders have no reorder button
  • Out-of-stock items are silently dropped — no error, no list of skipped items
  • Deleted or discontinued variants add nothing — the reorder silently fails for those line items
  • Discount codes from the original order are never re-applied — items are added at current full prices
  • Subscription products reorder incorrectly — they add as one-time purchases without the subscription plan
  • Switching to new customer accounts breaks any third-party reorder app built for legacy accounts

Failure Mode 1 — Reorder Button Missing from Account

The reorder button doesn't appear for every customer or every order. Multiple conditions can hide it.

Situation Reorder Button Why
Logged-in customer, paid fulfilled order Shown Standard case — works as expected
Guest checkout order Not available Guest orders have no customer account link; no order history page
Order pending payment Not shown Unfulfilled/pending orders don't trigger the reorder UI
New customer accounts (2023+) Changed behavior New account UI is Shopify-hosted; third-party reorder injections no longer work
Theme setting disabled Hidden Some themes allow toggling the reorder button off in the theme editor

How to restore the reorder button

  1. Go to Shopify Admin → Online Store → Themes → Customize
  2. Navigate to the Account or Order history section in the theme editor
  3. Look for a "Show reorder button" or similar toggle — enable it if present
  4. If you use new customer accounts, check Settings → Customer accounts — reorder is built into the Shopify-hosted UI and cannot be further customized via Liquid
  5. If using a third-party reorder app, verify it is compatible with your current account type (legacy vs new)

Failure Mode 2 — Reorder Silently Skips Out-of-Stock Items

This is the most common support complaint: "I clicked reorder and some items are missing from my cart." There is no visible error. No list of skipped items. The customer simply lands on a cart that is incomplete.

When cartAdd is called for a variant that has zero inventory and "Continue selling when out of stock" is disabled, Shopify does not add that item and does not surface any failure to the customer interface. The reorder call succeeds from Shopify's perspective — it just added zero items for that line.

Diagnosing which items were skipped

  1. Open the original order in Shopify Admin → Orders and note all line items
  2. Compare against what appeared in the cart after reorder — the missing items are the skipped ones
  3. For each missing item, go to Products → [product] → [variant] and check inventory at the fulfillment location
  4. If inventory is 0 and "Continue selling when out of stock" is unchecked, this is the cause
  5. Either restock the item, enable "Continue selling when out of stock" on that variant, or notify the customer via email that the item is unavailable
Customer Experience Impact
A customer who places a reorder, goes through checkout, and pays — then later realizes items are missing — will submit a support ticket and likely request a partial refund. The silent skip is the root cause. Consider adding a post-reorder message or using a third-party app that surfaces which items weren't added.

Failure Mode 3 — Reorder Adds Wrong Items (Discontinued Variants)

If a variant from the original order has been deleted or its ID has changed, the reorder call for that variant either fails silently or adds a different, incorrect item.

Common scenarios that create this problem:

  • Variant deleted — you removed "Size XS" from a product because you no longer carry it; customers who ordered XS get nothing added when they reorder
  • Product replaced — you deleted an old product and created a new one with new variant IDs; the original variant IDs no longer exist
  • Bulk import overwrote variant IDs — some CSV import tools create new variant IDs while appearing to update existing products, silently invalidating old reorder data
  • Product duplicated incorrectly — duplicating a product creates new IDs; if customers were sent links to the old product and you later archived it, those variant IDs are gone

Fix: never delete variants that appear in completed orders

Instead of deleting discontinued variants, set their inventory to 0 and mark them unavailable. This preserves the variant ID so historical reorder calls can still reference the item — you then handle the out-of-stock scenario rather than the silent-404 scenario, which at least gives you a place to redirect the customer.

Failure Mode 4 — Reorder Does Not Re-Apply Discounts

Shopify reorder is not "repeat this order exactly." It is "add these variant IDs to a new cart." The new cart has no knowledge of:

  • Any discount code used in the original order
  • Staff-applied discounts set in admin on the original order
  • Automatic discounts that were active at the time of the original order but may have expired
  • Loyalty points or store credit that were redeemed at original checkout
  • Any promotional pricing that was in effect (e.g., a flash sale price)

Items are added to the new cart at current full price. If prices have changed since the original order, the reorder reflects the current price — higher or lower.

Options for discount continuity on reorder

Approach How Plan Required
Manual discount code email Use Shopify Flow to detect a completed order and send a follow-up email with a discount code for next purchase Shopify plan+
Automatic repeat-purchase discount Use a Shopify Function to apply a discount automatically when the same customer reorders the same product Shopify and higher
Loyalty app Apps like Smile.io, LoyaltyLion, or Yotpo Loyalty track repeat purchases and auto-apply points/discounts at checkout Any plan (app cost)
Third-party reorder app Apps like Reorder Master can pre-fill discount codes as part of the reorder flow Any plan (app cost)

Failure Mode 5 — Guest Order Reorder Not Available

A guest checkout order is not attached to any customer account. Without an account, there is no order history page, and without an order history page, there is no reorder button. This is a structural limitation — Shopify cannot provide reorder for orders that don't belong to an account.

Workarounds for repeat purchase from guest customers:

  • Post-purchase upsell on the thank-you page — apps like ReConvert, Zipify, or Shopify's native post-purchase extensibility can show a "Buy again" option immediately after checkout while the customer is still authenticated
  • Cart permalink in follow-up email — create a cart URL that pre-fills the items: your-store.myshopify.com/cart/[variant_id]:[qty] and include it in your post-purchase email sequence
  • Prompt to create an account — include an account creation CTA in the order confirmation email; once the guest creates an account and links their order history, reorder becomes available for future orders
  • Winback email with product links — 30/60/90 day winback flows in Klaviyo or Omnisend can deep-link directly to the product page
Cart Permalink Format
To pre-fill a cart with a specific product: https://your-store.myshopify.com/cart/[VARIANT_ID]:[QUANTITY]. For multiple items: /cart/VARIANT_ID1:QTY1,VARIANT_ID2:QTY2. Find variant IDs in Shopify Admin → Products → [product] → click a variant → the ID is in the URL.

Failure Mode 6 — Subscription Products Reorder Incorrectly

Subscription products — managed by Recharge, Bold Subscriptions, Skio, Ordergroove, or similar apps — are not standard Shopify line items. They carry a "selling plan" ID and subscription metadata. Shopify's native reorder button does not know about this metadata.

When a customer tries to reorder a subscription product using the standard Shopify reorder button:

  • The item may be added to cart without its subscription plan — appearing as a one-time purchase. The customer doesn't realize they are buying one unit instead of subscribing again.
  • The subscription app may block the add-to-cart call and silently skip the item.
  • If the selling plan ID is included in the cart call and the plan has expired or been archived, the add will fail silently.

Correct approach for subscription reorders

  1. Direct customers to the subscription management portal (Recharge customer portal, Skio portal, etc.) rather than using the standard order history reorder button
  2. In the subscription portal, customers can skip, pause, or re-activate subscriptions properly
  3. Consider adding a note to the order confirmation email for subscription orders: "To reorder or manage your subscription, visit [portal link]"
  4. If you need a "Buy again" one-time purchase option for subscription products, implement it separately with explicit messaging that it is a one-time purchase, not a subscription

Failure Mode 7 — New Customer Accounts Broke Third-Party Reorder Apps

Shopify launched "new customer accounts" in late 2023 as a replacement for the legacy Liquid-templated /account page. The new account UI is hosted and rendered by Shopify — it is not customizable via Liquid or theme files.

This broke any third-party reorder app that worked by injecting JavaScript or Liquid code into the legacy account template. Apps in this category include:

  • Apps that added a "Reorder All" button to the order history list
  • Apps that added a "Reorder Selected Items" interface
  • Apps that pre-filled discount codes as part of a custom reorder flow
  • Theme customizations that added reorder functionality to the legacy customers/order.liquid template

What to do if you switched to new customer accounts

  1. Go to Shopify Admin → Settings → Customer accounts and confirm which version is active
  2. If on new customer accounts, check the native reorder button — Shopify's built-in UI does include a basic reorder function
  3. For advanced reorder functionality, look for apps that explicitly support "new customer accounts" in their compatibility notes
  4. If your reorder app was business-critical and doesn't yet support new accounts, Shopify still allows reverting to legacy accounts in Settings → Customer accounts → switch to "Classic customer accounts"
Revert Window
Shopify has indicated that legacy customer accounts will eventually be sunset, but as of mid-2026 the revert option is still available. If you need legacy account reorder apps to keep working, revert now and monitor the Shopify changelog for the sunset date.

Failure Mode 8 — Reorder in Headless / Hydrogen Not Implemented

Headless Shopify storefronts (Hydrogen, custom frontends using the Storefront API) do not have a native reorder button. Shopify's admin UI features like the account order history reorder button are not part of the Storefront API — you must build the reorder flow yourself.

Building reorder in a headless Shopify storefront

  1. Query the customer's orders via the Storefront API customer query — include lineItems with variant { id availableForSale } on each order
  2. When the customer clicks "Reorder", collect the variantId and quantity for each line item in that order
  3. Filter out variants where availableForSale is false — surface these to the customer with a message rather than silently skipping them
  4. Call the Storefront API cartLinesAdd mutation with the available variant IDs and quantities
  5. Redirect the customer to the checkout URL from the cart response

Key differences from the native reorder button: you control the stock-check UX, you can prompt the customer about unavailable items, and you can optionally pre-fill a discount code field. Do not automatically re-apply the original discount — verify the discount is still valid first.

Preventing Repeat-Purchase Failures

The reorder button is a convenience feature, not a reliable sales channel on its own. Merchants who depend on repeat purchases should layer additional tools on top:

  • Post-purchase email sequences — trigger a "buy again" email 30-60 days after the original order (Klaviyo, Omnisend, Shopify Email) with direct product links
  • Back-in-stock alerts — if reorder commonly fails due to stockouts, enable back-in-stock notifications so customers are alerted when the item is available again
  • Subscription upsell at checkout — for products customers frequently reorder, offer a subscription option at checkout. This removes the reorder friction entirely.
  • Wishlist / save for later apps — customers who save items can return and purchase without relying on order history
  • CheckPulse monitoring — if your checkout itself has errors, reorders fail before reaching the cart issue level. Monitor checkout health independently.
