Shopify Checkout Troubleshooting

Shopify Age Verification Not Working at Checkout — Age Gate Blocking Customers, Checkout Extensibility Break

Shopify has no native age verification system. Every age gate for alcohol, vape, CBD, tobacco, or adult content is a third-party app — and most of those apps only protect the product page, not the checkout. Three failure modes kill more orders than merchants realize: Checkout Extensibility migrations silence old age gate apps, direct checkout URLs bypass product-page gates entirely, and subscription renewals skip age verification altogether.

Updated June 2026 Covers liquor · vape · CBD · tobacco · adult content merchants
The Core Insight: Most Shopify age gate apps only verify customers on the product page — a popup modal that stores a "verified" flag in the browser. That flag cannot follow the customer to the Shopify checkout domain. Any customer who receives a direct checkout URL, clears cookies, or uses a different device can complete a purchase on an age-restricted product without ever seeing an age gate. Only apps with a Checkout UI Extension close this gap.
How It Works

The Three-Layer Age Verification Architecture: Product Gate → Cart Gate → Checkout Gate

Age verification on Shopify can happen at three distinct points in the purchase journey. Most apps only implement the first layer. Understanding which layers your app covers determines what your actual compliance posture looks like.

Layer 1 — Product Gate
Product Page Modal / Popup
A JavaScript modal that appears when a customer lands on a product page or the storefront. Captures a date of birth entry or a yes/no confirmation. Stores a verified flag in browser localStorage or a cookie. This is the layer most apps implement. It is bypassable via direct URLs, cleared cookies, incognito mode, or direct API access to cart endpoints.
Layer 2 — Cart Gate
Cart Page / Drawer Verification
Some apps add a secondary verification step at the cart level — a banner, redirect, or requirement to verify before proceeding to checkout. This layer is more resilient than the product page gate but still runs on the storefront domain. It cannot intercept purchases made via Shopify's native "Buy Now" button (which skips the cart entirely) or via accelerated checkouts like Shop Pay or Google Pay.
Layer 3 — Checkout Gate
Checkout UI Extension Verification
A verification component embedded directly into Shopify's checkout via the Checkout Extensibility API. This is the only layer that cannot be bypassed by direct URL access. It runs on checkout.shopify.com, not on the storefront. Apps must have a published Checkout UI Extension to implement this layer. Only AgeChecker.Net, Minimum.cc, and a small number of other apps currently support this.

Deep Dive

Checkout Extensibility Migration: The Silent Age Gate Killer

The most common age gate failure in 2025–2026 is merchants migrating to Shopify's new checkout architecture (Checkout Extensibility) and their age gate app losing all checkout-level verification silently.

The Problem

checkout.liquid Is Dead — and Most Age Gate Apps Haven't Adapted

Shopify's old checkout customization model allowed apps to inject JavaScript and HTML into checkout.liquid — the template file that rendered the checkout. Age gate apps used this to show a date-of-birth verification step during checkout itself. When Shopify launched Checkout Extensibility (the new model for Plus and standard checkouts on newer themes), checkout.liquid was deprecated. Shopify's new checkout does not execute JavaScript injected via checkout.liquid. It only renders Checkout UI Extensions — React components built using Shopify's Checkout UI SDK.

If your age gate app renders its checkout layer via checkout.liquid and your store migrated to Checkout Extensibility, the checkout layer is gone. The product page gate may still work. But the checkout has no age verification. The failure is silent — no error message, no broken UI. Customers just complete checkout without being asked for verification.

To check if this is your situation:

Shopify Admin → Settings → Checkout → scroll to "Checkout customization" If it says "Checkout Extensibility" → you are on the new architecture If it says "checkout.liquid" → you are on the old architecture Then: Shopify Admin → Apps → [your age gate app] → check for "Checkout UI Extension" If no Checkout UI Extension is listed → your checkout layer is broken on Extensibility

The fix requires your age gate app to have released and you to have installed a Checkout UI Extension. AgeChecker.Net published their Checkout UI Extension in late 2024. Minimum.cc supports it natively. If your current age gate app does not offer a Checkout UI Extension, you must either wait for them to release one or switch to an app that already supports it.

Diagnosis Steps

How to Verify Your Checkout Extensibility Status and Age Gate Coverage

Step 1: Confirm checkout architecture. Go to Shopify Admin → Settings → Checkout. The "Checkout customization" section shows whether you're on checkout.liquid (old) or Checkout Extensibility (new).

Step 2: Check for Checkout UI Extension. Go to Shopify Admin → Settings → Checkout → scroll to "Checkout UI Extensions." Look for your age gate app in the list. If it's not there, the checkout layer is missing.

Step 3: Run an end-to-end test. Open a private/incognito browser window (no cookies). Add an age-restricted product to your cart. Copy the checkout URL from the browser address bar after clicking "Checkout." Open a new incognito tab and paste the URL directly. Does age verification appear? If not, your Layer 3 coverage is missing.

Step 4: Test accelerated checkouts. On your product page, click "Buy with Shop Pay" (if enabled). Does age verification appear before the Shop Pay modal closes? If not, accelerated checkouts bypass your age gate.

Subscription Renewals: The Headless Age Verification Gap

For merchants selling age-restricted products on subscription — wine clubs, cigar boxes, cannabis delivery, CBD supplements — subscription renewals represent a recurring compliance gap that no standard age gate app addresses by default.

The Problem

Recharge, Skio, and Bold Subscriptions Bypass All Age Gates at Renewal

When a subscription renewal fires (Recharge, Skio, Bold Subscriptions, or any other headless subscription processor), it triggers a Shopify checkout via the Admin API — not via the storefront. There is no browser session. No product page loads. No cart is displayed. The checkout is created programmatically on the server side and completed without any UI rendering at all.

This means every single layer of your age verification stack is bypassed at every renewal: the product page modal (Layer 1) has no page to inject into, the cart gate (Layer 2) has no cart drawer to show, and the Checkout UI Extension (Layer 3) does not execute during API-triggered headless checkouts.

The regulatory risk here is real. If a customer's subscription continues after they turn 18 (and your product requires 21+), or if a customer cancels and someone else takes over the payment method, renewals will process without re-verification. For alcohol merchants in particular, this creates potential liability under state ABC regulations.

The Fix

Customer Tag Verification: The Only Reliable Bridge to Headless Checkout

The mechanism that survives headless renewals is a customer tag. AgeChecker.Net supports tagging verified customers in Shopify with a custom tag (e.g., age-verified) at the time of initial verification. Recharge and Skio both support customer tag checks — you can configure them to refuse renewal processing if the customer does not have the required tag.

Recharge: Customer Portal → Settings → Customer Management → Customer Tags → Add required tag: age-verified → Renewals for customers without this tag: Pause or Cancel (your choice) AgeChecker.Net: Dashboard → Settings → Shopify Integration → Enable "Tag verified customers" → Tag: age-verified

This approach has a gap: customers verified before you enabled tagging won't have the tag. You'll need to bulk-export verified customers from your age gate app and bulk-import the tag into Shopify. AgeChecker.Net's support team can assist with this backfill. Without it, enabling the Recharge tag requirement will pause legitimate subscriptions for existing verified customers.

For merchants on Skio, the same pattern applies via Skio's customer tag requirement feature. Contact Skio support to configure a verification tag requirement on your age-restricted subscription products.

Step-by-Step Diagnosis

Age Verification Checkout Failure — 5-Step Diagnosis

Work through these steps in order. Most age gate checkout failures trace to one of the first three.

  • 1
    Identify which layers your app covers Go to your age gate app's dashboard and check its documentation. Does it have a Checkout UI Extension? Does it cover Layer 1 (product page), Layer 2 (cart), or Layer 3 (checkout)? Log this before testing — you need to know what the app is supposed to do before you can determine whether it's failing.
  • 2
    Test the direct checkout URL bypass Add an age-restricted product to a fresh cart in incognito mode. Note the checkout URL. Open a brand new incognito window (no cookies at all) and paste the checkout URL directly. If you can reach the Shopify checkout page without any age verification prompt, you have a Layer 3 gap. This is the most common undetected failure mode.
  • 3
    Check App Embed status in current active theme After any theme update, Shopify does not automatically transfer App Embeds or App Blocks to the new theme. Go to Admin → Online Store → Themes → click Customize on your live theme → click App Embeds in the left panel → verify your age gate app is listed and toggled ON. If it's missing, you need to reinstall the app embed in the new theme.
  • 4
    Check cookie consent manager (CMP) conflict If you have a cookie consent banner (OneTrust, Cookiebot, Axeptio, etc.), open the product page in incognito and open the browser console (F12). Click "Accept All" on the cookie banner, then reload the page. If the age gate appears only after accepting cookies, your CMP is blocking the age gate script. Fix: in your CMP settings, categorize age verification cookies as "Strictly Necessary" so they load before consent.
  • 5
    Test subscription renewal path (if applicable) If you sell age-restricted subscription products: check whether your subscription app (Recharge, Skio) supports customer tag requirements. Check whether your age gate app (AgeChecker.Net) can tag verified customers in Shopify. If both support tags, configure the integration. If your age gate app does not support customer tagging, evaluate switching to AgeChecker.Net or contact your current app's support about their subscription renewal compliance roadmap.

FAQ

Shopify Age Verification — Frequently Asked Questions

Why is Shopify age verification blocking all customers, not just underage ones?
When an age verification app blocks all customers rather than just those who fail verification, the most common cause is a broken session state — the app cannot write or read the verified flag to browser storage. This happens most often after a theme update, a cookie consent banner blocking localStorage access, or a Content Security Policy change preventing the age gate app's JavaScript from executing. Open the product page in an incognito window, open DevTools Console, and look for JavaScript errors from the age gate app. Check the Application tab → Storage to see whether the verified cookie or localStorage key is written after you click "I am of legal age." If no key is written, the script is failing silently. The second most common cause is a short session TTL — some apps use a 24-hour or 7-day verification window. Check your app's session settings.
Can customers bypass Shopify age verification by going directly to the checkout URL?
Yes — this is one of the most common and least discussed age verification gaps on Shopify. Most age gate apps protect the storefront (product page, collection page, sometimes cart). But the Shopify checkout runs on checkout.shopify.com — a separate application. An age gate that only injects JavaScript into your theme cannot reach the checkout. A customer who receives a direct checkout URL from an abandoned cart email, a shared cart link, or a friend can complete the purchase without ever encountering the age gate. The only age verification that cannot be bypassed this way is a Checkout UI Extension — an age gate component built specifically for Shopify's Checkout Extensibility layer. AgeChecker.Net and Minimum.cc both offer checkout extensions. If you sell age-restricted products, treat any age gate without a Checkout UI Extension as incomplete compliance.
Why did my Shopify age gate stop working after a theme update?
Theme updates break age gate apps in two ways. First, if your theme updated to Checkout Extensibility, any age gate that rendered via checkout.liquid is now on a deprecated path — Shopify's new checkout doesn't execute checkout.liquid modifications. Second, theme updates remove App Embeds from the old theme — they don't automatically transfer to the new one. Go to Admin → Online Store → Themes → Customize on your live theme → App Embeds in the left panel → verify your age gate app is listed and toggled ON. If it's missing or off, re-enable it. For the checkout layer, you need to check whether your app has a Checkout UI Extension and install it via Admin → Settings → Checkout → Checkout UI Extensions.
Does Shopify age verification work with subscription renewals (Recharge, Skio)?
No — subscription renewals bypass age verification entirely, and this is a serious compliance gap for merchants selling age-restricted subscription products (wine clubs, cigar subscriptions, CBD subscriptions). When Recharge, Skio, or Bold Subscriptions processes a renewal, it triggers a headless checkout via the Shopify API — no browser session, no storefront, no UI. Every age verification layer is bypassed. The fix is customer tagging: AgeChecker.Net can tag verified customers in Shopify (e.g., age-verified). Recharge and Skio can be configured to refuse renewal for customers missing this tag. This requires configuring both apps and backfilling the tag for existing verified customers — contact AgeChecker.Net support for the backfill process.
What age verification method does Shopify require for UK merchants selling alcohol or adult content?
Shopify itself has no age verification requirement — it's the merchant's legal obligation under UK law. For UK alcohol merchants, BBFC age verification standards (from the Online Safety Act and preceding regulations) require "robust" verification methods — a birthday entry alone is not sufficient for certain content categories. Robust methods include credit card checks, digital ID verification (via Post Office EasyID, Yoti, etc.), or voucher-based systems. For standard alcohol retail in the UK, a date-of-birth modal with a clear terms statement is the current industry norm, though this is evolving. Minimum.cc is the most commonly used Shopify app for UK/EU merchants that need BBFC-tier or GDPR-compliant age verification. For German merchants, KJM (Kommission für Jugendmedienschutz) requirements for tobacco and alcohol go beyond what most Shopify apps support — consult a German digital law firm for KJM-compliant age verification options.
