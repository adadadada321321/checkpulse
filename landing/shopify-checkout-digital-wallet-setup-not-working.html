Shopify Digital Wallet Setup Not Working — Apple Pay Domain Verification, Google Pay Merchant ID Registration, Shop Pay Activation, Installments Eligibility

Three separate setup requirements, each silent when incomplete: Apple Pay needs a domain verification file at a specific URL. Google Pay requires a separate merchant ID registration with a 1-3 day approval window. Shop Pay won't process payments until Shopify Payments passes full identity verification. Switching to any third-party gateway removes all three wallets instantly.

📅 Updated June 2026 ⏱ 11 min read 💡 Applies to: All Shopify plans (Shopify Payments required)
Critical — All Three Express Wallets Require Shopify Payments
Apple Pay, Google Pay, and Shop Pay are not standalone payment methods on Shopify — they are accelerated checkout experiences built on top of Shopify Payments. If you switch to any third-party payment gateway (Stripe direct, PayPal only, Braintree, Authorize.net), all three wallets are removed from your checkout immediately, with no warning. Shopify Payments must remain your active payment processor for any express wallet to work.

Which Wallet, What's Required, and What Fails

Failure Mode 1 — Apple Pay Domain Verification Fails Silently

Apple Pay on the web uses a domain verification process defined by Apple's spec. To render the Apple Pay button, the merchant's domain must be registered with Apple and must be able to serve a specific verification file at a specific URL. If either condition fails, the Apple Pay button simply does not appear — no error is shown to the customer or merchant.

How Shopify handles domain verification

Shopify automatically verifies your .myshopify.com subdomain and Shopify's checkout domain. For the vast majority of stores whose checkout runs on Shopify's hosted checkout (checkout.shopify.com or the legacy yourstore.myshopify.com/checkout), domain verification is handled automatically and you never need to touch it.

The problem arises in these specific scenarios:

  • Custom domain used on storefront pages where Apple Pay buttons render (cart page, product pages with Buy Now) — if your storefront is on yourstore.com, that domain must also be verified
  • Custom domain on a non-Shopify-managed host — third-party hosting platforms may not automatically serve the verification file at the required path
  • Headless storefront on a custom domain — Shopify's auto-verification does not cover custom frontend domains in Hydrogen or custom builds
  • Multiple domains (www vs non-www vs custom subdomain) — each domain variant requires separate verification

The verification file location

Apple requires a file to be served at exactly:

https://yourdomain.com/.well-known/apple-developer-merchantid-domain-association

This file must be accessible without redirects, must return HTTP 200, and must contain Apple's merchant domain association data. Shopify hosts this automatically for its managed domains. For custom domains on external hosts, you may need to manually upload this file or configure your host to serve it.

HTTPS Everywhere — Mixed Content Blocks Apple Pay
Apple Pay requires all pages where the Apple Pay button renders to be served over HTTPS with no mixed content. If any resource on your cart page or product page loads over HTTP (images, fonts, scripts), Apple Pay will be silently blocked even if domain verification passes. Check your browser console for mixed content warnings.

How to diagnose Apple Pay domain verification failure

  1. Open Safari (macOS or iOS) — Apple Pay is unavailable in Chrome, Firefox, or any non-Apple browser by design
  2. Navigate to your cart page or a product page with a Buy Now button
  3. Open Safari Developer Tools → Console and look for Apple Pay-related errors
  4. Manually request the verification URL: https://yourdomain.com/.well-known/apple-developer-merchantid-domain-association — it should return a JSON file, not a 404
  5. If the file is missing, contact Shopify support to re-trigger domain verification, or manually upload the file to your hosting provider

Failure Mode 2 — Shop Pay Activation Blocked by Pending Payments Verification

Shop Pay is Shopify's own accelerated checkout — it saves customer information for faster re-purchase across Shopify stores. To offer Shop Pay, your Shopify Payments account must reach "Active" verification status, not just be connected.

Shopify Payments verification stages

Status Shop Pay Behavior What to Do
Pending Button may appear but transactions fail at authorization step Complete bank account + identity verification in Admin
Action Required Shop Pay hidden or failing — Shopify awaiting documents Upload requested ID/business documents
Suspended All payments including Shop Pay blocked Contact Shopify Payments support via Admin
Active Shop Pay fully functional No action needed

Where to check Shopify Payments verification status

  1. Admin → Settings → Payments
  2. Under "Shopify Payments," click "Manage"
  3. Check the "Account status" section — must show "Active" for Shop Pay to work
  4. If status is Pending or Action Required, complete all requested verification steps (bank account routing/account number, SSN/EIN for US stores, business registration documents for non-US)
Verification Timing — New Stores Are Most Affected
Identity verification typically completes within 1-5 business days for new Shopify Payments accounts. This is the most common reason new stores report Shop Pay "not working" immediately after launch — the store opened before verification completed. Shop Pay will activate automatically once Shopify Payments becomes Active.

Failure Mode 3 — Google Pay Merchant ID Not Registered (1-3 Day Approval Window)

Google Pay on Shopify is enabled through Shopify Payments, but it requires an additional step that many merchants miss: you must register your business at pay.google.com/business/console to obtain production payment access. Without this registration, Google Pay runs in test mode only — it renders the payment sheet but rejects real card payments with a MERCHANT_NOT_FOUND or MERCHANT_NOT_APPROVED error.

What Google Pay requires beyond Shopify Payments

  • Business registration at pay.google.com/business/console
  • Business name (must match your Shopify Payments / Stripe account name)
  • Website URL (must be your live storefront, HTTPS required)
  • Contact email address for Google's review
  • Approval period: typically 1-3 business days (Google sends email confirmation)

Test mode vs production mode — the key difference

Google Pay Test Mode Shows Correct UI But No Real Payments
In Shopify's development theme preview, password-protected store, or before Google Pay merchant approval, Google Pay runs in ENVIRONMENT_TEST. The payment sheet opens, test card numbers work, but no real money moves. The Google Pay button is indistinguishable between test and production mode visually. Merchants who tested with test cards before going live often discover on launch day that real customer cards fail.

The approval is separate from Shopify Payments and separate from Shopify's Google Sales Channel. Connecting Google Ads or Google Shopping to Shopify does not give you Google Pay merchant approval — these are different systems.

  1. Go to pay.google.com/business/console and sign in with your Google account
  2. Click "Integrate Google Pay" → select "Website"
  3. Fill in business name (must match your payment processor name), website URL, contact email
  4. Submit for review — Google typically approves within 1-3 business days
  5. Check for Google's approval email, then test with a real card on your live store

Failure Mode 4 — Shop Pay Installments Eligibility Not Met

Shop Pay Installments is Shopify's built-in buy now, pay later (BNPL) option. It is separate from Shop Pay itself — enabling Shop Pay does not automatically enable Installments. It also has hard eligibility requirements that are rarely documented clearly in Shopify's Admin UI.

Shop Pay Installments eligibility checklist

Requirement Eligible Not Eligible
Geography US only All non-US stores, even USD transactions
Order value $50 – $17,500 USD Under $50 or over $17,500
Product type Physical products Digital products, subscriptions, gift cards, duty/tax-only orders
Payment processor Shopify Payments (Active) Any third-party gateway
Toggle Installments separately enabled in Admin Only Shop Pay enabled, Installments toggle off

Where to enable Installments separately

  1. Admin → Settings → Payments
  2. Under Shopify Payments, click "Manage"
  3. Scroll to "Buy now, pay later options"
  4. Enable "Shop Pay Installments" toggle
  5. Save — Installments will appear for eligible orders (US, $50+, physical products)
Competing BNPL Apps Can Override Installments Display
If you have a third-party BNPL app (Afterpay, Klarna, Affirm) installed, it may render its own BNPL option at checkout while Shop Pay Installments does not appear. Shopify's checkout shows one BNPL provider at a time. Test in an Incognito window with all extensions disabled to isolate whether the conflict is between providers.

Failure Mode 5 — Apple Pay Only Available in Safari and Apple Ecosystem

This is the single most common reason developers and merchants believe Apple Pay is "not working" after setup: they test on Chrome and see no Apple Pay button, conclude the setup failed, and start troubleshooting a problem that does not exist.

Apple Pay on the web is architecturally restricted by Apple. It is available only on:

  • Safari on macOS 10.12 (Sierra) and later — with a card added to Apple Wallet via iCloud
  • Safari on iOS 10 and later — with a card added to Apple Wallet
  • Safari on iPadOS — with a card added to Apple Wallet

Apple Pay is never available in:

  • Google Chrome (any platform)
  • Mozilla Firefox (any platform)
  • Microsoft Edge (any platform)
  • Any browser on Android or Windows
  • Non-Safari WebViews on iOS (Chrome for iOS, Firefox for iOS use WebKit but Apple Pay JS API is blocked)
Testing Correctly — Use Safari on macOS or iOS Safari
To confirm Apple Pay is working: open Safari on a Mac with a card in iCloud Keychain / Apple Wallet, or open Safari on an iPhone with a card in Apple Wallet. Navigate to your cart page. The Apple Pay button should appear above the "Check out" button. If you see it there but not in Chrome, your setup is correct — this is expected browser behavior.

Failure Mode 6 — Google Pay Shows Test Environment, Real Cards Fail

Shopify's theme preview (via the Customize editor), development themes, and password-protected stores all cause Google Pay to run in Google's ENVIRONMENT_TEST mode. In this mode:

  • The Google Pay payment sheet opens normally
  • Test card numbers (e.g., Google's test card ending in 1111) complete successfully
  • Real card numbers are rejected or show $0.00 transaction value
  • No live transaction processing occurs

Merchants who test thoroughly with Shopify's theme preview before launch often discover on launch day that real customers cannot pay via Google Pay. The fix is to test Google Pay on your live, published storefront after Google merchant registration has been approved.

How to confirm you are in production mode

  1. Ensure your theme is published (not in preview mode)
  2. Ensure your store is not password-protected
  3. Confirm Google merchant registration is approved at pay.google.com/business/console (status: Production)
  4. Add a physical product to cart and proceed to checkout
  5. Select Google Pay — the payment sheet should show your actual cart total (not $0.00)
  6. Complete with a real debit/credit card to confirm end-to-end

Failure Mode 7 — Third-Party Payment Gateway Removes All Express Wallets

Shopify's express checkout wallets (Apple Pay, Google Pay, Shop Pay) are built into Shopify Payments' infrastructure. They are not configurable through external payment gateways. When you disable Shopify Payments to use a third-party gateway, all three wallets are removed from your checkout simultaneously — there is no way to keep them active with an external gateway.

Which Gateway Setups Kill Express Wallets
  • Shopify Payments disabled → Stripe.js direct integration: all wallets gone
  • Shopify Payments disabled → PayPal Express only: all wallets gone (PayPal Express is different from Apple/Google/Shop Pay)
  • Shopify Payments disabled → Braintree, Authorize.net, Square: all wallets gone
  • Shopify Payments active + third-party as secondary: wallets still available (Shopify Payments primary)

The architectural reason: Apple Pay and Google Pay require a merchant identifier registered with Apple/Google respectively, a payment session obtained from a certified payment processor, and a server-side transaction completion. Shopify built all of this into Shopify Payments. There is no supported path to use Apple Pay or Google Pay through a third-party gateway on Shopify's hosted checkout.

If you need a third-party gateway AND express wallets

The options are:

  • Keep Shopify Payments active with the third-party gateway added as an alternative payment method — customers who cannot use Shopify Payments can select the alternative at checkout
  • Headless storefront — build a custom frontend that handles Apple Pay JS API and Google Pay API directly, using your own payment processor (Stripe, Braintree) that supports these wallet APIs natively

Failure Mode 8 — Headless / Hydrogen Storefronts Require Manual Wallet Implementation

If you run Shopify Hydrogen, a custom Next.js storefront, or any headless frontend, Shopify's hosted checkout handles the payment step — and the express wallets work there because Shopify Payments manages the checkout session. However, if you want express wallet buttons on your storefront pages (product page, cart page, custom checkout), you must implement them manually.

What "manual implementation" means in headless

Wallet API to Implement Shopify's Role
Apple Pay Apple Pay JS API — ApplePaySession, merchant validation, payment authorization Merchant session validation via Shopify Payments API endpoint
Google Pay PaymentRequest API or Google Pay API — payment data request, payment authorization Payment token processing via Shopify Payments
Shop Pay Storefront API shopPayPaymentRequestSessionCreate mutation Session creation and redirect to Shop Pay checkout
Most Common Headless Mistake — Visual Buttons Without Payment Session Wiring
A frequent pattern in headless builds is to render Apple Pay or Google Pay buttons as styled HTML elements that visually look correct, without wiring the underlying payment session APIs. The button appears, the customer taps it, but the payment sheet never opens (Apple Pay) or the Google Pay sheet opens with no real payment connection. Always implement the full payment session flow, not just the UI.

Quick reference: Shopify Storefront API for Shop Pay express

mutation shopPayPaymentRequestSessionCreate($paymentRequest: ShopPayPaymentRequestInput!) {
  shopPayPaymentRequestSessionCreate(paymentRequest: $paymentRequest) {
    shopPayPaymentRequestSession {
      token
      sourceIdentifier
      checkoutUrl
    }
    userErrors { field message }
  }
}

Use the returned checkoutUrl to redirect the customer into Shop Pay's accelerated checkout flow. For Apple Pay and Google Pay, use Shopify's Hydrogen payment processing documentation for the full implementation guide.

Diagnosing Your Wallet Setup — Quick Checklist

Step-by-Step Diagnostic
  1. Confirm Shopify Payments is active (not a third-party gateway) in Admin → Settings → Payments
  2. Check Shopify Payments verification status — must be "Active" for Shop Pay to work
  3. For Apple Pay: test in Safari on macOS or iOS with a card in Apple Wallet
  4. For Google Pay: check pay.google.com/business/console for "Production" approval status
  5. For Apple Pay on custom domain: verify the /.well-known/apple-developer-merchantid-domain-association file is accessible
  6. For Shop Pay Installments: confirm US store, $50+ order, physical product, Installments toggle ON
  7. For headless: confirm full payment session API is wired, not just visual button rendered
