Shopify Checkout Custom Fields Not Working — Line Item Properties, Note Attributes, Checkout Extensibility

checkout.liquid was deprecated June 2025, line item properties live on the product page not inside checkout, Buy Now bypasses all of it, and note_attributes is not order.note. Here is the complete failure mode guide.

📅 Updated June 2026 ⏱ 10 min read 💡 Applies to: All Shopify plans
checkout.liquid Is Gone — All Custom Checkout Fields Built on It Are Broken
Shopify deprecated checkout.liquid on June 30, 2025. Every custom field injected via checkout.liquid code — built by you, your developer, or an app — stopped rendering after that date. If your custom checkout fields disappeared around mid-2025, this is why. Migration path: Checkout UI Extensions (Shopify Plus only).

How Custom Fields Actually Work in Shopify Checkout

Custom checkout fields in Shopify are more fragmented than they appear. There is no single "custom field inside checkout" feature available on all plans. Instead, there are four distinct mechanisms, each with different plan requirements, different placement, and different where-the-data-lives behavior:

Mechanism Where it lives Data saved to Plan required
Line item properties Product page form line_items[].properties All plans
Cart attributes Cart page template order.note_attributes All plans
Order note Cart page template order.note All plans
Checkout UI Extension — TextField Inside checkout flow order.note_attributes Shopify Plus only

The key consequence: no non-Plus Shopify plan supports custom input fields rendered inside the checkout flow itself. All non-Plus custom field collection must happen before the customer clicks "Check out."

Failure Mode 1 — checkout.liquid Deprecated (June 2025)

Prior to June 30, 2025, Shopify allowed Plus merchants to customize the checkout UI by editing checkout.liquid directly. Many merchants, developers, and checkout-customization apps used this to inject custom input fields into the checkout flow.

On June 30, 2025, Shopify removed support for checkout.liquid across all plans as part of the Checkout Extensibility migration. Custom code in checkout.liquid — including custom fields, JavaScript injections, and additional_scripts modifications — no longer runs.

Signs You Were Affected
Custom checkout field disappeared around mid-2025 with no other changes made. App that "added custom fields to checkout" stopped working. Developer-built checkout customization no longer renders. All of these point to checkout.liquid deprecation.

Migration path

The replacement for checkout.liquid customization is Checkout UI Extensions, available on Shopify Plus only. A Checkout UI Extension is a React-based app extension that renders inside the checkout flow using Shopify's controlled component library.

For non-Plus merchants whose app or customization relied on checkout.liquid:

  • Contact your app provider — most checkout-customization apps have migrated to Checkout UI Extensions (which requires the merchant to upgrade to Plus)
  • If upgrading to Plus is not viable, move field collection to the product page (line item properties) or cart page (cart attributes)
  • Consider collecting additional info post-purchase via a thank-you page or follow-up email

Failure Mode 2 — Line Item Properties Not Showing at Checkout (They're Not Supposed To)

Line item properties are one of the most misunderstood features in Shopify. They work correctly and do reach the order — merchants just look for them in the wrong place.

Line item properties are set on the product page via hidden or visible inputs in the product form:

<input type="text" name="properties[Engraving]" />
<input type="hidden" name="properties[Gift message]" value="{{ some_value }}" />

When a customer adds the product to cart with these inputs present, the property values are captured and attached to that line item. They travel through checkout invisibly — there is no input field inside the checkout step itself — and they arrive on the order record.

Where line item properties appear after checkout

  1. Go to Shopify Admin → Orders and open the order
  2. Find the line item (product) in the order
  3. Properties appear directly below the product name, listed as key–value pairs
  4. They do NOT appear under Notes or Additional details — those are for order-level fields
  5. In order notification emails, properties display only if your email template includes {% for p in line_item.properties %}
Confirming Line Item Properties Are Saving Correctly
In Shopify Admin, go to Store → Preview, add the product to cart with your custom fields filled in, complete a test checkout, then inspect the resulting order. Look under the line item — if the properties appear there, the mechanism is working. If you don't see them, the product form input names are likely missing the properties[ prefix.

Failure Mode 3 — Buy Now / Express Checkout Bypasses All Custom Fields

This is the silent killer for product customization. Every express checkout path bypasses the product form entirely:

Checkout Path Line Item Properties Captured? Cart Attributes Captured?
Standard "Add to Cart" → cart → Check out Yes Yes
Shopify native "Buy Now" button No — bypasses product form No — bypasses cart page
Shop Pay express button No No
Apple Pay / Google Pay No No
Cart permalink (/cart/add?id=...&properties[x]=y) Yes — if encoded in URL Partial — attributes not in URL

What to do about express checkout bypassing custom fields

  • Hide or disable Buy Now / express buttons on products that require customization — add a notice explaining why ("Please use Add to Cart to add personalization")
  • Shopify Plus only: use a Checkout UI Extension TextField to re-collect the field inside checkout — this works regardless of checkout entry path
  • Encode properties in cart permalink URL for programmatic cart links: /cart/add?id=VARIANT_ID&properties[Engraving]=text&quantity=1
  • Post-purchase follow-up: collect customization details in an automated email after purchase for orders missing the property

Failure Mode 4 — note_attributes ≠ order.note (Wrong Place to Look)

Two separate storage locations in Shopify look similar but are completely different fields, and merchants routinely look in the wrong one.

Liquid source Shopify Admin label Admin API field Where to look in Admin
cart.note → textarea name="note" Notes order.note Order detail → Notes section (right sidebar)
cart.attributes → input name="attributes[Label]" Additional details order.note_attributes[] Order detail → Additional details (scroll past Notes)

If you added a field to your cart template using name="attributes[Gift message]" and are checking the Notes section, you will always see nothing — not because the field is broken but because it saves to Additional details instead.

Quickest Verification Method
Use the Shopify Admin API: GET /admin/api/2024-01/orders/{id}.json?fields=note,note_attributes. The raw JSON shows both fields at once, with no UI confusion about which section you're in.

Failure Mode 5 — Checkout UI Extension TextField Not Saving (Plus Only)

On Shopify Plus with checkout extensibility, a TextField component inside a Checkout UI Extension renders a visible input field inside the checkout flow. However, the value is never automatically persisted — you must explicitly call applyAttributeChange() in your extension code:

import { useApplyAttributeChange } from '@shopify/ui-extensions-react/checkout';

const applyAttributeChange = useApplyAttributeChange();

// Called when the field value changes
async function handleChange(value) {
  await applyAttributeChange({
    key: 'Gift message',
    type: 'updateAttribute',
    value: value,
  });
}

Without this call, the TextField renders and accepts input, but the value is discarded when the customer moves to the next checkout step. The value saved via applyAttributeChange appears in order.note_attributes — visible under "Additional details" in the order, not under "Notes."

Extension deployment checklist

  1. Extension must be deployed to production — development mode only loads in a local preview
  2. Extension must be assigned to a checkout customization in Shopify Admin → Settings → Checkout → Customize → Add apps
  3. Verify the target placement — if targeting purchase.checkout.contact.render-after, the extension only loads after the contact section; wrong placement = never renders
  4. Test in a real checkout session (not the theme editor preview) with a real order to verify the attribute appears on the resulting order
  5. Check for console errors — a runtime error in the extension silently prevents it from rendering

Failure Mode 6 — App-Injected Fields Lost When Cart Update Overwrites Attributes

Cart-modifying apps — upsell apps, cart drawer replacements, discount apps, subscription billing apps — frequently call the Cart API to update cart contents. If the API call updates the cart without preserving the existing attributes payload, all previously captured custom field values are silently erased.

This is a known bug pattern with apps that use POST /cart/update.js without reading and re-sending the existing attributes in the same request body.

How to diagnose

  1. Complete a test purchase with all custom fields filled in correctly (confirm they're in the cart with GET /cart.js)
  2. Trigger the cart-modifying app's functionality (trigger the upsell, apply a discount, change quantity via the cart drawer)
  3. Immediately fetch /cart.js again — if attributes is now empty, the app's cart update call is overwriting it
  4. Contact the app vendor — this is a bug in their Cart API integration. The fix: read current attributes, merge the new data, send the merged payload

Failure Mode 7 — Custom Field App JavaScript Blocked (Infinite Options, SC Product Options)

Apps like Infinite Options, SC Product Options, Hulk Product Options, and Bold Product Options inject custom field inputs into the product page using JavaScript. If their JavaScript doesn't load, no custom fields appear and no properties are captured.

Common blockers:

  • Cookie consent manager: many CMP tools (Cookiebot, OneTrust, Axeptio) block all third-party scripts until consent is given. If the app's script is categorized as "marketing", it won't load for visitors who haven't consented — no fields, no property capture.
  • Ad blockers: browser ad blockers may block the app's CDN URL if it matches known tracking patterns
  • Content Security Policy: a CSP header on the store that doesn't include the app's script domain will block loading
  • App not properly installed: the app's theme app embed must be enabled in the theme customizer for each theme. If the live theme is different from the theme where the app was configured, the embed may not be active

Quick check

Open your product page in a private browser window (no extensions, no logged-in state), open browser DevTools → Console, and look for JavaScript errors. If you see the app's domain in a blocked resource error, you have a CSP or CMP blocking issue.

Failure Mode 8 — Headless / Storefront API Must Use cartAttributesUpdate

Headless Shopify storefronts using the Storefront API do not automatically capture cart attributes or line item properties from HTML form inputs — there are no Liquid form helpers. All custom field data must be passed explicitly via API mutations.

For cart-level custom fields (note_attributes equivalent), use the cartAttributesUpdate mutation:

mutation cartAttributesUpdate($cartId: ID!, $attributes: [AttributeInput!]!) {
  cartAttributesUpdate(cartId: $cartId, attributes: $attributes) {
    cart {
      attributes {
        key
        value
      }
    }
    userErrors {
      field
      message
    }
  }
}

For line item properties, pass the customAttributes array when adding items via cartLinesAdd:

mutation cartLinesAdd($cartId: ID!, $lines: [CartLineInput!]!) {
  cartLinesAdd(cartId: $cartId, lines: $lines) {
    cart {
      lines(first: 10) {
        edges {
          node {
            attributes { key value }
          }
        }
      }
    }
  }
}

# Variables:
# lines: [{ merchandiseId: "gid://shopify/ProductVariant/...",
#            quantity: 1,
#            attributes: [{ key: "Engraving", value: "Hello World" }] }]
Common Headless Mistake
Many headless storefronts collect custom field inputs in React state but never call cartAttributesUpdate or pass customAttributes in cartLinesAdd. The field data exists in the browser but never reaches the Shopify cart — the resulting order has no custom field data at all.

Displaying Custom Fields in Order Notification Emails

Even when custom field data is correctly saved to an order, it may not appear in confirmation emails. Order notification email templates do not automatically include line item properties or note_attributes — you must add the display code explicitly.

Adding line item properties to confirmation email

In Shopify Admin → Settings → Notifications → Order confirmation, find the line items section and add:

{% for property in line_item.properties %}
  {% unless property.last == blank %}
    {{ property.first }}: {{ property.last }}
  {% endunless %}
{% endfor %}

Adding note_attributes to confirmation email

{% if attributes %}
  {% for attribute in attributes %}
    {{ attribute.first }}: {{ attribute.last }}
  {% endfor %}
{% endif %}
