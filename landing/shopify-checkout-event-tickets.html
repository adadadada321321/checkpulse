Shopify Event Ticket Delivery Failure

Shopify Event Ticket Not Delivered After Purchase — No Native Event Type, App Webhook Failures

Shopify has no native event ticket product type. Every ticket is generated by a third-party app via an order webhook. When that webhook fails — billing lapse, API timeout, high-traffic overload — the payment goes through but no ticket is generated. Six silent failure modes leave customers ticketless while Shopify Admin shows the order as Fulfilled.

Updated June 2026 Affects: Evey Events, BookThatApp, Events Calendar by Mintt, Cowlendar
The Shopify Blind Spot: Shopify has no event product type. It cannot track event capacity, generate tickets, or know whether a ticket email was delivered. When your event app's webhook fails or its email is filtered, Shopify Admin shows the order as Fulfilled — because the payment succeeded. Whether the ticket was generated and delivered is entirely invisible to Shopify.
Architecture

How Shopify Event Tickets Actually Work — The Webhook Chain

Shopify has four built-in product types: Physical, Digital, Gift Card, and Service. There is no Event or Ticket type. Every Shopify event ticket flows through this chain — and every failure point in the chain is invisible to Shopify itself.

Step 1 — Shopify Checkout
Customer Completes Purchase
Customer buys an "event ticket" product on Shopify. Payment is processed. Shopify marks the order Paid and Fulfilled. Shopify fires an order-creation webhook to the registered event app.
Step 2 — Webhook Delivery
Shopify Sends Webhook to App
Shopify sends the order data to the event app's webhook endpoint. If the endpoint is down, slow to respond, or returns an error, Shopify retries up to 19 times over 48 hours — but silently. No merchant notification is sent during retries or final failure.
Step 3 — Ticket Generation
App Creates Ticket + QR Code
If the webhook is received, the event app generates a unique ticket, creates a QR code or barcode for entry scanning, and records an attendee in its database. This step fails if the app's subscription has lapsed, the app is uninstalled, or the app has an internal processing error.
Step 4 — Email Delivery
App Sends Ticket Confirmation
The event app sends the ticket confirmation email with the PDF ticket or QR code attached. Email may go to spam, bounce if the customer's address has a permanent bounce flag, or fail if the app's email sending service has an outage. Shopify has no visibility into this step.

The Webhook Failure: When Payment Succeeds But No Ticket Is Generated

The most damaging event ticket failure is a silent webhook failure. Payment processes. Order shows Fulfilled. But the event app never received the order data, so no ticket was generated and no email was sent.

The Problem

How Shopify Webhook Failures Produce Ticketless Orders

When a Shopify order is completed, Shopify fires an orders/create webhook to every app that has registered for it. If the event app's webhook endpoint is unavailable, slow, or returns an HTTP error (5xx), Shopify retries. Shopify's webhook retry policy: exponential backoff, up to 19 attempts over approximately 48 hours. After the final failed attempt, Shopify stops retrying and logs the failure — but does not alert the merchant via the Shopify Admin dashboard in a visible way.

The most common cause of a persistent webhook failure is app billing lapse. If the merchant's credit card on file for the app subscription fails and the app subscription expires, the app vendor typically takes the webhook endpoint offline or behind an authentication wall that Shopify cannot pass. Every order placed during the subscription gap produces a ticketless order.

During high-traffic sales periods — concert announcements, festival ticket drops — webhook endpoints can be overloaded. If the event app's infrastructure cannot handle the concurrent webhook volume, endpoints may return timeouts. Shopify will retry, but if the app recovers before the final retry, some orders may still miss their ticket generation window.

To check for webhook failures: Shopify Admin → Settings → Notifications → Webhooks (legacy) OR Shopify Admin → Partners Dashboard → Apps → [your event app] → Webhook logs Look for: "Failed delivery" or "Max retries exceeded" entries. The timestamp and order number are visible in the webhook failure log. Cross-reference failed webhook order numbers against your event app's attendee list to identify which orders need manual processing.
Recovery

Manually Recovering Webhook-Failed Orders

For each order where the webhook failed and no ticket was generated, you must manually create the attendee record in the event app:

Recovery path for webhook-failed event ticket orders: 1. Identify the affected orders: - Shopify Admin → Orders → filter by date range of the webhook failure - OR check Shopify's webhook failure log for "Max retries exceeded" 2. In your event app dashboard, check for each affected order number: - Evey Events: Attendees tab → search by order number or customer email - BookThatApp: Bookings → search by order reference - Events Calendar: Attendees → search by name or order number 3. For orders with no attendee record, manually create the attendee: - Most event apps have an "Add attendee manually" option - Enter the customer name, email, and order number - The app will generate the ticket and QR code - Use the "Resend ticket" function from the attendee record 4. Contact your event app's support: - Most apps can replay missed webhooks from their backend - Provide the Shopify order numbers and the approximate time window

If your event app supports it, enable webhook replay in the app settings. This allows the app to periodically pull recent Shopify orders and cross-check them against its attendee database, automatically catching orders that missed the original webhook.

Event Overselling: Why Shopify Inventory Cannot Enforce Event Capacity

Shopify's inventory system was built for physical goods. For events, independent variant inventory counts can allow more total tickets to be sold than the venue holds.

The Problem

Variants Don't Share Inventory — Shopify Doesn't Know It's a Venue

Merchants typically structure event tickets as a single Shopify product with variants for each ticket tier: General Admission (200 qty), VIP (50 qty), Front Row (20 qty). In Shopify's data model, each variant has an independent inventory count. Shopify will happily sell all 270 tickets even if the venue only holds 200 people — because Shopify does not know that all three variants share the same physical space.

The underlying issue: Shopify inventory is per-variant. There is no shared pool concept. Two customers could theoretically buy the last General Admission ticket simultaneously during a high-traffic sale (if the inventory check and reservation happen between stock-check and order-lock), though Shopify does handle concurrent purchases with row-level locking in most scenarios.

The more common overselling path is merchant configuration error: setting GA to 200 and VIP to 50 with no awareness that the venue total is 200, not 250.

The Fix

Use the Event App for Capacity, Not Shopify Inventory

Every production event ticketing setup on Shopify should enforce capacity in the event app, not solely in Shopify's inventory. Shopify inventory is a secondary signal.

Capacity enforcement checklist: 1. Set a total event capacity in your event app dashboard: Evey Events: Event Settings → Capacity → Total Attendees BookThatApp: Service Settings → Max Bookings Per Slot Events Calendar: Event → Capacity → Max Registrations 2. Set Shopify inventory counts per tier with a buffer under the event app cap: - If event capacity is 200 total: set Shopify inventory so the sum of all variant quantities does not exceed 200 - The event app's cap is the hard limit; Shopify inventory is a soft limit 3. Enable "Continue selling when out of stock" = OFF on all ticket variants. Shopify Admin → Products → [event product] → each variant → uncheck "Continue selling when out of stock" 4. During high-demand sales: temporarily lower Shopify inventory to below your expected concurrent purchase rate to prevent double-selling in the checkout race condition window. 5. Monitor the event app's attendee count in real time during sales — not Shopify's inventory count.

Buy Now Button Bypass: Orders With No Attendee Information

The Shopify Buy Now button skips the cart entirely. For event tickets that require attendee information collection, this creates orders with incomplete attendee records and tickets that cannot be personalized.

The Problem

What Gets Skipped When Buy Now Bypasses the Cart

Most event ticket setups collect attendee information through one of three mechanisms: custom product page fields (via the event app's theme App Block), cart page fields, or a post-checkout form. The Shopify Buy Now button — which appears as a secondary button below "Add to Cart" on product pages — skips directly to checkout without loading the cart. This means any attendee data collection that happens on the product page (via an App Block) or on the cart page is bypassed.

The result: the event app receives the order webhook, generates a ticket, and creates an attendee record — but the record contains no name, no dietary restriction response, no seat preference, and no emergency contact information. The ticket PDF may show "Guest" or have blank fields where the attendee name should appear.

On the day of the event, attendees with blank-name tickets cannot be looked up by name in the check-in system. Entry scanning still works via QR code — but the experience is degraded and the attendee data is permanently lost for this order.

Fix

Disabling Buy Now on Event Products / Moving Fields to Checkout

Option 1: Disable the Buy Now button on event ticket products only - Use a custom product template that removes the Buy Now button section - Shopify Admin → Online Store → Themes → Customize → Products → Create a "Event Ticket" template → remove the "Buy buttons" section or hide it - Assign the template to all event ticket products: Product → Template → [Event Ticket] - This prevents accidental Buy Now purchases entirely Option 2: Collect attendee info at checkout via Checkout Extensibility - Shopify Plus merchants only (Checkout Extensibility is Plus-only for input fields) - Add a Checkout UI Extension that presents attendee name, email, and any required fields on the Information step of checkout - The event app receives these as order metafields in the webhook payload - Works regardless of whether the customer used Add to Cart or Buy Now Option 3: Post-purchase attendee collection (all Shopify plans) - Send a post-purchase email with a link to a registration form - Form submits to the event app directly (most apps support this) - Downside: relies on customer completing a second step — completion rate is typically 70-85%, leaving 15-30% of attendees with incomplete records

Step-by-Step Diagnosis

Event Ticket Not Delivered — 5-Step Diagnosis

Work through these steps in order. Most "ticket not received" reports resolve at step 1 or 2.

  • 1
    Check if the ticket was generated in the event app at all Log into your event app dashboard (Evey Events, BookThatApp, Events Calendar, etc.) and search for the order number or customer email address. If no attendee record exists: the webhook failed and the ticket was never generated. If a record exists: the ticket was generated — the problem is email delivery, not ticket creation. Go to step 2 for email delivery issues. Go to step 3 for missing attendee records.
  • 2
    Ask the customer to check their spam folder Event ticket confirmation emails from third-party apps come from the app's own sending domain, not from Shopify. Corporate email accounts (Office 365, Google Workspace) frequently filter these to spam or the Clutter folder. Ask the customer to search their spam folder for the event name, the app name, or your store name. If the email is there, the customer can mark it as "Not Spam" and the QR code is in the email attachment. If not found, proceed to resend from the app dashboard.
  • 3
    Resend the ticket from inside the event app Do NOT try to resend from Shopify Admin — Shopify has no access to the ticket. Find the attendee record in your event app and use the "Resend ticket" or "Resend confirmation email" function. Before resending, verify that the email address on the attendee record matches what the customer intended. If the address has a typo, update it in the app dashboard before resending.
  • 4
    If no attendee record: check Shopify webhook health Go to Shopify Admin → Settings → Notifications → Webhooks (or check your app's webhook log). Look for failed deliveries around the time of the customer's order. If the webhook shows failed delivery or max retries exceeded, the app never received the order. Manually create the attendee record in the event app using the order details from Shopify. Most apps allow manual attendee addition with an attached Shopify order reference.
  • 5
    Share the QR code directly if email fails Once the attendee record exists in the event app and the ticket is generated, you can retrieve the QR code or ticket PDF directly from the app dashboard. Send it to the customer via another channel (text message, WhatsApp, a direct download link). For day-of-event emergencies, most event apps support manual check-in by name or order number — the QR code is not strictly required for entry if you have the attendee on your list.

FAQ

Shopify Event Tickets — Frequently Asked Questions

Why did my Shopify event ticket not arrive after purchase?
Shopify has no native event ticket product type — every ticket is generated by a third-party app (Evey Events, BookThatApp, Events Calendar) when Shopify fires an order webhook. The most common reason a ticket is not delivered is that this webhook failed: the app's subscription lapsed, the app's endpoint was temporarily down, or there was a timeout during a high-traffic sale. Check your event app dashboard for the customer's attendee record. If no record exists, the webhook failed and no ticket was generated. If a record exists, the problem is email delivery — ask the customer to check their spam folder, then resend from the app dashboard.
Does Shopify have a native event ticket product type?
No. Shopify's built-in product types are Physical, Digital (for file downloads), Gift Card, and Service. There is no Event or Ticket type. Merchants selling event tickets on Shopify must use a third-party app that hooks into Shopify's order webhook to generate tickets. The most widely used apps are Evey Events and Tickets, BookThatApp Appointment Booking, Events Calendar by Mintt, and Cowlendar. Each of these apps listens for Shopify's order-creation webhook and generates a ticket when a matching product is purchased.
How do I find and resend an event ticket in Shopify?
You cannot resend an event ticket from Shopify Admin — Shopify has no record of the ticket. Go to your event app's dashboard, find the customer's attendee record by email or order number, and use the "Resend ticket" function. If no attendee record exists in the app, the ticket was never generated (webhook failure). In that case, manually create the attendee record in the app using the Shopify order details, then use the app's resend function to send the ticket.
Why does Shopify oversell event tickets?
Shopify's inventory system is per-variant and does not understand shared venue capacity. If a merchant creates an event product with variants (General Admission: 150 qty, VIP: 80 qty) for a venue that holds 200 total, Shopify can sell up to 230 tickets because it does not know the variants share the same space. Event capacity must be enforced by the event app, not by Shopify inventory alone. Set a total event capacity in your event app's dashboard and treat Shopify's variant inventory counts as a secondary guardrail, not the primary capacity control.
What happens when a customer uses Buy Now on an event ticket product?
The Buy Now button skips the cart and goes directly to checkout. For event ticket products, this bypasses any attendee information fields that your event app collects on the product page (via an App Block) or on the cart page. The order is created and the ticket is generated — but the attendee record has no name, seat preference, dietary restriction, or other required event data. To prevent this: create a custom product template for event tickets that removes the Buy Now button section, or use Shopify Checkout Extensibility (Shopify Plus) to collect required attendee fields at checkout instead of on the product or cart page.
