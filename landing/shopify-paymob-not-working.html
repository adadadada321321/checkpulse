Shopify Paymob Not Working — Cards, Mobile Wallets, Fawry, InstaPay & 3DS Issues

Updated June 2026 · 7 failure modes · Egypt & MENA merchants · Multi-method payment gateway

Paymob is Egypt's largest and most widely adopted payment gateway, aggregating cards (Visa, Mastercard, Meeza), mobile wallets (Vodafone Cash, Orange Cash, e& Cash, WE Pay), Fawry offline payments, InstaPay (Egypt's real-time interbank network), and BNPL methods (Shahry, valU) under a single merchant account. Unlike single-method BNPL providers, Paymob manages multiple distinct payment integrations — each with its own integration ID, activation state, and failure modes. When Paymob stops working on Shopify, the root cause is almost always a specific method's integration being misconfigured, not a gateway-wide failure. This guide covers all seven failure modes merchants encounter.

How Paymob Differs From Other Shopify Payment Providers in Egypt

Paymob is a payment gateway aggregator, not a single payment method. This is the most important distinction for Shopify merchants. When you integrate Paymob, you are connecting a gateway that can surface multiple payment methods at checkout — but each method (cards, Vodafone Cash, Fawry, InstaPay) is a separate Paymob "integration" with its own numeric integration ID. You must configure each integration ID you want to offer, separately, inside the Paymob Shopify app. This is why Paymob troubleshooting is more complex than BNPL providers like Tabby or Shahry, which are single-method integrations.

Paymob is not Egypt-exclusive. It operates across Egypt, UAE, Saudi Arabia, Kuwait, Oman, Pakistan, and Palestine. However, Egypt is its home market and the market with the broadest method coverage — including Egypt-specific methods like Meeza (national debit), mobile wallets from all four Egyptian telecoms (Vodafone Egypt, Orange Egypt, e& Egypt, WE), and Fawry (Egypt's largest offline payment network with over 300,000 agent locations). For UAE and KSA merchants, Paymob is primarily a card gateway, not a multi-method aggregator.

Paymob sits between Shopify and every other provider. When a merchant also offers Shahry or valU via Paymob — rather than directly via their own Shopify apps — those BNPL methods appear as Paymob integrations, not as standalone Shopify payment providers. This matters for troubleshooting: a Shahry failure on a Paymob-integrated store may be a Paymob integration ID issue, not a Shahry app configuration issue.

Paymob's Shopify integration requires KYB completion. Unlike PayPal or Stripe (which use instant or near-instant onboarding for some account types), Paymob requires Egyptian commercial registration, Tax Card, and a KYB review process before the merchant account goes live. Until KYB is approved, the Paymob account is in test/sandbox mode — payment methods may appear in the Shopify checkout UI but will fail when customers attempt to pay.

7 Failure Modes Covered

  1. Paymob not showing at checkout (integration IDs not configured)
  2. Mobile wallets not appearing (Vodafone Cash / Orange Cash / e& Cash / WE Pay)
  3. Card payment declined (3DS / Meeza / bank restrictions)
  4. Fawry reference not generated or order not updating to paid
  5. 3D Secure / bank OTP failure
  6. InstaPay not configured or not showing
  7. Merchant KYB pending — test mode blocking live payments
1

Paymob Not Showing at Checkout

Merchant-Side

Paymob not appearing at Shopify checkout is the most reported issue, and the root cause is almost always missing integration IDs — not a Shopify-Paymob connection failure. Paymob works differently from other payment providers: connecting your Paymob account to Shopify enables the gateway, but the specific payment methods that appear at checkout depend on which integration IDs you configure inside the Paymob Shopify app's settings panel.

Each payment method in Paymob — cards, Vodafone Cash, Orange Cash, Fawry, InstaPay — is a separate "integration" inside your Paymob merchant dashboard, each with its own numeric integration ID (typically a 5–6 digit number). If you do not copy these integration IDs into the Paymob Shopify app's configuration fields for each method, those methods will not appear at checkout. Merchants commonly configure only the card integration ID and overlook mobile wallet and Fawry integration IDs entirely.

A second common cause is the Paymob Shopify app being activated in test mode. In test mode, the app connects to Paymob's sandbox environment using test API keys — checkout UI elements may appear but transactions will fail when submitted. The visual appearance of the checkout is identical whether Paymob is in test or live mode, so merchants who go live without switching to production API keys experience a working-looking but non-functioning checkout.

Paymob also requires that you enable it as an alternative payment provider in Shopify's payment settings. Installing the Paymob app from the Shopify App Store is the first step — but you must also navigate to Shopify Admin → Settings → Payments → Alternative payment providers and ensure Paymob is set to Active.

How to Fix

  1. Go to Shopify Admin → Settings → Payments → Alternative payment providers — confirm Paymob is listed and set to Active
  2. Open the Paymob Shopify app settings → verify your API key is the live production key, not the sandbox/test key
  3. Log into your Paymob merchant dashboard → Developers → Integration IDs — find the integration ID for each payment method you want to offer
  4. In the Paymob Shopify app settings, enter each integration ID in the corresponding field (card integration ID, Vodafone Cash integration ID, Fawry integration ID, etc.)
  5. Save the settings and test checkout with a real shipping address in Egypt — each configured method should now appear
  6. If methods still do not appear, confirm your Paymob account KYB is approved (not pending) — see Failure Mode 7
Integration IDs vs API keys: These are two different things in Paymob. Your API key connects Shopify to your Paymob account. Integration IDs tell Paymob which specific payment methods to surface. Both must be correctly configured — the API key alone is not sufficient to make payment methods appear.
2

Mobile Wallets Not Appearing (Vodafone Cash / Orange Cash / e& Cash / WE Pay)

Merchant-Side

Egypt's four main mobile wallet platforms — Vodafone Cash, Orange Cash, e& Cash (formerly Etisalat Cash), and WE Pay — are among the most used payment methods for Egyptian online shoppers who do not have bank accounts or credit cards. Offering these wallets via Paymob significantly expands the addressable customer base in Egypt, particularly for customers in the 18–35 age range with smartphones but without formal banking.

Each mobile wallet is a separate Paymob integration. In your Paymob merchant dashboard, you will find individual integration IDs for Vodafone Cash, Orange Cash, e& Cash, and WE Pay — each distinct from your card integration ID and Fawry integration ID. If you configure only the card integration ID in the Paymob Shopify app, only card payments will appear at checkout. Mobile wallets will be invisible, causing silent checkout abandonment from the segment of customers who specifically came to your store expecting to pay via wallet.

When a customer selects a mobile wallet at checkout powered by Paymob, they are redirected to the wallet's payment page (or receive an OTP on their wallet-registered mobile number) to confirm the payment. The wallet must have sufficient balance, and the customer's mobile number must be the one registered with that wallet. A customer who uses Vodafone Cash on their Vodafone SIM cannot use Orange Cash unless they also have an Orange SIM with an active Orange Cash wallet.

Some mobile wallet methods in Paymob are only available after you apply to enable them in your Paymob merchant account. Not all wallet integrations are automatically available — you may need to request activation of specific wallets from your Paymob account manager.

How to Fix

  1. Log into Paymob merchant dashboard → Developers → Integration IDs — look for wallet-specific integrations (Vodafone Cash, Orange Cash, e& Cash, WE Pay)
  2. If wallet integrations are not listed, contact your Paymob account manager to request activation for the mobile wallet methods you want to offer
  3. Once wallet integration IDs appear in your dashboard, copy each ID into the Paymob Shopify app settings in the corresponding wallet field
  4. Test by going through checkout with an Egyptian shipping address — wallet options should now appear alongside card payment
  5. If a specific wallet appears but transactions fail, verify the customer's mobile number matches their registered wallet number and the wallet has sufficient balance
Per-wallet integration IDs: Vodafone Cash, Orange Cash, e& Cash, and WE Pay each have their own integration ID — they are not interchangeable. You must configure each wallet's ID separately. A merchant who only enables Vodafone Cash will not show Orange Cash to customers, even if both use Paymob.
3

Card Payment Declined

Customer-Side

Card declines through Paymob on Shopify have several distinct causes depending on the card type. Paymob processes Egyptian domestic cards (Visa and Mastercard issued by Egyptian banks), Meeza cards (Egypt's national debit card scheme operated by NAPS — National Automated Payment System), and international cards. Each category has different decline patterns.

Egyptian bank-issued Visa and Mastercard debit cards frequently fail on e-commerce due to restrictions set by the issuing bank. Many Egyptian banks require customers to manually activate their card for online (card-not-present) transactions through their mobile banking app or by calling the bank. Customers with a physical debit card that works at ATMs but has never been used online may hit this restriction without knowing it exists.

Meeza cards — Egypt's national debit scheme launched in 2019 — require a separate Paymob integration specifically for Meeza acceptance. If the Paymob card integration is configured for Visa/Mastercard only and the Meeza integration ID is not added, Meeza cards will be declined even when the customer correctly selects card payment. Meeza is the default card type for government employees and many public sector workers in Egypt — a meaningful segment of the Egyptian consumer base.

International cards (Visa/Mastercard issued outside Egypt) may face higher decline rates from Paymob due to Egypt's foreign exchange controls and the regulatory environment for cross-border card-not-present transactions. Egyptian merchants selling primarily to local Egyptian customers should ensure their Paymob integration is optimized for EGP-denominated domestic transactions rather than defaulting to international card processing flows.

How to Diagnose Card Declines

  1. Check the Paymob merchant dashboard → Transactions for the specific decline reason code — Paymob provides issuer response codes for card declines
  2. If the decline code indicates online transactions not enabled, tell the customer to contact their bank to activate online/e-commerce transactions on their card
  3. For Meeza card declines, confirm the Meeza integration ID is configured in your Paymob Shopify app settings — Meeza requires a separate integration from Visa/Mastercard
  4. For 3D Secure failures, see Failure Mode 5 — OTP is the most common Meeza and Egyptian domestic card decline reason
  5. For international card declines, contact your Paymob account manager about international card acceptance settings on your merchant account
  6. Verify the customer entered the correct billing details, CVV, and card expiry — Paymob applies AVS/CVV validation for some card types
Meeza vs Visa/Mastercard: Meeza is Egypt's national debit card scheme and is not the same as a Visa or Mastercard debit card. Meeza cards display the Meeza logo and are issued by Egyptian banks. They require the Meeza integration ID in Paymob — separate from the Visa/Mastercard integration. If you only have a Visa/Mastercard integration configured, Meeza cards will be declined.
4

Fawry Reference Not Generated or Order Not Updating to Paid

Merchant & Customer

Fawry is Egypt's largest offline payment network, with over 300,000 agent locations including supermarkets, pharmacies, and banks. When a customer selects Fawry at checkout, Paymob generates a unique Fawry reference code. The customer then pays this reference at any Fawry outlet within the code's validity window — typically 24 to 72 hours. The order remains in a pending state in Shopify until Fawry notifies Paymob of the payment, at which point the order automatically updates to paid.

If no Fawry reference code is generated at checkout, the Fawry integration ID is missing from the Paymob Shopify app configuration. This is the most common Fawry-specific failure. Go to your Paymob merchant dashboard, find the Fawry integration ID under Developers → Integration IDs, and add it to the Paymob Shopify app settings.

A more operationally complex failure occurs when the Fawry reference is successfully generated but the Shopify order never updates from pending to paid — even after the customer has paid at a Fawry agent. This is almost always caused by a missing or misconfigured webhook (IPN — Instant Payment Notification) from Paymob to Shopify. Paymob must send a payment confirmation webhook to Shopify when Fawry confirms the payment. If this webhook is not configured or the Shopify URL Paymob is sending to is incorrect, Fawry payments never trigger order fulfillment.

Reference code expiry is a third failure mode. Fawry references have a configurable validity window set in your Paymob account. If a customer does not pay within this window, the reference expires and the pending order in Shopify will never receive a payment notification. The customer must initiate a new checkout to generate a new reference. Consider setting a longer validity window (48–72 hours) and communicating the expiry deadline clearly to customers after order placement.

How to Fix Fawry Issues

  1. Confirm the Fawry integration ID is configured in the Paymob Shopify app settings — find it in Paymob dashboard → Developers → Integration IDs
  2. Check your Paymob webhook settings — the IPN/callback URL must point to your Shopify store's Paymob webhook endpoint (found in the Paymob Shopify app settings or Shopify Payments documentation for the app)
  3. For pending orders that were paid at Fawry but not updated in Shopify, go to Paymob dashboard → Transactions and confirm the payment is recorded there — if it is, the issue is the Shopify webhook; if not, Fawry has not yet confirmed the payment
  4. Set an appropriate Fawry reference validity period in your Paymob account (48–72 hours recommended) — shorter periods increase expired-reference complaints
  5. Display the Fawry reference code and payment deadline clearly in the order confirmation email — customers may not know the code expires
  6. For expired references, the customer must place a new order — Fawry references cannot be regenerated for the same order
Order fulfillment risk: Never fulfill a Fawry order until Paymob has confirmed the payment in your Shopify dashboard. A pending order with a Fawry reference is not a paid order — the customer may never pay. Configure your fulfillment workflow to only process Fawry orders with confirmed paid status.
5

3D Secure / Bank OTP Failure

Customer-Side

Egypt's banking system mandates 3D Secure (3DS) authentication for card-not-present online transactions. When a customer pays by card through Paymob, their issuing bank sends a one-time password (OTP) to the mobile phone number registered with their bank account. The customer must enter this OTP within a time window (typically 60–90 seconds) to complete the payment. If the OTP is not received, entered incorrectly, or entered after expiry, the transaction fails with a 3DS authentication failure — even though there is nothing wrong with the card itself.

The most common cause of 3DS OTP failure is a mismatch between the customer's current mobile number and the number registered with their bank. Egyptian customers frequently change SIM cards or phone numbers without updating their bank records. The bank sends the OTP to the old registered number, which the customer no longer controls. The solution is for the customer to contact their bank and update their registered mobile number — this cannot be fixed at checkout.

International roaming causes a related issue. Egyptian customers traveling abroad with their Egyptian SIM may experience OTP delivery delays due to SMS roaming. Egyptian banks do not always support OTP delivery to roaming SIMs reliably. Customers in this situation may receive OTPs with delays of several minutes — past the bank's OTP expiry window — causing authentication failures even though the card is valid.

Some Egyptian banks have implemented 3DS in a way that requires the customer to be enrolled in their mobile banking app or to have previously activated online banking. First-time online shoppers with older bank accounts may encounter 3DS failures because their bank has not automatically enrolled them. The fix requires the customer to activate online banking through their branch or mobile app.

How to Diagnose 3DS OTP Failures

  1. Check the Paymob transaction log for the specific error — 3DS failures typically appear as "Authentication failed" or "3DS verification failed"
  2. Ask the customer: "Did you receive an OTP SMS from your bank?" — if no, the issue is the registered phone number or OTP delivery; if yes but it failed, it likely expired
  3. If the customer did not receive the OTP, advise them to contact their bank to verify and update their registered mobile number
  4. If the customer received the OTP but it expired before entry, advise them to have the OTP ready before clicking "Pay" — the bank sends it immediately when the 3DS step loads
  5. For traveling customers with OTP delays, advise them to use a mobile wallet (Vodafone Cash, Orange Cash) which does not require 3DS, or to use their bank's mobile app as an alternative payment channel
  6. For customers who never activated online banking, they must contact their bank to enable card-not-present transactions
Alternative for 3DS-blocked customers: Mobile wallets (Vodafone Cash, Orange Cash) do not use the same 3DS OTP mechanism as bank cards — they have their own wallet-level OTP. If a customer cannot complete 3DS for their card, offering them a mobile wallet payment option often resolves the checkout abandonment without requiring any bank interaction.
6

InstaPay Not Configured or Not Showing

Merchant-Side

InstaPay is Egypt's national real-time interbank payment system, launched by the Central Bank of Egypt (CBE) in 2022 and integrated into major Egyptian banks' mobile apps. It allows instant transfers between any two bank accounts or registered wallets in Egypt using a mobile number or IPA (InstaPay Address) as the identifier, without needing a recipient's bank account number. InstaPay has grown rapidly among Egyptian consumers as a preferred method for peer-to-peer and merchant payments.

Paymob supports InstaPay as a payment method for Shopify merchants through a dedicated InstaPay integration ID. However, InstaPay is newer than Paymob's card and wallet integrations, and many merchants who set up their Paymob integration before InstaPay became available have not added the InstaPay integration to their checkout. Customers who prefer InstaPay — a growing segment, particularly among younger Egyptian consumers — will not see it as an option and may abandon checkout.

InstaPay integration via Paymob typically requires the merchant to be enrolled in Paymob's InstaPay partnership — this may involve a separate activation request to your Paymob account manager. Not all Paymob merchant accounts have InstaPay automatically available. If InstaPay is not listed as an integration option in your Paymob dashboard, request activation from Paymob's merchant support team.

When InstaPay is available and configured, the checkout flow redirects the customer to their bank's mobile app or the InstaPay interface to confirm the transfer. Unlike Fawry (which has an expiry window), InstaPay transfers complete in real-time. However, the webhook/IPN mechanism for InstaPay payment confirmation to Shopify must be configured — the same requirement as Fawry — to ensure the Shopify order updates automatically when the customer completes payment.

How to Add InstaPay to Paymob Checkout

  1. Log into Paymob merchant dashboard → Developers → Integration IDs — look for an InstaPay integration
  2. If no InstaPay integration is listed, contact your Paymob account manager to request InstaPay activation on your merchant account
  3. Once activated and an integration ID is issued, add the InstaPay integration ID to the Paymob Shopify app settings in the InstaPay field
  4. Confirm your webhook/IPN configuration handles InstaPay payment notifications — InstaPay is real-time but still requires a payment confirmation webhook to update the Shopify order
  5. Test checkout with an Egyptian shipping address — InstaPay should now appear alongside cards and wallets
Why offer InstaPay: InstaPay is growing fastest among Egyptian consumers aged 18–35 who actively use mobile banking apps. For merchants targeting young Egyptian consumers, InstaPay can meaningfully reduce checkout abandonment compared to requiring a card (which requires 3DS) or Fawry (which requires an in-person visit). It is particularly valuable for customers who have limited card credit but a funded bank account.

Merchant KYB Pending — Test Mode Blocking Live Payments

Merchant-Side

Paymob requires all merchants to pass a KYB (Know Your Business) review before their account is activated for live transaction processing. For Egyptian merchants, this involves submitting Egyptian commercial registration (سجل تجاري), Tax Card (بطاقة ضريبية), business bank account details, and business owner identity documents. Paymob's compliance team reviews these before setting the merchant account to Live status.

During KYB review — which typically takes 3 to 7 business days but can extend to 2 weeks for certain business categories or for documents requiring additional verification — the Paymob merchant account operates in test/sandbox mode. In this mode, the Paymob Shopify app can be installed, integration IDs configured, and the checkout UI will appear to work. Customers will see Paymob payment options. However, all payment attempts will fail at the final step — either returning a generic error or timing out — because the account is not authorized to process real transactions.

This is one of the most operationally dangerous Paymob failure modes because it is entirely invisible to the merchant without checking the Paymob dashboard. There is no visual difference at Shopify checkout between a merchant account in test mode and a live account. Merchants who go live with their Shopify store before confirming Paymob KYB approval lose real customers to payment failure without any indication that configuration is the cause.

Paymob may also periodically place existing live merchant accounts on compliance hold for re-verification — particularly if commercial registration or Tax Card has expired, if the business changed its legal structure, or if Paymob's compliance team flagged the account for periodic review. An account that was working can stop processing payments if it is placed on hold for re-verification.

How to Resolve KYB Issues

  1. Log into the Paymob merchant dashboard and check your account status in the dashboard home or under Settings → Account — it should show Live / Active, not Pending Review or Test Mode
  2. If your account shows Pending Review, contact Paymob merchant support to confirm receipt of your documents and expected approval timeline
  3. Ensure your Egyptian commercial registration and Tax Card are current — expired documents are the most common cause of KYB delays and compliance holds
  4. Upload all required documents directly to the Paymob merchant portal — do not send documents only by email without confirming they are attached to your account record
  5. If you were previously live and then experienced a payment stoppage, check for a compliance hold email from Paymob — Paymob sends notifications when accounts are placed on hold but these can end up in spam filters
  6. After KYB approval, verify that the API key configured in the Paymob Shopify app is the live key, not the sandbox key — switching from test to live requires updating the API key in the app settings
Critical pre-launch check: Never launch your Shopify store with Paymob configured without first confirming your Paymob account shows "Live" status in the merchant dashboard. A test-mode account looks functional but will fail every payment attempt by real customers. This is the most common cause of "Paymob stops working after launch" reports.

Frequently Asked

Common Questions About Paymob on Shopify

Can I use Paymob together with Tabby or Shahry on the same Shopify store?

Yes — in two ways. First, you can have both Paymob (configured for cards and wallets) and Tabby or Shahry (configured as separate alternative payment providers directly via their own Shopify apps) active simultaneously in Shopify's payment settings. Each provider handles its own checkout flow and neither interferes with the other. Second, Paymob also supports Shahry and valU as BNPL integrations within the Paymob gateway itself — allowing merchants who want a single gateway to manage cards, wallets, Fawry, and BNPL all under one Paymob merchant account. Choose the approach that matches your integration preference — both work on Shopify.

Why did Paymob stop working after it was working before?

A previously working Paymob integration can stop for several reasons: the merchant's commercial registration or Tax Card expired, triggering a compliance hold; Paymob pushed an API version update requiring the Shopify app to be updated; the webhook URL configuration was accidentally changed or reset, causing order status updates to stop; or Paymob rotated API keys and the merchant's Shopify app is still using the old key. Check the Paymob merchant dashboard for account status and any compliance notifications. If the account is Active, verify the API key in the Paymob Shopify app matches the current live key in your Paymob dashboard. If the API key matches, check the webhook configuration to ensure callbacks are reaching Shopify.

Does Paymob work with Shopify's one-page checkout?

Paymob's compatibility with Shopify's one-page checkout depends on the Paymob Shopify app version. The Paymob Shopify app is periodically updated to maintain compatibility with Shopify's checkout architecture changes. If you are on Shopify's one-page checkout and Paymob methods are not appearing, first update the Paymob Shopify app to its latest version. If methods still do not appear, contact Paymob merchant support — one-page checkout compatibility may require Paymob to update their integration on their side. Some payment methods offered through Paymob (particularly mobile wallets) may have specific constraints on one-page checkout due to the redirect flow required to complete wallet authentication.
