Shopify Order Notes Not Saving — Cart Note Missing, Special Instructions Lost at Checkout

Shopify order notes live on the cart page, not inside checkout. Express checkout buttons (Shop Pay, Apple Pay, Google Pay, Buy Now) bypass the cart entirely — those customers can never add a note. If your note field is missing, notes disappear silently, or special instructions don't reach the order, here is the complete diagnostic.

📅 Updated June 2026 ⏱ 8 min read 💡 Applies to: All Shopify plans (Plus required for checkout UI extensions)
Critical Architecture Fact
Shopify's order note field is on the cart page (/cart), not inside the checkout flow. Once a customer clicks "Check out", they cannot add a note. Any customer who uses an express checkout button (Buy Now, Shop Pay, Apple Pay, Google Pay) bypasses the cart page and has no way to add an order note at all.

How Shopify Order Notes Work (and Where They Break)

Shopify stores order notes in the cart.note field. When checkout completes, this value becomes order.note on the order record. The note appears in Shopify Admin on the order detail page, in the order timeline, and in notification email templates that include the {{ note }} Liquid variable.

What most merchants don't realize:

  • The note field must be enabled in your cart template — it's hidden or absent in many themes by default
  • Express checkout completely skips the cart page — no note is possible for those customers
  • Cart attributes (note_attributes) are a separate field used by apps — they appear differently on orders than the main note
  • A note field inside checkout requires Shopify Plus + Checkout UI Extensions
  • Cart-modifying apps can silently strip the note when they update the cart object

Failure Mode 1 — Note Field Not Visible on Cart Page

The most common cause: the cart template doesn't render the note textarea. This is a theme setting, not a Shopify Admin setting — you must enable or add it at the theme level.

Option A — Theme has a built-in toggle (Dawn and most modern themes)

  1. Go to Shopify Admin → Online Store → Themes → Customize
  2. Navigate to the Cart page in the theme editor (not the checkout page)
  3. In the left panel, click on the Cart section or look for a block labelled "Order note" or "Cart note"
  4. Toggle "Show order note" or "Enable cart note" to ON
  5. Click Save and test in a fresh browser window at /cart
Dawn Theme Specific
In Dawn, go to Customize → Cart page → click the "Cart" section in the left sidebar → look for "Enable order note" checkbox in the section settings panel on the right. It defaults to OFF in some Dawn versions.

Option B — Theme requires code edit

If your theme has no toggle, add the note textarea to sections/cart-template.liquid or templates/cart.liquid inside the cart form:

What to addWhere
<textarea name="note" id="CartSpecialInstructions">{{ cart.note }}</textarea> Inside the <form action="/cart" method="post"> tag on the cart page, before the checkout button

The name="note" attribute is required — this is what Shopify reads when the cart form is submitted. Without it, the value is silently ignored.

Failure Mode 2 — Express Checkout Bypasses the Cart Page

This affects a large and growing share of checkouts. Every express checkout flow skips the cart page entirely — and therefore skips the order note field.

Checkout Method Cart Page Shown Order Note Available
Add to Cart → standard Check out button Yes Yes — if note field enabled in theme
Buy Now (product page button) No No — cart page bypassed
Shop Pay express No No — direct to payment confirmation
Apple Pay No No — native payment sheet
Google Pay No No — native payment sheet
Shopify Buy Button (embedded) No No — goes directly to checkout
Cart permalink (/cart/add?...) Depends Only if permalink includes ¬e= param
Workaround for Express Checkout
Add a notice near the Buy Now button and express payment buttons on the product page: "Need to add special instructions? Use 'Add to Cart' instead." This is the only native solution for non-Plus stores. Shopify Plus stores can add a note field inside checkout via Checkout UI Extensions.

Failure Mode 3 — Cart-Modifying App Strips the Note

Apps that modify the cart object — discount apps, upsell apps, cart drawer apps, gift wrapping apps — make API calls to update cart contents. If those API calls use PUT /cart/update.js or POST /cart/change.js without preserving the existing note field, the note is silently overwritten with an empty string.

How to diagnose

  1. Enter an order note in the cart note field
  2. Apply a discount code, add an upsell item, or trigger any cart modification app action
  3. Check if the note field is now empty — if so, that app is the culprit
  4. Test by disabling cart-related apps one at a time in Shopify Admin → Apps
  5. Contact the app developer: they must include "note": document.getElementById("CartSpecialInstructions").value in their cart update payload
Silent Data Loss
Shopify's cart update API silently accepts an empty note field and overwrites the existing note with nothing. There is no error, no warning to the customer, and no indication in Shopify Admin that a note was lost. This is one of the most underdiagnosed checkout data loss issues in Shopify stores.

Failure Mode 4 — Order Note Not Showing in Admin or Emails

The note saved correctly but isn't appearing where expected.

Finding the note in Shopify Admin

Order notes appear in Shopify Admin → Orders → [order number] → "Notes" section in the right sidebar. If you don't see a Notes section or it appears blank, the note was empty when the order was placed. Notes also appear in the order timeline at the bottom of the page when they are present.

Adding the note to order confirmation emails

Shopify's default order confirmation template includes the note by default. If it's missing from your emails, your template was likely customized and the note block was removed. To restore it:

  1. Go to Shopify Admin → Settings → Notifications → Order confirmation
  2. Click "Edit code"
  3. Search for {% if note %} in the template
  4. If the block is missing, add it in the appropriate location — the default Shopify template places it just before the line items table
  5. The block should contain: {% if note %}<p>Note: {{ note }}</p>{% endif %}
  6. Save and send a test notification to verify

Order note in packing slips

Third-party packing slip apps (Order Printer, Sufio, Printout Designer) control their own templates. Check the template settings for each app — you may need to manually add {{ order.note }} to the packing slip template. The Shopify-native Order Printer app uses {{ note }} in order templates.

Failure Mode 5 — Order Note vs Cart Attribute Confusion

Many apps that add "additional information" fields to the cart use cart attributes (cart.attributes), not the order note (cart.note). These end up as order.note_attributes on the order, not order.note. They look different in Admin and email templates.

Field Type Cart Variable Order Variable How It Shows in Admin
Order note cart.note order.note "Notes" section in order sidebar
Cart attribute cart.attributes["key"] order.note_attributes "Additional details" section below line items
Line item property item.properties["key"] line_item.properties Shown per line item in order detail

If you're looking for a note in the "Notes" section of the order and it's empty — but you know the customer filled in a field — check the "Additional details" section. The field may be saving as a cart attribute, not the note.

Failure Mode 6 — Cart Permalink or API Order Without Note

Cart permalinks

Shopify cart permalinks (/cart/add?id=...&quantity=1) add items to the cart but do not carry a note. If you're using a cart permalink as a direct-to-checkout link (adding &checkout=true), the customer skips the cart page and any existing note in the cart may be overwritten. To include a note in a cart permalink, append ¬e=Your+note+here to the URL — Shopify will populate the cart note field with that value.

Orders via Admin API or Storefront API

Orders created programmatically via the Shopify Admin API or Storefront API must explicitly set the note field. For Admin API: include "note": "special instructions" in the order creation payload. For Storefront API cart mutations (cartNoteUpdate): call cartNoteUpdate(cartId: ..., note: "...") before initiating checkout. If the note field is omitted from the API call, it defaults to null — no error is thrown.

Failure Mode 7 — Adding Note Field Inside Checkout (Plus Only)

Non-Plus merchants frequently ask how to add a "special instructions" or "delivery notes" field inside the checkout flow itself — not on the cart page. This is only possible on Shopify Plus.

Shopify Plus — Checkout UI Extension approach

  1. Create a Checkout UI Extension using the Shopify CLI: shopify app generate extension --type checkout_ui_extension
  2. Use the TextField component to render a text input at a target position in checkout (e.g., purchase.checkout.shipping-option-list.render-after)
  3. Save the customer's input as a cart attribute using the applyAttributeChange API — it saves as a key-value pair under order.note_attributes
  4. Alternatively, use applyMetafieldChange to save it as an order metafield for more structured access
  5. Update your notification templates and packing slips to display {% for attribute in note_attributes %}{{ attribute.name }}: {{ attribute.value }}{% endfor %}
Plan Requirement
Checkout UI Extensions require Shopify Plus and the new checkout extensibility platform. Basic, Shopify, and Advanced plans cannot add custom fields inside the checkout flow. The only native option for those plans is the cart page note field.

Frequently Asked Questions

Can I make the order note field required at checkout?

Not natively. Shopify's cart page doesn't support required field validation on the note textarea without custom JavaScript. If you add a note field via Checkout UI Extensions (Shopify Plus), you can mark it as required using the required prop on the TextField component — the checkout will block progress until it's filled. For non-Plus stores, you need a small JavaScript snippet on the cart page that checks the note textarea before allowing the "Check out" button click.

Does the order note count against the character limit in Shopify?

Shopify's order note field accepts up to 5,000 characters. The cart note textarea in most themes doesn't enforce this visually — a customer can type more than 5,000 characters in the textarea, but the value will be truncated to 5,000 when saved to the cart and subsequently to the order. If you're collecting lengthy notes (e.g., detailed delivery instructions), add a maxlength="5000" attribute to the textarea and a visible character counter via JavaScript.

Are order notes visible to customers after checkout?

Order notes appear on the order confirmation page immediately after checkout completes, and in the order confirmation email if the template includes the {{ note }} variable. Customers can also see their note on the order status page (the post-checkout tracking page) and in their account order history if you include it in the customer account order template. They cannot edit the note after the order is placed from the storefront — only merchants can edit notes in Shopify Admin.

How do I access order notes via the Shopify API?

Via the Admin REST API: GET /admin/api/2024-01/orders/{order_id}.json — the note is at order.note. Via GraphQL Admin API: query the Order object and include the note field. For cart attributes (note_attributes): the order object has a noteAttributes array (GraphQL) or note_attributes array (REST) where each element has name and value. Via Storefront API: use cartNoteUpdate mutation to set the note during the cart phase, and it transfers to the order on checkout completion.
