Shopify Checkout Payment Retry Not Working — Declined Card Retry, 3DS Infinite Loop, Session Expiry, Guest Checkout No Recovery Path

Shopify has no native "retry payment" button for existing orders. The 3DS failure redirect loops back to the same declined card form. Guest checkout orders with failed payments have zero recovery path for the customer. Checkout sessions expire in 10–15 minutes, wiping all entered shipping info. Fraud filters block retry attempts from the same card even after the customer fixes the issue.

📅 Updated June 2026 ⏱ 12 min read 💡 Applies to: All Shopify plans
Critical — Shopify Has No Native Retry Payment Button for Completed Orders
On non-Plus Shopify plans, once a checkout session ends with a failed payment and an order record is created in "Payment pending" status, there is no one-click "retry payment" button for the customer or merchant. The customer must either start a new checkout, or the merchant must manually send an invoice link via Admin → Orders. This is the most common source of "how does my customer pay now?" confusion.

Failure Mode 1 — No Native Retry Payment Button for Existing Orders

This is the most frequently misunderstood behavior in Shopify's checkout. When a customer's payment is declined at the checkout payment step while they are still inside the active checkout session, they can immediately try a different card — the checkout page shows the error and keeps the payment form open. That is not the problem.

The problem occurs when the checkout session ends and an order record is created in Shopify with "Payment pending" status. This happens in two scenarios:

  • Capture-after-fulfillment stores: authorization succeeds at checkout but the actual capture (charge) is attempted later and fails. The order appears in Admin with "Authorized" status and then transitions to "Payment pending" or shows a capture failure in the order timeline.
  • Certain third-party payment gateways that create orders on authorization and then fail to process the final payment step — the customer receives an order confirmation email but the merchant never gets paid.

Once in this state, Shopify does not provide a "retry payment" or "pay now" link on the order status page. The customer's order confirmation email contains a link to the order status page, which shows their order details but has no payment action.

What merchants can do

  1. In Admin → Orders → [Order], look for a "Send invoice" button at the top right. If present, clicking this sends the customer a payment link via email to complete payment for that specific order.
  2. If "Send invoice" is not available (it's absent on some order types), create a new draft order with the same items in Admin → Orders → Create order, add the customer, and click "Send invoice" on the draft. Note the customer in the order that the original order will be cancelled once the new one is paid.
  3. Cancel the original unpaid order once the new payment is collected, to avoid fulfilling twice.
Prevention — Use Automatic Capture to Avoid "Authorized But Not Captured" Orders
Admin → Settings → Payments → Payment capture → set to "Automatically capture payment for orders". With automatic capture, payment is charged at checkout completion. If it fails, the customer sees the error immediately while still in the checkout session, and no order record is created. Manual capture introduces a window where orders exist without payment, creating the retry problem.

Failure Mode 2 — 3DS Redirect Loop Traps Customer

3D Secure (3DS) is the bank authentication step that pops up during checkout — the "Enter your OTP" or "Approve in your banking app" screen. When 3DS authentication fails, the payment gateway redirects the customer back to Shopify's checkout with an error message. The problem: the checkout page shows the error next to the same card that just failed, with no clear path to switch cards or try again.

How the loop forms

The sequence is:

  1. Customer enters card → clicks "Pay now" → bank 3DS popup appears
  2. Customer fails 3DS (wrong OTP, times out, bank app not responding)
  3. Gateway redirects back to Shopify checkout with "Payment failed" error
  4. Customer clicks "Try again" — same 3DS popup appears
  5. Customer fails 3DS again → back to step 3 indefinitely

The customer is stuck because they cannot easily switch payment methods from the error state, and retrying with the same card hits the same 3DS challenge (now with increasing bank-side friction from repeated failures).

How to break out of the 3DS loop

  1. Do NOT click "Try again" — this re-attempts 3DS with the same card and the same likely outcome.
  2. Look for a "Change" or "Edit" link next to the payment method. Click it to return to card entry.
  3. Enter a different card number or select a different payment method (PayPal, Shop Pay, Apple Pay/Google Pay if available).
  4. If the payment section is frozen and cannot be edited, do a hard refresh: Ctrl+Shift+R (Windows) / Cmd+Shift+R (Mac). This reloads the checkout page while keeping the session alive.
  5. If the hard refresh does not work, clear browser cookies and return to your cart page to start a fresh checkout session.
Bank Velocity Limits Apply After 3DS Failures
Most banks implement their own retry limits for 3DS — typically 3-5 failed authentication attempts trigger a temporary block on that card for that merchant. After hitting the limit, all subsequent payment attempts (even with the correct OTP) will be declined at the bank level regardless of what Shopify shows. The customer must wait 15–30 minutes or contact their bank before retrying the same card.

Failure Mode 3 — Checkout Session Expires During Retry Window

Shopify's hosted checkout sessions are not permanent. An inactive checkout session expires after approximately 10–15 minutes. For a customer who has just had their card declined and needs time to: look up a different card, call their bank, check their account balance, or switch to a different device — that window can close before they return to complete payment.

What happens when a session expires

When a customer returns to an expired checkout URL:

  • The checkout page may show an error: "This checkout is no longer available"
  • Or it may redirect to the cart page with items still present but all checkout progress (email, shipping address, delivery method) erased
  • The customer must restart checkout from the beginning
  • If they had applied a discount code, it may or may not reapply automatically depending on the code type

Reducing session expiry abandonment

  • Enable Shop Pay: Shop Pay stores the customer's shipping, billing, and payment information in Shopify's accelerated checkout system. Even after a session expires, returning customers who use Shop Pay have their information repopulated automatically, reducing friction on retry.
  • Abandoned checkout recovery: Shopify automatically triggers an abandoned checkout email after 1 hour if the customer entered their email. This brings them back to checkout with a recovery URL. Ensure abandoned recovery is enabled in Admin → Settings → Checkout.
  • Cart persistence messaging: add text near your cart that reassures customers their cart will be saved if they need to leave and return.

Failure Mode 4 — Guest Checkout Orders Have No Customer-Facing Retry Path

Guest checkouts — where customers complete purchase without creating a Shopify account — account for the majority of first-time purchases on most stores. When a guest checkout results in a "Payment pending" order (authorization succeeded but capture failed, or a delayed payment scenario), the customer has no way to retry payment on their own:

  • They have no Shopify account to log into
  • The order status page link (from their confirmation email) shows their order but provides no payment action
  • There is no "Pay now" button accessible without account login
  • The original checkout URL has expired

Merchant recovery steps for guest failed-payment orders

  1. In Admin → Orders → [Order], check the order timeline for payment failure details. Note the customer's email address from the order.
  2. Click "Send invoice" if the button is available — this generates a payment link and sends it to the customer's email. The link works for guest customers (no login required).
  3. If "Send invoice" is not available, go to Admin → Orders → Create order. Add the same products, select the customer by email, and click "Send invoice" — this sends the guest a payment link that does not require account creation.
  4. In the note field of the new draft, reference the original order number so your fulfillment team knows to cancel the original once payment is received.
New Customer Accounts Can Reduce Guest Retry Friction
Shopify's new customer accounts (launched 2023, available to all plans) allow customers to access their orders via email one-time login link — no password required. Customers who completed guest checkout can still log in via the email OTP link and potentially access payment options. Enable at Admin → Settings → Customer accounts → New customer accounts.

Failure Mode 5 — Fraud Filter Blocks Legitimate Retry Attempts

After a card decline, Shopify's fraud detection system and the payment gateway's own velocity controls may flag subsequent payment attempts from the same customer as high-risk. This is a common source of "I fixed my billing address but my card keeps getting declined" complaints from customers.

What triggers the fraud block

  • Multiple declined attempts in a short time: 3+ declined payment attempts within 10–15 minutes on the same store from the same IP triggers velocity controls
  • AVS mismatch on first attempt: if the first decline was due to billing address not matching, subsequent attempts (even with corrected address) may be pre-screened with elevated suspicion
  • CVV failure: a single CVV failure often results in the bank flagging that card number for the next 24 hours regardless of what the customer does
  • High Shopify risk score inherited: if the first failed attempt generated a "High" risk score in Shopify's system, subsequent orders from the same customer may inherit elevated risk scoring even with a new card

Resolution steps for customers experiencing repeated declines after fixing the issue

  1. Wait 15–30 minutes before retrying — bank-side velocity limits typically reset within this window.
  2. Try from a different browser or device (different IP address) — some velocity limits are IP-scoped.
  3. Try a different payment method entirely: PayPal, Shop Pay (if enrolled), Apple Pay, or Google Pay.
  4. Contact the issuing bank directly to confirm the card is not temporarily blocked and to authorize the merchant.
  5. As a merchant: if you can verify the customer is legitimate, contact Shopify support to review the order's risk score — Shopify support can sometimes override automatic declines for specific orders.

Failure Mode 6 — Subscription Renewal Retry Email Contains Wrong or Expired Link

Subscription businesses using ReCharge, Bold Subscriptions, Seal Subscriptions, or native Shopify subscription apps send "Update your payment method" emails when a recurring charge fails. The retry flow for subscriptions is entirely different from one-time checkout — the customer must update their card in the subscription management portal, not retry at Shopify's checkout.

Common subscription retry link failures

  • Link goes to wrong page: some app configurations send customers to their account dashboard rather than the specific payment update screen, causing confusion
  • Link has expired token: subscription payment update links typically have a 24–72 hour validity window. If the customer opens the email days later, the link returns an error
  • Multiple failed rebills before email: subscription apps typically retry automatically 3–5 times before sending the update email. By the time the customer receives the email, their subscription may already be in a "dunning" or "cancelled" state requiring reactivation, not just a card update
  • ReCharge-specific: after card update in ReCharge portal, the next billing cycle retry may not run immediately — customer may need to contact support to trigger a manual rebill
Subscription Retry ≠ Checkout Retry
A customer who received a "subscription renewal failed" email cannot retry through your store's checkout — they must use the subscription app's payment update flow. Sending them a checkout link for a new order will create a new subscription rather than renewing the existing one. Always direct subscription retry customers to their subscription management portal, not to the cart.

Failure Mode 7 — Silent Payment Capture Failure (Order Fulfilled Without Payment)

This is the highest-risk failure mode: an order is authorized at checkout, appears in Shopify Admin as "Authorized," and is then fulfilled (products shipped, download links sent, event tickets issued) before the merchant realizes the capture step failed.

How silent capture failure happens

  • Manual capture + staff fulfillment without checking: with manual capture enabled, someone on the team fulfills the order before checking that capture was completed
  • Authorization expiry: Shopify Payments and most gateways allow authorization holds of 7 days (some up to 30). If the merchant does not capture within this window, the authorization expires and capture will fail silently — the order shows as fulfilled but uncaptured
  • Gateway-side capture failure: in rare cases, the gateway accepts the authorization but fails the capture attempt with a transient error. The order timeline shows a capture failure entry, but if no one monitors the timeline, the order gets fulfilled
  • Shopify Flow automation fulfilling before capture: if you have a Flow that auto-fulfills on order creation, it may trigger before manual capture is completed

How to detect and prevent capture failures

  1. In Admin → Orders, filter by Payment status: Authorized to see orders that have an authorization hold but have not yet been captured. Review these daily.
  2. Check the order timeline for any "Payment capture failed" entries — these appear as red timeline events but do not send automatic merchant notifications.
  3. Set up a Shopify Flow automation: trigger on "Order payment status changes to Authorized" → wait 24 hours → if payment status is still "Authorized" → send internal notification email to your fulfillment team.
  4. Switch to automatic capture (Admin → Settings → Payments → Payment capture) unless you have a specific reason for manual capture (e.g., you need to verify inventory before charging).
Authorization Expiry Is Non-Recoverable Without Customer Action
Once a payment authorization expires, the only way to collect payment is to have the customer re-authorize — there is no way to re-capture an expired authorization. You must contact the customer and ask them to place a new order or send them a payment link for a new draft order. Expired authorizations are irreversible and cannot be extended.

Failure Mode 8 — "Send Invoice" Missing on Certain Order Types

The "Send invoice" button in Shopify Admin is the primary merchant tool for requesting payment on a pending order. However, it is not available on all order types:

Order Type "Send Invoice" Available? Alternative
Regular storefront order (Payment pending) Yes —
Draft order (unpaid) Yes (from draft screen) —
POS order with failed payment No Create draft order, send invoice
Exchange order (via Returns Manager) No Create draft order for exchange value
Order edited with balance due Sometimes Collect payment in-person or via draft
Subscription renewal failure No Subscription app payment update flow
B2B/Net Terms order (Plus) Yes (Net terms invoice) —
Draft Order Invoice = Universal Retry Payment Tool
When "Send invoice" is absent, create a new draft order (Admin → Orders → Create order). Add the same products, the same customer email, apply any applicable discounts, and click "Send invoice." The payment link in this email works for both logged-in customers and guests, requires no account, and can be paid with any enabled payment method on your store. This is the most reliable retry path for any order type where direct invoice is unavailable.
